# CBK-003: Block Reconstruction from Mempool

> **Authoritative requirement:** [NORMATIVE.md](../NORMATIVE.md)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **Spec reference:** [SPEC.md - Section 8.2](../../../resources/SPEC.md)

## Summary

When a node receives a `CompactBlock`, it reconstructs the full block by computing SipHash short IDs for all transactions in its local mempool using the `CompactBlock`'s `sip_hash_key`, matching those short IDs against the `short_tx_ids` list, and combining the matched mempool transactions with the prefilled transactions and block header. This is the core mechanism that enables 90%+ bandwidth savings: most transactions are already in the receiver's mempool and do not need to be retransmitted.

## Specification

### Reconstruction Algorithm

```rust
use std::collections::HashMap;

/// Result of attempting to reconstruct a full block from a CompactBlock.
pub struct ReconstructionResult {
    /// The full block header.
    pub header: BlockHeader,
    /// Successfully reconstructed transactions, in block order.
    /// `None` entries indicate missing transactions.
    pub transactions: Vec<Option<Vec<u8>>>,
    /// Indices of transactions that could not be matched.
    pub missing_indices: Vec<u16>,
}

impl CompactBlock {
    /// Attempt to reconstruct the full block using the local mempool.
    ///
    /// `mempool_txs` is an iterator of (tx_id, serialized_tx) pairs
    /// representing all transactions in the local mempool.
    pub fn reconstruct<'a, I>(
        &self,
        mempool_txs: I,
    ) -> ReconstructionResult
    where
        I: Iterator<Item = (&'a [u8], &'a [u8])>,
    {
        // Step 1: Build a lookup table from short ID -> mempool transaction
        let mut short_id_to_tx: HashMap<ShortTxId, &[u8]> = HashMap::new();
        for (tx_id, serialized_tx) in mempool_txs {
            let short_id = compute_short_tx_id(&self.sip_hash_key, tx_id);
            short_id_to_tx.insert(short_id, serialized_tx);
        }

        // Step 2: Build prefilled lookup
        let mut prefilled_map: HashMap<u16, &Vec<u8>> = HashMap::new();
        for prefilled in &self.prefilled_txs {
            prefilled_map.insert(prefilled.index, &prefilled.tx);
        }

        // Step 3: Determine total transaction count
        let total_txs = self.short_tx_ids.len() + self.prefilled_txs.len();

        // Step 4: Reconstruct transaction list in order
        let mut transactions = Vec::with_capacity(total_txs);
        let mut missing_indices = Vec::new();
        let mut short_id_idx = 0;

        for i in 0..total_txs as u16 {
            if let Some(tx) = prefilled_map.get(&i) {
                // Prefilled transaction -- use directly
                transactions.push(Some(tx.to_vec()));
            } else {
                // Must match from mempool via short ID
                let short_id = &self.short_tx_ids[short_id_idx];
                short_id_idx += 1;
                if let Some(tx) = short_id_to_tx.get(short_id) {
                    transactions.push(Some(tx.to_vec()));
                } else {
                    transactions.push(None);
                    missing_indices.push(i);
                }
            }
        }

        ReconstructionResult {
            header: self.header.clone(),
            transactions,
            missing_indices,
        }
    }
}
```

### Reconstruction Protocol Flow

```
Receiver:
  1. Receive CompactBlock
  2. For each transaction in local mempool:
     a. Compute SipHash(compact_block.sip_hash_key, tx_id)
     b. Truncate to 6 bytes -> short_id
     c. Store in lookup: short_id -> serialized_tx
  3. For each short_tx_id in compact_block.short_tx_ids:
     a. Look up short_id in mempool lookup table
     b. If found: slot the transaction into the block at the correct index
     c. If not found: record as missing
  4. Merge prefilled_txs into the transaction list at their specified indices
  5. If missing_indices is empty:
     -> Full block reconstructed successfully (header + all transactions)
  6. If missing_indices is non-empty:
     -> Proceed to CBK-004 (request missing) or CBK-005 (fallback)
```

## Acceptance Criteria

- [ ] `ReconstructionResult` contains `header`, `transactions` (with `Option` slots), and `missing_indices`
- [ ] SipHash is computed for each mempool transaction using the CompactBlock's `sip_hash_key`
- [ ] Mempool transactions are matched against `short_tx_ids` by their computed short IDs
- [ ] Prefilled transactions are placed at their specified indices without mempool lookup
- [ ] Transaction ordering in the reconstructed block matches the original block ordering
- [ ] Missing transactions (no mempool match) are recorded in `missing_indices`
- [ ] When all short_tx_ids match, `missing_indices` is empty and reconstruction is complete
- [ ] When no mempool transactions match, all non-prefilled indices are in `missing_indices`

## Implementation Notes

- Primary file: `src/gossip/compact_block.rs`
- Dependencies: `compute_short_tx_id` from CBK-002, mempool access (provided by caller)
- Edge cases:
  - SipHash collisions: two different mempool transactions may produce the same short ID. The implementation should handle this gracefully (e.g., by trying both and validating the resulting block). In practice, collisions are extremely rare (~1 in 2^48).
  - A transaction in the mempool may match a short ID but be the wrong transaction (collision). Block validation (performed by the caller after reconstruction) will catch this.
  - The mempool may contain transactions not in the block -- these are simply not matched and ignored.
  - Prefilled transaction indices must not overlap with short_tx_id positions -- the sender guarantees this per CBK-001.
  - Performance consideration: for large mempools (10,000+ transactions), the HashMap lookup is O(1) per short ID, making reconstruction efficient.
