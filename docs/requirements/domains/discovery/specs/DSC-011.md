# DSC-011: /16 Group Filter

> **Authoritative requirement:** [NORMATIVE.md](../NORMATIVE.md)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **Spec reference:** [SPEC.md - Section 6.4](../../../resources/SPEC.md)

## Summary

The /16 group filter enforces that at most one outbound connection is maintained per IPv4 /16 subnet (the first two octets of the IP address). This is a fast first-pass filter applied before the more expensive AS-level diversity check (DSC-010). It matches Chia's behavior in `node_discovery.py:296-306` and provides basic eclipse attack resistance by preventing an attacker with many IPs in the same /16 from dominating outbound connections.

## Specification

### /16 Group Computation

```rust
use std::net::IpAddr;

/// Compute the /16 group for an IP address.
/// For IPv4: first two octets (e.g., 192.168.x.x -> (192, 168))
/// For IPv6: first four bytes of the address (first /32)
pub fn get_group(ip: &IpAddr) -> u32 {
    match ip {
        IpAddr::V4(v4) => {
            let octets = v4.octets();
            ((octets[0] as u32) << 8) | (octets[1] as u32)
        }
        IpAddr::V6(v6) => {
            let octets = v6.octets();
            ((octets[0] as u32) << 24)
                | ((octets[1] as u32) << 16)
                | ((octets[2] as u32) << 8)
                | (octets[3] as u32)
        }
    }
}
```

### Group Filter

```rust
/// Filter that enforces at most one outbound connection per /16 group.
/// Fast first-pass before the AS-level diversity check (DSC-010).
/// Ported from node_discovery.py:296-306.
pub struct SubnetGroupFilter {
    /// /16 groups of current outbound connections.
    outbound_groups: HashSet<u32>,
}

impl SubnetGroupFilter {
    pub fn new() -> Self {
        Self {
            outbound_groups: HashSet::new(),
        }
    }

    /// Check whether a candidate peer is allowed given current /16 diversity.
    /// Returns true if the candidate's /16 group is not already represented.
    pub fn is_allowed(&self, ip: &IpAddr) -> bool {
        let group = get_group(ip);
        !self.outbound_groups.contains(&group)
    }

    /// Record that a new outbound connection has been established.
    pub fn add_outbound(&mut self, ip: &IpAddr) {
        let group = get_group(ip);
        self.outbound_groups.insert(group);
    }

    /// Record that an outbound connection has been closed.
    pub fn remove_outbound(&mut self, ip: &IpAddr) {
        let group = get_group(ip);
        self.outbound_groups.remove(&group);
    }
}
```

### Integration with Connection Establishment

The /16 filter is the first-pass check in the peer selection pipeline:

```rust
// In connect_batch (DSC-009):
for candidate in candidates {
    // 1. Fast first pass: /16 group filter
    if !subnet_filter.is_allowed(&candidate.peer_info.ip()) {
        continue; // already have a peer in this /16
    }
    // 2. Stronger second pass: AS-level diversity (DSC-010)
    if !as_filter.is_allowed(&candidate.peer_info.ip()) {
        continue; // already have a peer in this AS
    }
    // 3. Proceed with connection attempt
    futures.push(try_connect_with_info(candidate, config));
}
```

## Acceptance Criteria

- [ ] At most one outbound connection per IPv4 /16 subnet
- [ ] IPv4 /16 group is computed from the first two octets
- [ ] IPv6 group is computed from the first four bytes (/32)
- [ ] Filter is checked before AS-level check (fast first-pass)
- [ ] Group set is updated when outbound connections are established or closed
- [ ] `is_allowed()` returns true when no outbound exists in the candidate's group
- [ ] `is_allowed()` returns false when an outbound exists in the candidate's group
- [ ] Filter is consistent with Chia's `node_discovery.py:296-306` behavior

## Implementation Notes

- Primary files: `src/util/ip_address.rs` (get_group), `src/discovery/node_discovery.rs` (SubnetGroupFilter)
- Dependencies: `std::net` (IpAddr)
- Edge cases:
  - IPv6 addresses: use first 4 bytes as group (equivalent to /32 subnet)
  - Localhost (127.0.0.1): group is (127, 0) = 32512; unlikely to conflict with real peers
  - Private ranges (10.x.x.x, 192.168.x.x): each /16 is treated independently
  - Filter must be updated atomically with connection state: removing a closed connection and adding a new one in the same group should work correctly
  - Inbound connections are NOT subject to this filter -- it only applies to outbound
