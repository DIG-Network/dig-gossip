# DSC-003: DNS Seeding

> **Authoritative requirement:** [NORMATIVE.md](../NORMATIVE.md)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **Spec reference:** [SPEC.md - Section 6.2](../../../resources/SPEC.md)

## Summary

DNS seeding is the first method used to discover initial peers when the address manager is empty. It reuses `chia-sdk-client::Network::lookup_all()` which handles DNS resolution with timeout and batching. The DIG network configures its own DNS introducer hostnames rather than Chia's defaults.

## Specification

### Network Configuration

```rust
use chia_sdk_client::Network;
use std::time::Duration;

/// Create a Network configured for DIG DNS seeding.
fn create_dig_network(config: &GossipConfig) -> Network {
    Network {
        default_port: config.default_port, // DEFAULT_P2P_PORT
        genesis_challenge: config.genesis_challenge,
        dns_introducers: config.dns_introducers.clone(),
        // e.g., vec!["dns-introducer.dignetwork.org".to_string()]
    }
}
```

### DNS Lookup

```rust
/// Perform DNS seeding to discover initial peers.
/// Returns a list of socket addresses resolved from DNS introducers.
pub async fn dns_seed(
    network: &Network,
    timeout: Duration,
    batch_size: usize,
) -> Vec<SocketAddr> {
    // Delegates to chia-sdk-client::Network::lookup_all()
    network.lookup_all(timeout, batch_size).await
}
```

### Configuration

```rust
/// DNS seeding configuration (part of GossipConfig).
pub struct DnsSeedConfig {
    /// DNS introducer hostnames.
    pub dns_introducers: Vec<String>,
    /// Timeout for DNS resolution.
    pub dns_timeout: Duration, // default: 30 seconds
    /// Batch size for parallel DNS queries.
    pub dns_batch_size: usize, // default: 2
    /// Default P2P port for resolved addresses.
    pub default_port: u16,
}
```

### Integration with Address Manager

Resolved addresses from DNS seeding MUST be added to the address manager's new table:

```rust
// After DNS lookup
let addrs = dns_seed(&network, config.dns_timeout, config.dns_batch_size).await;
let timestamped: Vec<TimestampedPeerInfo> = addrs
    .iter()
    .map(|addr| TimestampedPeerInfo {
        host: addr.ip().to_string(),
        port: addr.port(),
        timestamp: current_timestamp(),
    })
    .collect();
address_manager.add_to_new_table(&timestamped, &self_peer_info, 0).await;
```

## Acceptance Criteria

- [ ] DNS seeding uses `chia-sdk-client::Network::lookup_all()` -- no custom DNS implementation
- [ ] DNS introducer hostnames are configurable via `GossipConfig`
- [ ] DNS timeout is configurable (default 30 seconds)
- [ ] DNS batch size is configurable (default 2)
- [ ] Resolved addresses are converted to `TimestampedPeerInfo` and added to address manager
- [ ] DNS lookup failures are logged but do not crash the discovery loop
- [ ] Empty DNS results are handled gracefully (proceed to introducer fallback)

## Implementation Notes

- Primary file: `src/discovery/node_discovery.rs` (DNS seeding is part of the discovery loop)
- Dependencies: `chia-sdk-client` (`Network`, `lookup_all()`), `chia-protocol` (`TimestampedPeerInfo`)
- Edge cases:
  - All DNS introducers may be unreachable: log warning, return empty list
  - DNS may return duplicate addresses: the address manager handles deduplication via bucket assignment
  - DNS may return IPv6 addresses: these should be accepted and handled by the address manager
  - Timeout must be per-batch, not per-individual-query, matching `lookup_all()` semantics
