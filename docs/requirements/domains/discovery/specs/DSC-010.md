# DSC-010: AS-Level Diversity

> **Authoritative requirement:** [NORMATIVE.md](../NORMATIVE.md)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **Spec reference:** [SPEC.md - Section 6.4](../../../resources/SPEC.md)

## Summary

AS-level diversity enforces that at most one outbound connection is maintained per Autonomous System (AS) number. This provides stronger eclipse attack resistance than Chia's /16 subnet grouping, because a single entity may control multiple /16 subnets within the same AS, but AS numbers represent distinct network operators. AS numbers are resolved via a cached BGP prefix table lookup.

## Specification

### AS Lookup

```rust
use std::net::IpAddr;

/// Autonomous System number.
pub type AsNumber = u32;

/// Cached BGP prefix table for AS number lookup.
/// Loaded from a BGP prefix dump and cached in memory.
pub struct AsLookupTable {
    /// Prefix-to-AS mapping, sorted for binary search.
    /// Each entry: (prefix_network, prefix_len, as_number)
    prefixes: Vec<(IpAddr, u8, AsNumber)>,
}

impl AsLookupTable {
    /// Load the BGP prefix table from a data file.
    pub fn load(path: &Path) -> Result<Self, GossipError>;

    /// Look up the AS number for a given IP address.
    /// Uses longest-prefix-match on the cached table.
    pub fn lookup(&self, ip: &IpAddr) -> Option<AsNumber>;
}
```

### Diversity Enforcement

```rust
/// Filter that enforces at most one outbound connection per AS number.
/// Used during peer selection in the discovery loop.
pub struct AsDiversityFilter {
    /// AS lookup table.
    as_table: AsLookupTable,
    /// AS numbers of current outbound connections.
    outbound_as_numbers: HashSet<AsNumber>,
}

impl AsDiversityFilter {
    /// Check whether a candidate peer is allowed given current AS diversity.
    /// Returns true if the candidate's AS number is not already represented.
    pub fn is_allowed(&self, candidate_ip: &IpAddr) -> bool {
        match self.as_table.lookup(candidate_ip) {
            Some(asn) => !self.outbound_as_numbers.contains(&asn),
            None => true, // allow if AS unknown (fail open)
        }
    }

    /// Record that a new outbound connection has been established.
    pub fn add_outbound(&mut self, ip: &IpAddr) {
        if let Some(asn) = self.as_table.lookup(ip) {
            self.outbound_as_numbers.insert(asn);
        }
    }

    /// Record that an outbound connection has been closed.
    pub fn remove_outbound(&mut self, ip: &IpAddr) {
        if let Some(asn) = self.as_table.lookup(ip) {
            self.outbound_as_numbers.remove(&asn);
        }
    }
}
```

### Integration with Connection Establishment

AS-level diversity check is the second pass after the /16 group filter (DSC-011):

```rust
// In connect_batch (DSC-009):
// 1. Fast first pass: /16 group filter (DSC-011)
if !group_filter.is_allowed_by_subnet(&candidate.peer_info.ip()) {
    continue;
}
// 2. Stronger second pass: AS-level diversity
if !as_filter.is_allowed(&candidate.peer_info.ip()) {
    continue;
}
```

## Acceptance Criteria

- [ ] At most one outbound connection per AS number is maintained
- [ ] AS numbers are resolved via cached BGP prefix table lookup
- [ ] BGP prefix table is loaded from a data file at startup
- [ ] Lookup uses longest-prefix-match for accurate AS assignment
- [ ] Unknown AS (IP not in table) fails open (connection allowed)
- [ ] AS filter is checked after /16 group filter (two-pass)
- [ ] AS set is updated when outbound connections are established or closed
- [ ] BGP table is cached in memory (not queried per-lookup from disk or network)

## Implementation Notes

- Primary files: `src/util/as_lookup.rs`, `src/discovery/node_discovery.rs`
- Dependencies: BGP prefix data file (bundled or downloaded at first run)
- Edge cases:
  - BGP table may not be available: fall back to /16-only filtering (DSC-011)
  - IPv6 addresses: AS lookup applies equally to IPv6 prefixes
  - Multiple IPs in the same AS may appear in the new table: only the first one to connect is accepted
  - AS table staleness: BGP tables change over time. The table should be periodically refreshed (out of scope for this requirement)
  - Private/local IPs (127.0.0.0/8, 10.0.0.0/8, etc.): not in BGP tables, treated as unknown AS (allowed)
  - Large BGP tables (~900K prefixes): optimize for memory and lookup speed (sorted vec + binary search, or prefix trie)
