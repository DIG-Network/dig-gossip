# DSC-001: AddressManager with Tried/New Tables

> **Authoritative requirement:** [NORMATIVE.md](../NORMATIVE.md)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **Spec reference:** [SPEC.md - Section 6.3](../../../resources/SPEC.md)

## Summary

The AddressManager is a Rust port of Chia's `address_manager.py` (itself a port of Bitcoin's `CAddrMan`). It maintains two tables of peer addresses -- "tried" (peers we have successfully connected to) and "new" (peers we have heard about but not yet verified) -- organized into deterministic buckets for eclipse attack resistance. This is the single largest piece of new code in the discovery module, as no Chia Rust crate provides this functionality.

## Specification

### Constants

```rust
/// Number of buckets in the tried table.
pub const TRIED_BUCKET_COUNT: usize = 256;

/// Number of buckets in the new table.
pub const NEW_BUCKET_COUNT: usize = 1024;

/// Maximum number of entries per bucket.
pub const BUCKET_SIZE: usize = 64;

/// Maximum number of entries in the tried table.
pub const TRIED_TABLE_SIZE: usize = TRIED_BUCKET_COUNT * BUCKET_SIZE; // 16,384

/// Maximum number of entries in the new table.
pub const NEW_TABLE_SIZE: usize = NEW_BUCKET_COUNT * BUCKET_SIZE; // 65,536
```

### AddressManager Public API

```rust
use std::path::Path;

/// Manages peer addresses using tried/new bucket tables.
/// Ported from Chia's address_manager.py (Bitcoin CAddrMan).
/// Interior mutability via async-aware locking (e.g., RwLock).
pub struct AddressManager {
    // Internal fields: tried table, new table, key, metadata, peers_file_path
}

impl AddressManager {
    /// Create a new AddressManager, loading persisted state from peers_file_path if it exists.
    pub async fn create(peers_file_path: &Path) -> Result<Self, GossipError>;

    /// Add addresses to the new table. `source` is the peer that told us about them.
    /// `penalty` is added to the timestamp to delay re-attempts on penalized addresses.
    pub async fn add_to_new_table(
        &self,
        addrs: &[TimestampedPeerInfo],
        source: &PeerInfo,
        penalty: u64,
    );

    /// Mark an address as good (successfully connected). Moves from new to tried table.
    pub async fn mark_good(&self, addr: &PeerInfo);

    /// Record a connection attempt. If `count_failure` is true, increment the failure counter.
    pub async fn attempt(&self, addr: &PeerInfo, count_failure: bool);

    /// Record that we have connected to this address (updates last-connected timestamp).
    pub async fn connect(&self, addr: &PeerInfo);

    /// Select a random peer from the address manager.
    /// If `new_only` is true, only select from the new table.
    /// Returns None if no suitable peer is found.
    pub async fn select_peer(&self, new_only: bool) -> Option<ExtendedPeerInfo>;

    /// Select a peer from the tried table that collided during a mark_good operation.
    pub async fn select_tried_collision(&self) -> Option<ExtendedPeerInfo>;

    /// Resolve any pending tried table collisions by evicting stale entries.
    pub async fn resolve_tried_collisions(&self);

    /// Return the total number of addresses tracked (tried + new).
    pub async fn size(&self) -> usize;
}
```

### Bucket Assignment

Bucket assignment MUST be deterministic based on a secret key, the peer's IP group, and the source address group. This prevents an attacker from predicting or controlling which bucket a peer lands in.

- **Tried bucket:** `hash(key, IP, /16 group) % TRIED_BUCKET_COUNT`
- **New bucket:** `hash(key, source_group, IP group) % NEW_BUCKET_COUNT`
- **Position within bucket:** `hash(key, IP, bucket) % BUCKET_SIZE`

### Eviction on Collision

When adding to a full bucket, the existing entry at the computed position is evicted if the new entry is "better" (more recent timestamp, fewer failures). For tried table collisions during `mark_good()`, the collision is queued for resolution via `select_tried_collision()` and `resolve_tried_collisions()`.

### ExtendedPeerInfo

```rust
/// Extended peer info tracked by the address manager.
pub struct ExtendedPeerInfo {
    pub peer_info: PeerInfo,
    /// Last time we successfully connected.
    pub last_success: u64,
    /// Last time we attempted a connection.
    pub last_try: u64,
    /// Number of failed connection attempts since last success.
    pub num_attempts: u32,
    /// Reference count in the new table (how many buckets reference this entry).
    pub ref_count: u32,
    /// Whether this address is in the tried table.
    pub in_tried: bool,
}
```

## Acceptance Criteria

- [ ] `AddressManager::create()` returns a valid instance with empty tables when no peers file exists
- [ ] `AddressManager::create()` loads persisted state when peers file exists
- [ ] `add_to_new_table()` places addresses into deterministic new-table buckets
- [ ] `add_to_new_table()` applies penalty to timestamp
- [ ] `add_to_new_table()` evicts on bucket collision using quality comparison
- [ ] `mark_good()` moves an address from new table to tried table
- [ ] `mark_good()` queues collision when tried bucket is full
- [ ] `attempt()` increments failure counter when `count_failure` is true
- [ ] `attempt()` updates last-try timestamp without incrementing failures when `count_failure` is false
- [ ] `connect()` updates last-connected timestamp
- [ ] `select_peer(false)` can return addresses from either tried or new table
- [ ] `select_peer(true)` only returns addresses from the new table
- [ ] `select_peer()` returns None when tables are empty
- [ ] `select_tried_collision()` returns collided entries from mark_good operations
- [ ] `resolve_tried_collisions()` evicts stale tried entries to make room for collisions
- [ ] `size()` returns total count of tracked addresses
- [ ] TRIED_BUCKET_COUNT is 256
- [ ] NEW_BUCKET_COUNT is 1024
- [ ] BUCKET_SIZE is 64

## Implementation Notes

- Primary file: `src/discovery/address_manager.rs`
- Dependencies: `rand` (for secret key generation), `siphasher` or `std` hashing (for bucket computation), `serde` (for persistence via DSC-002)
- Edge cases:
  - Secret key must be generated once and persisted with the address manager state
  - Bucket assignment must match Chia's `address_manager.py` algorithm for interoperability
  - `select_peer()` uses weighted random selection: tried table is preferred with probability proportional to its relative size, but never exclusively
  - `mark_good()` collision resolution is deferred -- collisions are queued, not resolved inline
  - Thread safety: all methods take `&self` and use interior mutability (async RwLock or Mutex)
