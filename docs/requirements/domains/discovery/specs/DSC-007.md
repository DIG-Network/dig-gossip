# DSC-007: Peer Exchange

> **Authoritative requirement:** [NORMATIVE.md](../NORMATIVE.md)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **Spec reference:** [SPEC.md - Section 6.6](../../../resources/SPEC.md)

## Summary

Peer exchange enables connected nodes to share known peers with each other via `chia-protocol`'s `RequestPeers`/`RespondPeers` messages. On every outbound connection, the node sends a `RequestPeers` to learn about additional peers. When a `RespondPeers` is received, the contained peer list is added to the address manager. This is ported from `node_discovery.py:135-136`.

## Specification

### Outbound Peer Exchange

```rust
use chia_protocol::{RequestPeers, RespondPeers, TimestampedPeerInfo};
use chia_sdk_client::Peer;

/// Request peers from a newly connected outbound peer.
/// Called immediately after a successful outbound connection.
/// Ported from node_discovery.py:135-136.
pub async fn request_peers_from(
    peer: &Peer,
    peer_info: &PeerInfo,
    address_manager: &AddressManager,
) -> Result<(), GossipError> {
    // Send RequestPeers via Peer::request_infallible
    let response: RespondPeers = peer
        .request_infallible(RequestPeers::new())
        .await
        .map_err(GossipError::from)?;

    // Add received peers to address manager's new table
    // Source is the peer that told us about them, penalty is 0
    address_manager
        .add_to_new_table(&response.peer_list, peer_info, 0)
        .await;

    tracing::debug!(
        "Received {} peers from {}",
        response.peer_list.len(),
        peer_info
    );

    Ok(())
}
```

### Responding to Peer Requests

```rust
/// Handle an incoming RequestPeers from a connected peer.
/// Returns our known peers for the peer to add to their address manager.
pub async fn handle_request_peers(
    address_manager: &AddressManager,
) -> Result<RespondPeers, GossipError> {
    // Select a subset of known peers to share
    // Chia shares up to 2000 peers per response
    let peers = address_manager.get_peers_for_sharing().await;

    Ok(RespondPeers {
        peer_list: peers,
    })
}
```

### Trigger Conditions

- **Outbound connect:** `request_peers_from()` is called immediately after a successful outbound connection and handshake.
- **Inbound request:** `handle_request_peers()` is called when an inbound `RequestPeers` message is received from any connected peer.

## Acceptance Criteria

- [ ] `RequestPeers` is sent on every successful outbound connection
- [ ] `RespondPeers.peer_list` is added to address manager via `add_to_new_table()`
- [ ] Source parameter in `add_to_new_table()` is set to the responding peer's info
- [ ] Penalty parameter is 0 for peer exchange
- [ ] Uses `chia-protocol::RequestPeers` directly (not a custom type)
- [ ] Uses `chia-protocol::RespondPeers` directly (not a custom type)
- [ ] Uses `Peer::request_infallible()` for the request
- [ ] Empty peer lists are handled gracefully (no error)
- [ ] Incoming `RequestPeers` are responded to with known peers
- [ ] `RespondPeers` MUST cap accepted peers at `MAX_PEERS_RECEIVED_PER_REQUEST` (1000); peers beyond the cap MUST be silently discarded
- [ ] Total peers received from all peers across all requests MUST be capped at `MAX_TOTAL_PEERS_RECEIVED` (3000); excess peers MUST be silently discarded

## Implementation Notes

- Primary file: `src/discovery/node_discovery.rs`
- Dependencies: `chia-protocol` (`RequestPeers`, `RespondPeers`, `TimestampedPeerInfo`), `chia-sdk-client` (`Peer`), AddressManager (DSC-001)
- Edge cases:
  - Peer may respond with an empty peer list: handle gracefully, do not treat as error
  - Peer may disconnect before responding: `request_infallible` returns error, logged and ignored
  - Peer list may contain duplicates or already-known peers: address manager handles dedup
  - Should not request peers from inbound connections (only outbound) to prevent information leakage about our address manager state
  - Rate limiting applies: `Peer` already enforces `V2_RATE_LIMITS` on RequestPeers/RespondPeers
