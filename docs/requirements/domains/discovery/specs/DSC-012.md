# DSC-012: IntroducerPeers/VettedPeer Tracking

> **Authoritative requirement:** [NORMATIVE.md](../NORMATIVE.md)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **Spec reference:** [SPEC.md - Section 10.1](../../../resources/SPEC.md)

## Summary

IntroducerPeers tracks the vetting state of peers discovered through the introducer. Each peer has a vetting score: 0 means unvetted (never attempted), negative means failed (connection attempts that failed), and positive means successfully vetted (connection succeeded). This is a Rust port of Chia's `introducer_peers.py:12-28` and is used by the introducer server to only share peers that have been successfully vetted.

## Specification

### VettedPeer

```rust
use std::net::SocketAddr;

/// A peer with vetting state, tracked by the introducer.
/// Ported from introducer_peers.py:12-28.
#[derive(Debug, Clone, Serialize, Deserialize)]
pub struct VettedPeer {
    /// The peer's address.
    pub addr: SocketAddr,
    /// Vetting state:
    ///   0     = unvetted (never attempted)
    ///   < 0   = failed (absolute value = number of consecutive failures)
    ///   > 0   = successfully vetted (value = number of consecutive successes)
    pub vetted: i32,
    /// Timestamp of last vetting attempt.
    pub last_attempt: u64,
    /// Timestamp when this peer was first added.
    pub added_at: u64,
}

impl VettedPeer {
    /// Create a new unvetted peer.
    pub fn new(addr: SocketAddr) -> Self {
        Self {
            addr,
            vetted: 0,
            last_attempt: 0,
            added_at: current_timestamp(),
        }
    }

    /// Whether this peer has been successfully vetted at least once.
    pub fn is_vetted(&self) -> bool {
        self.vetted > 0
    }

    /// Whether this peer has never been attempted.
    pub fn is_unvetted(&self) -> bool {
        self.vetted == 0
    }

    /// Whether this peer has failed vetting.
    pub fn is_failed(&self) -> bool {
        self.vetted < 0
    }
}
```

### IntroducerPeers

```rust
use std::collections::HashMap;

/// Tracks vetting state for peers discovered through the introducer.
/// Ported from introducer_peers.py.
pub struct IntroducerPeers {
    /// Map from socket address to vetted peer state.
    peers: HashMap<SocketAddr, VettedPeer>,
}

impl IntroducerPeers {
    /// Create a new empty IntroducerPeers tracker.
    pub fn new() -> Self {
        Self {
            peers: HashMap::new(),
        }
    }

    /// Add a peer to track. Initial state is unvetted (0).
    pub fn add(&mut self, addr: SocketAddr) {
        self.peers.entry(addr).or_insert_with(|| VettedPeer::new(addr));
    }

    /// Record a successful vetting (connection succeeded).
    /// Increments the vetted counter (or resets to 1 if was negative).
    pub fn mark_vetted(&mut self, addr: &SocketAddr) {
        if let Some(peer) = self.peers.get_mut(addr) {
            if peer.vetted < 0 {
                peer.vetted = 1; // reset from failed
            } else {
                peer.vetted += 1;
            }
            peer.last_attempt = current_timestamp();
        }
    }

    /// Record a failed vetting (connection failed).
    /// Decrements the vetted counter (or sets to -1 if was non-negative).
    pub fn mark_failed(&mut self, addr: &SocketAddr) {
        if let Some(peer) = self.peers.get_mut(addr) {
            if peer.vetted > 0 {
                peer.vetted = -1; // reset from success
            } else {
                peer.vetted -= 1;
            }
            peer.last_attempt = current_timestamp();
        }
    }

    /// Remove a peer from tracking.
    pub fn remove(&mut self, addr: &SocketAddr) {
        self.peers.remove(addr);
    }

    /// Get all successfully vetted peers.
    pub fn get_vetted(&self) -> Vec<&VettedPeer> {
        self.peers.values().filter(|p| p.is_vetted()).collect()
    }

    /// Get all unvetted peers (never attempted).
    pub fn get_unvetted(&self) -> Vec<&VettedPeer> {
        self.peers.values().filter(|p| p.is_unvetted()).collect()
    }

    /// Get all failed peers.
    pub fn get_failed(&self) -> Vec<&VettedPeer> {
        self.peers.values().filter(|p| p.is_failed()).collect()
    }

    /// Total number of tracked peers.
    pub fn len(&self) -> usize {
        self.peers.len()
    }

    /// Whether no peers are tracked.
    pub fn is_empty(&self) -> bool {
        self.peers.is_empty()
    }
}
```

### Vetting State Machine

```
                  add()
                    |
                    v
             +------------+
             | unvetted=0 |
             +------+-----+
                    |
          +---------+---------+
          |                   |
    mark_vetted()       mark_failed()
          |                   |
          v                   v
    +----------+        +-----------+
    | vetted>0 |        | failed<0  |
    +----+-----+        +-----+-----+
         |                    |
    mark_failed()       mark_vetted()
         |                    |
         v                    v
    +-----------+       +----------+
    | failed=-1 |       | vetted=1 |
    +-----------+       +----------+
```

## Acceptance Criteria

- [ ] `VettedPeer` has a `vetted` field: 0=unvetted, negative=failed, positive=success
- [ ] `VettedPeer::new()` creates with `vetted=0` (unvetted)
- [ ] `is_vetted()` returns true when `vetted > 0`
- [ ] `is_unvetted()` returns true when `vetted == 0`
- [ ] `is_failed()` returns true when `vetted < 0`
- [ ] `mark_vetted()` increments counter (or resets to 1 from failed)
- [ ] `mark_failed()` decrements counter (or resets to -1 from success)
- [ ] `add()` does not overwrite existing peer state (uses `entry().or_insert()`)
- [ ] `get_vetted()` returns only peers with `vetted > 0`
- [ ] `get_unvetted()` returns only peers with `vetted == 0`
- [ ] `get_failed()` returns only peers with `vetted < 0`
- [ ] `remove()` removes a peer from tracking
- [ ] `last_attempt` is updated on each `mark_vetted()` and `mark_failed()` call

## Implementation Notes

- Primary file: `src/discovery/introducer_peers.rs`
- Dependencies: `serde` (Serialize/Deserialize), `std::collections::HashMap`
- Edge cases:
  - `mark_vetted()` on unknown address: no-op (peer not in map)
  - `mark_failed()` on unknown address: no-op (peer not in map)
  - `add()` called multiple times for same address: state preserved, not reset
  - Integer overflow on `vetted`: practically impossible (would need billions of consecutive successes), but could be capped at i32::MAX/MIN
  - Thread safety: `IntroducerPeers` methods take `&mut self`; caller is responsible for synchronization (wrap in Mutex or RwLock if shared)
  - This is used by the introducer server side; the client side (DSC-004, DSC-005) does not use this directly
