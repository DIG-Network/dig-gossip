# DSC-002: Address Manager Persistence

> **Authoritative requirement:** [NORMATIVE.md](../NORMATIVE.md)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **Spec reference:** [SPEC.md - Section 6.3](../../../resources/SPEC.md)

## Summary

The AddressManager must persist its state (tried/new tables, secret key, metadata) to disk so that peer knowledge survives process restarts. Serialization uses bincode for compact binary encoding and fast serialization/deserialization. The persistence layer is implemented in a separate `address_manager_store.rs` module to keep storage concerns decoupled from the address manager logic.

## Specification

### Persistence API

```rust
use std::path::Path;

/// Persistent storage for AddressManager state.
/// Uses bincode for binary serialization.
pub struct AddressManagerStore;

impl AddressManagerStore {
    /// Save the address manager state to the specified file path.
    /// Writes atomically: serialize to temp file, then rename.
    pub async fn save(state: &AddressManagerState, path: &Path) -> Result<(), GossipError>;

    /// Load address manager state from the specified file path.
    /// Returns None if the file does not exist.
    /// Returns Err if the file exists but is corrupted or incompatible.
    pub async fn load(path: &Path) -> Result<Option<AddressManagerState>, GossipError>;
}

/// Serializable snapshot of the address manager's internal state.
#[derive(Serialize, Deserialize)]
pub struct AddressManagerState {
    /// Format version for forward compatibility.
    pub version: u32,
    /// Secret key used for deterministic bucket assignment.
    pub key: [u8; 32],
    /// All tracked addresses with their metadata.
    pub entries: Vec<ExtendedPeerInfo>,
    /// Tried table bucket assignments: bucket_index -> [entry_index; BUCKET_SIZE].
    pub tried_table: Vec<Vec<Option<usize>>>,
    /// New table bucket assignments: bucket_index -> [entry_index; BUCKET_SIZE].
    pub new_table: Vec<Vec<Option<usize>>>,
}
```

### AddressManager Integration

```rust
impl AddressManager {
    /// Save the current address manager state to the configured peers file path.
    /// Called periodically and on shutdown.
    pub async fn save(&self) {
        let state = self.snapshot().await;
        if let Err(e) = AddressManagerStore::save(&state, &self.peers_file_path).await {
            tracing::error!("Failed to save address manager: {}", e);
        }
    }

    /// Load address manager state during create().
    /// Part of AddressManager::create() initialization.
    async fn load(peers_file_path: &Path) -> Result<Option<AddressManagerState>, GossipError> {
        AddressManagerStore::load(peers_file_path).await
    }
}
```

### Serialization Format

- **Format:** bincode (binary, compact, fast)
- **Version field:** `AddressManagerState.version` for forward compatibility. Current version: 1.
- **Atomic writes:** Save writes to a temporary file (same directory, `.tmp` suffix), then renames to the target path. This prevents corruption from interrupted writes.

## Acceptance Criteria

- [ ] `save()` writes the address manager state to the configured file path
- [ ] `save()` uses atomic write (temp file + rename) to prevent corruption
- [ ] `load()` returns `Ok(None)` when the peers file does not exist
- [ ] `load()` returns `Ok(Some(state))` when the peers file exists and is valid
- [ ] `load()` returns `Err` when the peers file exists but is corrupted
- [ ] Round-trip: save then load produces identical state
- [ ] `AddressManagerState` includes a version field set to 1
- [ ] Serialization format is bincode
- [ ] `create()` calls `load()` and restores state if peers file exists
- [ ] `create()` generates new empty state if peers file does not exist

## Implementation Notes

- Primary file: `src/discovery/address_manager_store.rs`
- Dependencies: `bincode` (serialization), `serde` (Serialize/Deserialize derives), `tokio::fs` (async file I/O)
- Edge cases:
  - Corrupted files should be handled gracefully: log a warning and start with empty state rather than crashing
  - Version mismatch: if the loaded version is higher than supported, treat as incompatible and start fresh
  - File permissions: the peers file directory must be writable; propagate IO errors as GossipError
  - Concurrent access: only one AddressManager instance should access a given peers file at a time
  - Atomic rename may not be atomic across filesystems on all platforms; the temp file should be in the same directory as the target
