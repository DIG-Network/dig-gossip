# DSC-004: Introducer Query

> **Authoritative requirement:** [NORMATIVE.md](../NORMATIVE.md)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **Spec reference:** [SPEC.md - Section 6.5](../../../resources/SPEC.md)

## Summary

The introducer query allows a node to obtain initial peers from a known introducer server. The flow is: connect via WebSocket, perform Chia protocol handshake, send `RequestPeersIntroducer`, receive `RespondPeersIntroducer` containing a list of peers, and close the connection. This uses `chia-protocol` message types directly.

## Specification

### Query Flow

```rust
use chia_protocol::{RequestPeersIntroducer, RespondPeersIntroducer, TimestampedPeerInfo};
use chia_sdk_client::Peer;

/// Query an introducer for peers.
///
/// Flow:
///   1. Connect to introducer via WebSocket (TLS)
///   2. Perform Chia protocol handshake
///   3. Send RequestPeersIntroducer
///   4. Receive RespondPeersIntroducer
///   5. Close connection
///
/// Returns the list of peers from the introducer.
pub async fn query_introducer(
    introducer_addr: &str,
    handshake_config: &HandshakeConfig,
    timeout: Duration,
) -> Result<Vec<TimestampedPeerInfo>, GossipError> {
    // 1. Connect via WebSocket with TLS
    let peer = connect_to_introducer(introducer_addr, handshake_config, timeout).await?;

    // 2. Send RequestPeersIntroducer (handshake completed during connect)
    let response: RespondPeersIntroducer = peer
        .request_infallible(RequestPeersIntroducer::new())
        .await
        .map_err(GossipError::from)?;

    // 3. Extract peer list
    let peers = response.peer_list;

    // 4. Close connection
    drop(peer);

    Ok(peers)
}
```

### Protocol Messages

Uses `chia-protocol` types directly:
- **Request:** `RequestPeersIntroducer` (empty message, just signals "give me peers")
- **Response:** `RespondPeersIntroducer { peer_list: Vec<TimestampedPeerInfo> }`

## Acceptance Criteria

- [ ] Query connects to introducer via WebSocket with TLS
- [ ] Chia protocol handshake is performed before sending request
- [ ] `RequestPeersIntroducer` is sent using `Peer::request_infallible()`
- [ ] `RespondPeersIntroducer` is received and peer_list extracted
- [ ] Connection is closed after receiving response
- [ ] Timeout is enforced on the entire query operation
- [ ] Connection failures return `GossipError` (do not panic)
- [ ] Uses `chia-protocol::RequestPeersIntroducer` and `RespondPeersIntroducer` directly

## Implementation Notes

- Primary file: `src/discovery/introducer_client.rs`
- Dependencies: `chia-protocol` (`RequestPeersIntroducer`, `RespondPeersIntroducer`, `TimestampedPeerInfo`, `NodeType`, `Handshake`), `chia-sdk-client` (`Peer`, `connect_peer`, TLS utilities)
- Edge cases:
  - Introducer may be unreachable: return error, discovery loop handles retry with backoff
  - Introducer may return empty peer list: return Ok(empty vec), not an error
  - Handshake may fail (version mismatch, network ID mismatch): return GossipError
  - TLS certificate verification: uses Chia CA certificate from `chia-ssl`
  - Timeout applies to the entire operation (connect + handshake + request + response)
