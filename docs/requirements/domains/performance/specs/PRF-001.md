# PRF-001: Latency-Aware Scoring

> **Authoritative requirement:** [NORMATIVE.md](../NORMATIVE.md)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **Spec reference:** [SPEC.md - Sections 1.8, 2.5, 4.4](../../../resources/SPEC.md)

## Summary

Peers must be scored based on round-trip time (RTT) measured from Ping/Pong exchanges. A rolling average over the last `RTT_WINDOW_SIZE` (10) measurements is maintained in each peer's reputation record. The composite score combines trust with latency: `score = trust_score * (1 / avg_rtt_ms)`. This score drives outbound peer selection (PRF-002) and Plumtree tree optimization (PRF-003). All RTT state is stored in the `PeerReputation` struct fields: `avg_rtt_ms`, `rtt_history`, and `score`.

## Specification

### RTT Measurement

RTT is measured from the Ping/Pong protocol already present in the Chia wire protocol. When a `Ping` is sent, the local timestamp is recorded. When the corresponding `Pong` is received, the RTT is computed as `now - ping_sent_time` in milliseconds.

### Rolling Average

The last `RTT_WINDOW_SIZE` measurements are stored in a circular buffer (`VecDeque<u64>`). The rolling average is recomputed on each new sample.

### Composite Score

The composite score is recomputed whenever `avg_rtt_ms` or `trust_score` changes. A peer with no RTT data receives a default score of 0.0 (no preference until measured).

### Data Model

```rust
use std::collections::VecDeque;

/// RTT measurement window (number of recent pings to average).
/// SPEC Section 4.4.
pub const RTT_WINDOW_SIZE: usize = 10;

/// Maximum acceptable RTT before peer score is penalized (ms).
/// SPEC Section 4.4.
pub const RTT_PENALTY_THRESHOLD_MS: u64 = 5000;

/// Reputation tracking for a peer.
/// SPEC Section 2.5.
#[derive(Debug, Clone, Default)]
pub struct PeerReputation {
    /// Cumulative penalty points (higher = worse).
    pub penalty_points: u32,
    /// Whether this peer is temporarily banned.
    pub is_banned: bool,
    /// Ban expiry timestamp (Unix seconds).
    pub ban_until: Option<u64>,
    /// Last penalty reason.
    pub last_penalty_reason: Option<PenaltyReason>,
    /// Rolling average RTT in milliseconds (from Ping/Pong).
    /// Used for latency-aware peer selection and Plumtree tree optimization.
    pub avg_rtt_ms: Option<u64>,
    /// Recent RTT measurements (circular buffer, last RTT_WINDOW_SIZE pings).
    pub rtt_history: VecDeque<u64>,
    /// Composite peer score: trust_score * (1 / avg_rtt_ms).
    /// Higher = better. Used for outbound peer selection preference.
    pub score: f64,
    /// AS number for this peer's IP (cached from BGP lookup).
    pub as_number: Option<u32>,
}

impl PeerReputation {
    /// Record a new RTT measurement and recompute the rolling average and score.
    pub fn record_rtt(&mut self, rtt_ms: u64, trust_score: f64) {
        // Maintain circular buffer of RTT_WINDOW_SIZE entries
        if self.rtt_history.len() >= RTT_WINDOW_SIZE {
            self.rtt_history.pop_front();
        }
        self.rtt_history.push_back(rtt_ms);

        // Compute rolling average
        let sum: u64 = self.rtt_history.iter().sum();
        let avg = sum / self.rtt_history.len() as u64;
        self.avg_rtt_ms = Some(avg);

        // Recompute composite score
        self.recompute_score(trust_score);
    }

    /// Recompute composite score from current trust_score and avg_rtt_ms.
    pub fn recompute_score(&mut self, trust_score: f64) {
        self.score = match self.avg_rtt_ms {
            Some(avg) if avg > 0 => trust_score * (1.0 / avg as f64),
            _ => 0.0, // No RTT data yet; no preference
        };
    }
}
```

### RTT Tracker (util/latency.rs)

```rust
use std::collections::HashMap;
use std::time::Instant;

/// Tracks outstanding Ping timestamps for RTT computation.
pub struct RttTracker {
    /// Map of peer_id -> timestamp when Ping was sent.
    pending_pings: HashMap<PeerId, Instant>,
}

impl RttTracker {
    pub fn new() -> Self {
        Self {
            pending_pings: HashMap::new(),
        }
    }

    /// Record that a Ping was sent to peer_id at the current time.
    pub fn ping_sent(&mut self, peer_id: PeerId) {
        self.pending_pings.insert(peer_id, Instant::now());
    }

    /// Process a Pong from peer_id. Returns the RTT in milliseconds if
    /// a matching Ping was recorded.
    pub fn pong_received(&mut self, peer_id: &PeerId) -> Option<u64> {
        self.pending_pings.remove(peer_id).map(|sent| {
            sent.elapsed().as_millis() as u64
        })
    }
}
```

## Acceptance Criteria

- [ ] `PeerReputation` struct contains `avg_rtt_ms: Option<u64>` field
- [ ] `PeerReputation` struct contains `rtt_history: VecDeque<u64>` field
- [ ] `PeerReputation` struct contains `score: f64` field
- [ ] `record_rtt()` appends to `rtt_history` and evicts oldest when at capacity `RTT_WINDOW_SIZE`
- [ ] `record_rtt()` computes rolling average as `sum(rtt_history) / len(rtt_history)`
- [ ] `record_rtt()` stores rolling average in `avg_rtt_ms`
- [ ] `record_rtt()` computes `score = trust_score * (1.0 / avg_rtt_ms)`
- [ ] `RTT_WINDOW_SIZE` constant equals 10
- [ ] Peer with no RTT data has `score == 0.0`
- [ ] Peer with `avg_rtt_ms == 0` has `score == 0.0` (no division by zero)
- [ ] `RttTracker` records Ping send timestamps per peer
- [ ] `RttTracker.pong_received()` returns RTT in milliseconds for matching Ping
- [ ] `RttTracker.pong_received()` returns `None` if no matching Ping was sent

## Implementation Notes

- Primary files: `src/types/reputation.rs`, `src/util/latency.rs`
- Dependencies: `PeerId` from `types::peer`, `std::collections::VecDeque`, `std::time::Instant`
- Edge cases:
  - First RTT measurement: `rtt_history` has 1 element, average equals that single value
  - RTT of 0ms: score formula yields infinity; guard with `avg > 0` check, default to 0.0
  - Pong received without matching Ping (stale or unsolicited): `pong_received()` returns `None`, no score update
  - Trust score of 0.0: score is 0.0 regardless of RTT, correctly deprioritizing untrusted peers
  - RTT exceeding `RTT_PENALTY_THRESHOLD_MS` (5000ms): still recorded normally; the high RTT naturally produces a low score
