# CON-003: Handshake Validation

> **Authoritative requirement:** [NORMATIVE.md](../NORMATIVE.md)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **Spec reference:** [SPEC.md - Sections 5.1, 5.2](../../../resources/SPEC.md)

## Summary

All inbound and outbound handshakes must be validated before the connection is accepted. The `network_id` must match the local DIG network. The `protocol_version` must be compatible. The `software_version` string must be sanitized by stripping Unicode Cc (control) and Cf (format) characters, and must not exceed 128 bytes. Invalid handshakes result in immediate disconnection.

## Specification

### Network ID Validation

The `network_id` from the remote `Handshake` MUST match the local `config.network_id`. Mismatches MUST cause immediate disconnection:

```rust
fn validate_handshake(
    their_handshake: &Handshake,
    our_network_id: &str,
) -> Result<(), GossipError> {
    // Reject network_id mismatch
    if their_handshake.network_id != our_network_id {
        return Err(GossipError::ClientError(
            ClientError::NetworkIdMismatch {
                expected: our_network_id.to_string(),
                actual: their_handshake.network_id.clone(),
            }
        ));
    }

    // Validate protocol_version compatibility
    if !is_compatible_protocol_version(&their_handshake.protocol_version) {
        return Err(GossipError::ClientError(
            ClientError::IncompatibleProtocolVersion {
                version: their_handshake.protocol_version.clone(),
            }
        ));
    }

    // Validate software_version length after sanitization
    let sanitized = sanitize_version(&their_handshake.software_version);
    if sanitized.len() > MAX_SOFTWARE_VERSION_LEN {
        return Err(GossipError::ClientError(
            ClientError::InvalidHandshake("software_version too long".into())
        ));
    }

    Ok(())
}
```

### Software Version Sanitization

The `software_version` MUST be sanitized by stripping Unicode Cc and Cf characters (matching Chia `ws_connection.py:61-63`):

```rust
/// Strip Unicode Cc (control) and Cf (format) characters from a version string.
/// Reference: Chia ws_connection.py:61-63
fn sanitize_version(version: &str) -> String {
    version
        .chars()
        .filter(|c| {
            use unicode_general_category::{get_general_category, GeneralCategory};
            let cat = get_general_category(*c);
            cat != GeneralCategory::Control       // Cc
                && cat != GeneralCategory::Format  // Cf
        })
        .collect()
}
```

### Length Enforcement

After sanitization, the `software_version` MUST be no longer than 128 bytes:

```rust
let sanitized = sanitize_version(&handshake.software_version);

```

## Acceptance Criteria

- [ ] Handshake with mismatched `network_id` is rejected
- [ ] Handshake with incompatible `protocol_version` is rejected
- [ ] Unicode Cc characters (e.g., `\x00`, `\x1F`, `\x7F`) are stripped from `software_version`
- [ ] Unicode Cf characters (e.g., `\u{200B}` zero-width space, `\u{FEFF}` BOM) are stripped from `software_version`
- [ ] `software_version` exceeding 128 bytes after sanitization is rejected
- [ ] Valid handshakes with correct `network_id` and compatible `protocol_version` are accepted
- [ ] Sanitized `software_version` is stored in `PeerConnection`
- [ ] Validation is applied to both inbound and outbound handshakes

## Implementation Notes

- Primary files: `src/connection/mod.rs` (validation functions)
- Dependencies: `chia-protocol` (Handshake), `chia-sdk-client` (ClientError)
- Edge cases:
  - `software_version` consisting entirely of Cc/Cf characters results in an empty string (valid)
  - `software_version` at exactly 128 bytes is accepted; 129 bytes is rejected
  - Multi-byte UTF-8 characters: length check is on byte length, not character count
  - `connect_peer()` already validates `network_id` for outbound; inbound must validate independently
  - Empty `network_id` or `protocol_version` should be rejected
