# CON-002: Inbound Connection Listener

> **Authoritative requirement:** [NORMATIVE.md](../NORMATIVE.md)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **Spec reference:** [SPEC.md - Section 5.2](../../../resources/SPEC.md)

## Summary

`chia-sdk-client`'s `Peer` only supports outbound connections. For inbound connections, the gossip service must accept TCP connections via `TcpListener`, perform TLS handshake, upgrade to WebSocket via `tokio_tungstenite::accept_async()`, and create a `Peer` via `Peer::from_websocket()`. The inbound handshake must be received, validated, and responded to. The new peer is added to the address manager "new" table and its info is relayed to other connected peers.

## Specification

### Listener Setup

The inbound listener MUST bind to the configured listen address:

```rust
use tokio::net::TcpListener;

let listener = TcpListener::bind(&config.listen_addr).await?;
```

### Accept Loop

Each incoming connection MUST go through TLS and WebSocket upgrade:

```rust
loop {
    let (tcp_stream, remote_addr) = listener.accept().await?;

    // TLS handshake using chia-ssl certificate
    let tls_stream = tls_acceptor.accept(tcp_stream).await?;

    // WebSocket upgrade
    let ws_stream = tokio_tungstenite::accept_async(tls_stream).await?;

    // Create Peer from WebSocket
    let (peer, receiver) = Peer::from_websocket(ws_stream, peer_options.clone());

    // Spawn per-connection handler
    tokio::spawn(handle_inbound_connection(peer, receiver, remote_addr));
}
```

### Handshake Exchange

The inbound handler MUST receive and validate the peer's `Handshake`, then send its own:

```rust
async fn handle_inbound_connection(
    peer: Peer,
    mut receiver: mpsc::Receiver<Message>,
    remote_addr: SocketAddr,
) -> Result<(), GossipError> {
    // Step 5: Receive Handshake from connecting peer
    let handshake_msg = receiver.recv().await
        .ok_or(GossipError::ChannelClosed)?;
    let their_handshake: Handshake = Streamable::from_bytes(&handshake_msg.data)?;

    // Validate network_id (see CON-003)
    if their_handshake.network_id != config.network_id {
        return Err(GossipError::ClientError(/* network_id mismatch */));
    }

    // Step 6: Send our Handshake response
    let our_handshake = Handshake {
        network_id: config.network_id.to_string(),
        protocol_version: PROTOCOL_VERSION.to_string(),
        software_version: SOFTWARE_VERSION.to_string(),
        server_port: config.listen_addr.port(),
        node_type: NodeType::FullNode,
        capabilities: our_capabilities(),
    };
    peer.send(our_handshake).await?;

    // Step 7: Wrap in PeerConnection
    let peer_connection = PeerConnection {
        peer,
        peer_id,
        address: remote_addr,
        is_outbound: false,
        node_type: their_handshake.node_type,
        protocol_version: their_handshake.protocol_version.clone(),
        software_version: sanitize_version(&their_handshake.software_version),
        peer_server_port: their_handshake.server_port,
        capabilities: their_handshake.capabilities.clone(),
        creation_time: current_unix_timestamp(),
        bytes_read: 0,
        bytes_written: 0,
        last_message_time: current_unix_timestamp(),
        reputation: PeerReputation::default(),
        inbound_rx: receiver,
    };

    // Step 8: Add to address manager "new" table (node_discovery.py:120-125)
    address_manager.add_to_new_table(
        &[TimestampedPeerInfo {
            host: remote_addr.ip().to_string(),
            port: their_handshake.server_port,
            timestamp: current_unix_timestamp(),
        }],
        &our_peer_info,
        0,
    ).await;

    // Step 9: Relay peer info to other connected peers (node_discovery.py:126-127)
    relay_peer_info(&peer_connection).await;

    Ok(())
}
```

## Acceptance Criteria

- [ ] `TcpListener` binds to `config.listen_addr`
- [ ] Incoming TCP connections go through TLS handshake
- [ ] WebSocket upgrade via `tokio_tungstenite::accept_async()`
- [ ] `Peer::from_websocket()` creates a Peer from the WebSocket stream
- [ ] Inbound `Handshake` is received and decoded via `Streamable`
- [ ] `network_id` is validated against the local config (rejects mismatch)
- [ ] A `Handshake` response is sent with local node info
- [ ] The connection is wrapped in `PeerConnection` with `is_outbound: false`
- [ ] `software_version` is sanitized (see CON-008)
- [ ] Peer is added to address manager "new" table
- [ ] Peer info is relayed to other connected peers
- [ ] Each inbound connection is handled in its own spawned task

## Implementation Notes

- Primary files: `src/connection/listener.rs`, `src/connection/mod.rs`
- Dependencies: `tokio` (TcpListener, spawn), `tokio-tungstenite` (accept_async), `chia-sdk-client` (Peer::from_websocket, PeerOptions), `chia-protocol` (Handshake, Streamable, TimestampedPeerInfo)
- Edge cases:
  - TLS handshake failure should log and skip (not crash the listener)
  - WebSocket upgrade failure should log and skip
  - Handshake timeout: if no Handshake received within a reasonable period (e.g., 30s), disconnect
  - Max connections: reject inbound connections when `max_connections` is reached
  - Self-connection detection: reject if peer_id matches our own
