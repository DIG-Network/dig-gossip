# CON-007: Peer Banning via Reputation

> **Authoritative requirement:** [NORMATIVE.md](../NORMATIVE.md)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **Spec reference:** [SPEC.md - Sections 2.5, 2.13](../../../resources/SPEC.md)

## Summary

Peer banning extends `chia-sdk-client::ClientState`'s binary ban/trust model with numeric penalty accumulation via `PeerReputation`. Penalties are assigned for various misbehaviors (invalid blocks, spam, protocol violations, etc.). When a peer's cumulative `penalty_points` reaches `PENALTY_BAN_THRESHOLD` (100 points), the peer is banned via `ClientState::ban()` and the `PeerReputation` is flagged. Bans automatically expire after `BAN_DURATION_SECS` (3600 seconds).

## Specification

### Constants

```rust
/// Penalty points threshold at which a peer is banned.
pub const PENALTY_BAN_THRESHOLD: u32 = 100;

/// Duration of a temporary ban in seconds.
pub const BAN_DURATION_SECS: u64 = 3600;
```

### PenaltyReason and Point Values

Each `PenaltyReason` maps to a penalty point value:

```rust
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum PenaltyReason {
    InvalidBlock,
    InvalidAttestation,
    MalformedMessage,
    Spam,
    ConnectionIssue,
    ProtocolViolation,
    RateLimitExceeded,
    ConsensusError,
}

impl PenaltyReason {
    /// Return the penalty points for this reason.
    pub fn points(&self) -> u32 {
        match self {
            PenaltyReason::InvalidBlock => 100,      // Immediate ban
            PenaltyReason::InvalidAttestation => 50,
            PenaltyReason::MalformedMessage => 20,
            PenaltyReason::Spam => 25,
            PenaltyReason::ConnectionIssue => 10,
            PenaltyReason::ProtocolViolation => 50,
            PenaltyReason::RateLimitExceeded => 15,
            PenaltyReason::ConsensusError => 100,    // Immediate ban
        }
    }
}
```

### Penalty Application

When a penalty is applied, the `PeerReputation` MUST be updated and, if the threshold is reached, the peer MUST be banned:

```rust
impl PeerReputation {
    /// Apply a penalty to this peer's reputation.
    /// Returns true if the peer should be banned.
    pub fn apply_penalty(&mut self, reason: PenaltyReason) -> bool {
        self.penalty_points += reason.points();
        self.last_penalty_reason = Some(reason);

        if self.penalty_points >= PENALTY_BAN_THRESHOLD && !self.is_banned {
            self.is_banned = true;
            self.ban_until = Some(current_unix_timestamp() + BAN_DURATION_SECS);
            return true;
        }
        false
    }
}
```

### Ban Execution

When the penalty threshold is reached, the ban MUST be executed via both `PeerReputation` and `ClientState::ban()`:

```rust
async fn ban_peer(
    client_state: &ClientState,
    peer_connection: &mut PeerConnection,
    reason: PenaltyReason,
) -> Result<(), GossipError> {
    let should_ban = peer_connection.reputation.apply_penalty(reason);

    if should_ban {
        // Delegate to chia-sdk-client's ClientState
        client_state.ban(peer_connection.address.ip()).await;

        tracing::warn!(
            peer_id = %peer_connection.peer_id,
            reason = ?reason,
            penalty_points = peer_connection.reputation.penalty_points,
            "Peer banned (threshold {} reached)",
            PENALTY_BAN_THRESHOLD,
        );

        // Disconnect the peer
        peer_connection.peer.close().await;
    }

    Ok(())
}
```

### Auto-Unban

Banned peers MUST be automatically unbanned when the ban duration expires:

```rust
impl PeerReputation {
    /// Check if the ban has expired and unban if so.
    /// Returns true if the peer was unbanned.
    pub fn check_unban(&mut self, current_time: u64) -> bool {
        if self.is_banned {
            if let Some(ban_until) = self.ban_until {
                if current_time >= ban_until {
                    self.is_banned = false;
                    self.ban_until = None;
                    self.penalty_points = 0; // Reset on unban
                    return true;
                }
            }
        }
        false
    }
}
```

The auto-unban check MUST be performed periodically (e.g., when considering reconnection) and MUST also call `ClientState::unban()`:

```rust
async fn check_unbans(
    client_state: &ClientState,
    banned_peers: &mut HashMap<PeerId, PeerReputation>,
) {
    let now = current_unix_timestamp();
    for (peer_id, reputation) in banned_peers.iter_mut() {
        if reputation.check_unban(now) {
            client_state.unban(peer_addr).await;
            tracing::info!(peer_id = %peer_id, "Peer auto-unbanned after {}s", BAN_DURATION_SECS);
        }
    }
}
```

## Acceptance Criteria

- [ ] `PenaltyReason` enum defines all 8 penalty reasons from SPEC Section 2.5
- [ ] Each `PenaltyReason` maps to a specific point value
- [ ] `penalty_points` accumulates across multiple penalty applications
- [ ] Peer is banned when `penalty_points >= PENALTY_BAN_THRESHOLD` (100)
- [ ] Ban sets `PeerReputation.is_banned = true`
- [ ] Ban sets `PeerReputation.ban_until` to current time + `BAN_DURATION_SECS` (3600)
- [ ] Ban calls `ClientState::ban()` for the peer's IP
- [ ] Banned peer is disconnected
- [ ] Auto-unban fires when `current_time >= ban_until`
- [ ] Auto-unban resets `is_banned`, `ban_until`, and `penalty_points`
- [ ] Auto-unban calls `ClientState::unban()` for the peer's IP
- [ ] `last_penalty_reason` is updated on every penalty application

## Implementation Notes

- Primary files: `src/types/reputation.rs` (PeerReputation, PenaltyReason), `src/connection/mod.rs` (ban execution)
- Dependencies: `chia-sdk-client` (ClientState::ban, ClientState::unban)
- Edge cases:
  - A single `InvalidBlock` or `ConsensusError` penalty (100 points) triggers an immediate ban
  - Penalty points do not decay over time (only reset on unban)
  - Multiple penalties in rapid succession should all accumulate before the ban check
  - A peer that is already banned should not be banned again (idempotent)
  - Auto-unban check timing: the periodic check interval determines the resolution of unban timing
  - `penalty_points` resets to 0 on unban to give the peer a fresh start
