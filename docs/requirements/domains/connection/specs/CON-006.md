# CON-006: Connection Metrics Tracking

> **Authoritative requirement:** [NORMATIVE.md](../NORMATIVE.md)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **Spec reference:** [SPEC.md - Section 2.4](../../../resources/SPEC.md)

## Summary

Each `PeerConnection` must track connection-level metrics: `bytes_read`, `bytes_written`, `messages_sent`, `messages_received`, `last_message_time`, and `creation_time`. These metrics are updated on every message send and receive, and are exposed via `GossipStats` for monitoring and diagnostics.

## Specification

### PeerConnection Metric Fields

The following metric fields MUST be present on `PeerConnection` (from SPEC Section 2.4):

```rust
pub struct PeerConnection {
    // ... other fields ...

    /// Timestamp when connection was established (Unix seconds).
    pub creation_time: u64,

    /// Total bytes read from this peer.
    pub bytes_read: u64,

    /// Total bytes written to this peer.
    pub bytes_written: u64,

    /// Total messages sent to this peer.
    pub messages_sent: u64,

    /// Total messages received from this peer.
    pub messages_received: u64,

    /// Timestamp of last message received (Unix seconds).
    pub last_message_time: u64,
}
```

### Initialization

When a `PeerConnection` is created (both inbound and outbound), metrics MUST be initialized:

```rust
let peer_connection = PeerConnection {
    // ... other fields ...
    creation_time: current_unix_timestamp(),
    bytes_read: 0,
    bytes_written: 0,
    messages_sent: 0,
    messages_received: 0,
    last_message_time: current_unix_timestamp(),
};
```

### Update on Message Send

When a message is sent to a peer, the following MUST be updated:

```rust
fn record_message_sent(conn: &mut PeerConnection, message_size: u64) {
    conn.bytes_written += message_size;
    conn.messages_sent += 1;
}
```

### Update on Message Receive

When a message is received from a peer, the following MUST be updated:

```rust
fn record_message_received(conn: &mut PeerConnection, message_size: u64) {
    conn.bytes_read += message_size;
    conn.messages_received += 1;
    conn.last_message_time = current_unix_timestamp();
}
```

### Aggregation into GossipStats

The per-connection metrics MUST be aggregatable into `GossipStats`:

```rust
pub struct GossipStats {
    pub total_connections: usize,
    pub connected_peers: usize,
    pub inbound_connections: usize,
    pub outbound_connections: usize,
    pub messages_sent: u64,       // Sum of all connections' messages_sent
    pub messages_received: u64,   // Sum of all connections' messages_received
    pub bytes_sent: u64,          // Sum of all connections' bytes_written
    pub bytes_received: u64,      // Sum of all connections' bytes_read
    pub known_addresses: usize,
    pub seen_messages: usize,
    pub relay_connected: bool,
    pub relay_peer_count: usize,
}
```

## Acceptance Criteria

- [ ] `PeerConnection` has `creation_time` field set to Unix timestamp at connection time
- [ ] `PeerConnection` has `bytes_read` field initialized to 0
- [ ] `PeerConnection` has `bytes_written` field initialized to 0
- [ ] `PeerConnection` has `messages_sent` field initialized to 0
- [ ] `PeerConnection` has `messages_received` field initialized to 0
- [ ] `PeerConnection` has `last_message_time` field set to Unix timestamp at connection time
- [ ] `bytes_written` increments by the message size on every send
- [ ] `messages_sent` increments by 1 on every send
- [ ] `bytes_read` increments by the message size on every receive
- [ ] `messages_received` increments by 1 on every receive
- [ ] `last_message_time` updates to current timestamp on every receive
- [ ] `GossipStats` aggregates metrics across all connections

## Implementation Notes

- Primary files: `src/types/peer.rs` (PeerConnection fields), `src/connection/mod.rs` (metric update hooks), `src/types/stats.rs` (GossipStats aggregation)
- Dependencies: None beyond standard library (time)
- Edge cases:
  - `bytes_read`/`bytes_written` should count the serialized wire size, not the in-memory size
  - `last_message_time` should only update on receive, not send
  - `creation_time` is immutable after initialization
  - Concurrent access: if PeerConnection is shared across tasks, metric updates must be atomic or protected by a lock
  - Overflow: `u64` counters will not realistically overflow (18 exabytes), but the types should remain `u64`
