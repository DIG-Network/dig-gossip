# CON-008: Version String Sanitization

> **Authoritative requirement:** [NORMATIVE.md](../NORMATIVE.md)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **Spec reference:** [SPEC.md - Sections 5.1, 5.2](../../../resources/SPEC.md)

## Summary

The `software_version` field from the Chia `Handshake` must have Unicode Cc (control) and Cf (format) characters stripped before being stored or displayed. This sanitization matches the behavior in Chia's `ws_connection.py:61-63`, which strips these character categories to prevent injection of invisible or control characters into version strings displayed in logs, UIs, or peer exchange messages.

## Specification

### Unicode Categories Stripped

The following Unicode General Categories MUST be stripped:

- **Cc (Control)**: Characters in the range U+0000..U+001F (C0 controls) and U+007F..U+009F (C1 controls including DEL). Examples: null (`\x00`), backspace (`\x08`), newline (`\x0A`), carriage return (`\x0D`), escape (`\x1B`), DEL (`\x7F`).
- **Cf (Format)**: Invisible formatting characters. Examples: zero-width space (`\u{200B}`), zero-width non-joiner (`\u{200C}`), zero-width joiner (`\u{200D}`), left-to-right mark (`\u{200E}`), right-to-left mark (`\u{200F}`), byte order mark (`\u{FEFF}`), soft hyphen (`\u{00AD}`).

### Sanitization Function

```rust
/// Strip Unicode Cc (control) and Cf (format) characters from a version string.
///
/// This matches the sanitization in Chia's ws_connection.py:61-63:
/// ```python
/// printable_version = "".join(
///     [i for i in version if not unicodedata.category(i).startswith(("Cc", "Cf"))]
/// )
/// ```
pub fn sanitize_version(version: &str) -> String {
    version
        .chars()
        .filter(|c| {
            // Check Unicode General Category
            // Cc: General_Category=Control (U+0000..U+001F, U+007F..U+009F)
            // Cf: General_Category=Format (U+00AD, U+0600..U+0605, U+200B..U+200F, etc.)
            !c.is_control() && !is_unicode_format(*c)
        })
        .collect()
}

/// Check if a character is in Unicode General Category Cf (Format).
/// Rust's char::is_control() covers Cc but not Cf.
fn is_unicode_format(c: char) -> bool {
    matches!(unicode_general_category(c), GeneralCategory::Format)
}
```

### Application Points

The `sanitize_version()` function MUST be called:

1. **Outbound connections**: On the `Handshake` response received from the remote peer, before storing in `PeerConnection.software_version`.
2. **Inbound connections**: On the `Handshake` received from the connecting peer, before storing in `PeerConnection.software_version`.

```rust
// In both outbound (CON-001) and inbound (CON-002) connection handling:
let peer_connection = PeerConnection {
    // ...
    software_version: sanitize_version(&their_handshake.software_version),
    // ...
};
```

### Implementation Options for Cf Detection

Since Rust's standard library `char::is_control()` covers Cc but not Cf, Cf detection requires one of:

1. **`unicode-general-category` crate**: Provides `get_general_category()` returning `GeneralCategory::Format` for Cf.
2. **Manual range check**: Enumerate known Cf code point ranges (less maintainable but zero dependencies).
3. **`icu` or `unicode-properties` crates**: Heavier but comprehensive.

The recommended approach is option 1 (`unicode-general-category` crate) or a manual range check for the most common Cf characters:

```rust
/// Check if a character is Unicode General Category Cf (Format).
/// Covers the most common Cf characters encountered in practice.
fn is_unicode_format(c: char) -> bool {
    matches!(c,
        '\u{00AD}'           // Soft hyphen
        | '\u{0600}'..='\u{0605}'  // Arabic number sign..Arabic number mark above
        | '\u{061C}'         // Arabic letter mark
        | '\u{06DD}'         // Arabic end of ayah
        | '\u{070F}'         // Syriac abbreviation mark
        | '\u{08E2}'         // Arabic disputed end of ayah
        | '\u{180E}'         // Mongolian vowel separator
        | '\u{200B}'..='\u{200F}'  // Zero-width space..Right-to-left mark
        | '\u{202A}'..='\u{202E}'  // Bidi embedding controls
        | '\u{2060}'..='\u{2064}'  // Word joiner..Invisible plus
        | '\u{2066}'..='\u{2069}'  // Bidi isolate controls
        | '\u{FEFF}'         // Byte order mark / zero-width no-break space
        | '\u{FFF9}'..='\u{FFFB}'  // Interlinear annotations
        | '\u{110BD}'        // Kaithi number sign
        | '\u{110CD}'        // Kaithi number sign above
        | '\u{13430}'..='\u{13438}'  // Egyptian hieroglyph format characters
        | '\u{1BCA0}'..='\u{1BCA3}'  // Shorthand format controls
        | '\u{1D173}'..='\u{1D17A}'  // Musical symbol format characters
        | '\u{E0001}'        // Language tag
        | '\u{E0020}'..='\u{E007F}'  // Tag components
    )
}
```

## Acceptance Criteria

- [ ] Cc characters (U+0000..U+001F, U+007F..U+009F) are stripped from `software_version`
- [ ] Cf characters (U+00AD, U+200B..U+200F, U+FEFF, etc.) are stripped from `software_version`
- [ ] Normal printable characters are preserved
- [ ] Sanitization is applied to outbound handshake responses (CON-001)
- [ ] Sanitization is applied to inbound handshakes (CON-002)
- [ ] The sanitized version is stored in `PeerConnection.software_version`
- [ ] Empty string after sanitization is valid (no error)
- [ ] Multi-byte UTF-8 characters that are not Cc/Cf are preserved
- [ ] Behavior matches Chia `ws_connection.py:61-63`

## Implementation Notes

- Primary files: `src/connection/mod.rs` or `src/util/mod.rs` (sanitize_version function)
- Dependencies: Optional `unicode-general-category` crate for Cf detection, or manual range check
- Edge cases:
  - A version string consisting entirely of Cc/Cf characters becomes empty after sanitization
  - Emoji and other non-Latin characters (category Lo, So, etc.) MUST be preserved
  - The null character (`\x00`) is Cc and must be stripped
  - BOM (`\u{FEFF}`) at the start of a version string is Cf and must be stripped
  - Right-to-left override (`\u{202E}`) is Cf and must be stripped (prevents display attacks)
  - Tab (`\x09`), newline (`\x0A`), carriage return (`\x0D`) are all Cc and must be stripped
  - The sanitization should be idempotent: `sanitize(sanitize(s)) == sanitize(s)`
