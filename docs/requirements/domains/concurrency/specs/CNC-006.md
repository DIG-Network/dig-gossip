# CNC-006: Periodic Cleanup Task

> **Authoritative requirement:** [NORMATIVE.md](../NORMATIVE.md)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **Spec reference:** [SPEC.md - Section 2.13 (Constants)](../../../resources/SPEC.md)

## Summary

The periodic cleanup task MUST run at a configurable interval. It MUST remove connections with `last_pong` older than `PEER_TIMEOUT_SECS` (90 seconds). It MUST clear expired bans (check `ban_until` against current time). It MUST log removed connections and cleared bans. This prevents resource leaks from stale connections and ensures bans do not persist indefinitely.

## Specification

### Cleanup Task Loop

```rust
use std::time::{SystemTime, UNIX_EPOCH};
use tokio::time::{interval, Duration};

/// Default cleanup interval in seconds.
pub const CLEANUP_INTERVAL_SECS: u64 = 30;

/// Peer timeout (no pong received within this window = stale).
/// SPEC Section 2.13.
pub const PEER_TIMEOUT_SECS: u64 = 90;

/// Run the periodic cleanup task.
/// Spawned by GossipService::start() as CNC-002 task (d).
pub async fn run_cleanup_loop(
    shared_state: Arc<SharedState>,
    reputations: Arc<RwLock<HashMap<PeerId, PeerReputation>>>,
    cleanup_interval_secs: u64,
    mut shutdown: watch::Receiver<bool>,
) {
    let mut ticker = interval(Duration::from_secs(cleanup_interval_secs));

    loop {
        tokio::select! {
            _ = ticker.tick() => {
                run_cleanup_cycle(&shared_state, &reputations).await;
            }
            _ = shutdown.changed() => {
                tracing::info!("cleanup task shutting down");
                break;
            }
        }
    }
}
```

### Stale Connection Removal

```rust
/// Remove connections whose last_pong is older than PEER_TIMEOUT_SECS.
async fn remove_stale_connections(shared_state: &SharedState) -> Vec<PeerId> {
    let now = SystemTime::now()
        .duration_since(UNIX_EPOCH)
        .unwrap()
        .as_secs();

    let threshold = now.saturating_sub(PEER_TIMEOUT_SECS);

    let stale_peers: Vec<PeerId> = {
        let connections = shared_state.connections.read().await;
        connections
            .iter()
            .filter(|(_, conn)| conn.last_pong < threshold)
            .map(|(peer_id, _)| *peer_id)
            .collect()
    };

    for peer_id in &stale_peers {
        shared_state.remove_connection(peer_id).await;
        tracing::info!(
            peer_id = %peer_id,
            "removed stale connection (no pong within {} seconds)",
            PEER_TIMEOUT_SECS
        );
    }

    stale_peers
}
```

### Expired Ban Cleanup

```rust
/// Clear bans whose ban_until timestamp has passed.
async fn clear_expired_bans(
    reputations: &Arc<RwLock<HashMap<PeerId, PeerReputation>>>,
) -> usize {
    let now = SystemTime::now()
        .duration_since(UNIX_EPOCH)
        .unwrap()
        .as_secs();

    let mut reps = reputations.write().await;
    let mut cleared = 0;

    for (peer_id, rep) in reps.iter_mut() {
        if rep.is_banned {
            if let Some(ban_until) = rep.ban_until {
                if now >= ban_until {
                    rep.is_banned = false;
                    rep.ban_until = None;
                    rep.penalty_score = 0;
                    cleared += 1;
                    tracing::info!(
                        peer_id = %peer_id,
                        "cleared expired ban"
                    );
                }
            }
        }
    }

    cleared
}
```

### Combined Cleanup Cycle

```rust
/// Run a single cleanup cycle: remove stale connections + clear expired bans.
async fn run_cleanup_cycle(
    shared_state: &SharedState,
    reputations: &Arc<RwLock<HashMap<PeerId, PeerReputation>>>,
) {
    let stale = remove_stale_connections(shared_state).await;
    let cleared = clear_expired_bans(reputations).await;

    if !stale.is_empty() || cleared > 0 {
        tracing::debug!(
            stale_removed = stale.len(),
            bans_cleared = cleared,
            "cleanup cycle completed"
        );
    }
}
```

## Acceptance Criteria

- [ ] Cleanup task runs at a configurable interval (default: `CLEANUP_INTERVAL_SECS`)
- [ ] Connections with `last_pong` older than `PEER_TIMEOUT_SECS` (90s) are removed
- [ ] Removed stale connections are logged at info level with the peer ID
- [ ] Bans with `ban_until` in the past are cleared (set `is_banned = false`, `ban_until = None`)
- [ ] Cleared bans are logged at info level with the peer ID
- [ ] Cleanup task observes the shutdown signal and exits cleanly
- [ ] Cleanup cycle acquires read lock for scanning, write lock only for removal
- [ ] No cleanup action is taken if all connections are fresh and no bans are expired
- [ ] `penalty_score` is reset to 0 when a ban is cleared

## Implementation Notes

- Primary files: `src/service/cleanup.rs`
- Dependencies: `tokio::time::interval`, `tokio::sync::RwLock`, `std::time::SystemTime`
- Edge cases:
  - If the cleanup interval is shorter than the time to complete a cycle (very unlikely), the next tick will fire immediately after the current cycle completes (tokio interval behavior)
  - A peer that reconnects between the stale scan and the removal will have its connection erroneously removed; the scan and removal should be atomic (hold write lock for both)
  - Clock jumps may cause false positives/negatives; `saturating_sub` prevents underflow
  - `ban_until` may be `None` for permanently banned peers; these are never automatically cleared
