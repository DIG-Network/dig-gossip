# CNC-004: Graceful Shutdown

> **Authoritative requirement:** [NORMATIVE.md](../NORMATIVE.md)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **Spec reference:** [SPEC.md - Section 3.2](../../../resources/SPEC.md)

## Summary

GossipService::stop() MUST gracefully shut down all tasks: cancel discovery, serialization, cleanup, relay tasks. Disconnect all peers. Save address manager state. Close listener. All tasks MUST complete within a bounded timeout. This ensures no data loss (address manager saved) and no resource leaks (all connections closed, all tasks joined).

## Specification

### Shutdown Procedure

```rust
use tokio::time::{timeout, Duration};

/// Shutdown timeout. If tasks do not exit within this duration,
/// they are forcibly cancelled.
const SHUTDOWN_TIMEOUT_SECS: u64 = 10;

impl GossipService {
    /// Gracefully stop: disconnect all peers, stop discovery, close relay.
    /// SPEC Section 3.2.
    pub async fn stop(&self) -> Result<(), GossipError> {
        tracing::info!("stopping gossip service");

        // 1. Signal all tasks to stop via the shared watch channel.
        self.shutdown_tx.send(true)
            .map_err(|_| GossipError::ChannelClosed)?;

        // 2. Disconnect all peers gracefully.
        //    Each per-connection task will observe the shutdown signal
        //    and close its WebSocket connection.
        {
            let connections = self.shared_state.connections.read().await;
            tracing::info!(
                peer_count = connections.len(),
                "disconnecting all peers"
            );
        }
        // Drop all outbound senders to signal per-connection tasks.
        self.shared_state.senders.write().await.clear();

        // 3. Save address manager state.
        //    The serialization task also saves on shutdown,
        //    but we do an explicit save here for safety.
        {
            let am = self.address_manager.read().await;
            if let Err(e) = am.save(&self.config.peers_file_path).await {
                tracing::error!("failed to save address manager on shutdown: {e}");
            }
        }

        // 4. Wait for all tasks to complete (with timeout).
        let join_result = timeout(
            Duration::from_secs(SHUTDOWN_TIMEOUT_SECS),
            self.join_all_tasks(),
        ).await;

        match join_result {
            Ok(Ok(())) => {
                tracing::info!("gossip service stopped cleanly");
                Ok(())
            }
            Ok(Err(e)) => {
                tracing::error!("task error during shutdown: {e}");
                Err(GossipError::InternalError(format!("shutdown error: {e}")))
            }
            Err(_timeout) => {
                tracing::warn!(
                    timeout_secs = SHUTDOWN_TIMEOUT_SECS,
                    "shutdown timed out, forcibly cancelling remaining tasks"
                );
                self.abort_all_tasks();
                Ok(())
            }
        }
    }

    /// Join all spawned task handles.
    async fn join_all_tasks(&self) -> Result<(), String> {
        // Join listener, discovery, serialization, cleanup, relay tasks
        // and all per-connection tasks.
        // Implementation collects all JoinHandles and awaits them.
        Ok(())
    }

    /// Abort all tasks that have not yet completed.
    fn abort_all_tasks(&self) {

}
```

### Shutdown Order

1. Send shutdown signal (watch channel)
2. Disconnect all peers (clear sender map, drop senders)
3. Save address manager state
4. Wait for tasks to complete (bounded timeout)
5. Force-cancel any remaining tasks if timeout exceeded

## Acceptance Criteria

- [ ] `stop()` sends a shutdown signal to all tasks
- [ ] All tasks (listener, discovery, serialization, cleanup, relay, per-connection) observe the shutdown signal and exit
- [ ] All peer connections are closed during shutdown
- [ ] Address manager state is saved to disk before shutdown completes
- [ ] `stop()` waits for all tasks to complete with a bounded timeout
- [ ] If tasks do not complete within the timeout, they are forcibly cancelled
- [ ] `stop()` returns `Ok(())` on clean shutdown
- [ ] `stop()` logs the number of peers disconnected
- [ ] Calling `stop()` on an already-stopped service does not panic

## Implementation Notes

- Primary files: `src/service/gossip_service.rs`
- Dependencies: `tokio::sync::watch`, `tokio::task::JoinHandle`, `tokio::time::timeout`
- Edge cases:
  - A task may panic during shutdown; `JoinHandle::await` returns `Err(JoinError)` for panicked tasks, which should be logged but not propagated as a hard error
  - The listener task may be blocked on `TcpListener::accept()`; the shutdown signal must be checked via `tokio::select!`
  - Per-connection tasks may be waiting on `Peer::recv()`; dropping the sender side of the outbound channel will cause the task to observe closure
  - Double-stop: the second `stop()` call should be a no-op (watch channel already sent `true`)
  - Address manager save failure should be logged but should not prevent the rest of shutdown
