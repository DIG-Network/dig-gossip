# CNC-002: Task Architecture

> **Authoritative requirement:** [NORMATIVE.md](../NORMATIVE.md)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **Spec reference:** [SPEC.md - Section 3.2, Section 9.1](../../../resources/SPEC.md)

## Summary

GossipService::start() MUST spawn a well-defined set of long-lived tasks: (a) Listener task for inbound TCP/TLS connections, (b) Discovery task implementing the connect loop from node_discovery.py:244, (c) Serialization task for periodic address manager persistence, (d) Cleanup task for stale connection removal and expired ban cleanup, (e) Relay task (if configured, with auto-reconnect), and (f) Per-connection tasks (one per peer). All tasks MUST be cancellable via a shared shutdown signal.

## Specification

### Task Spawn on Start

```rust
use tokio::sync::watch;
use tokio::task::JoinHandle;

impl GossipService {
    /// Start the gossip service: bind listener, start discovery, connect to
    /// bootstrap peers, start relay (if configured).
    /// Returns a GossipHandle for the caller to interact with the service.
    /// SPEC Section 3.2.
    pub async fn start(&self) -> Result<GossipHandle, GossipError> {
        let (shutdown_tx, shutdown_rx) = watch::channel(false);
        let (inbound_tx, inbound_rx) = mpsc::channel(1024);

        // (a) Listener task: accepts inbound TCP/TLS connections (CON-002)
        let listener_handle = tokio::spawn({
            let shutdown = shutdown_rx.clone();
            let config = self.config.clone();
            async move {
                listener::run_listener(config, shutdown).await;
            }
        });

        // (b) Discovery task: connect loop (DSC-006, node_discovery.py:244)
        let discovery_handle = tokio::spawn({
            let shutdown = shutdown_rx.clone();
            let address_manager = self.address_manager.clone();
            async move {
                discovery::run_discovery_loop(address_manager, shutdown).await;
            }
        });

        // (c) Serialization task: periodic address manager save (DSC-002)
        let serialization_handle = tokio::spawn({
            let shutdown = shutdown_rx.clone();
            let address_manager = self.address_manager.clone();
            let peers_file = self.config.peers_file_path.clone();
            async move {
                address_manager_store::run_periodic_save(
                    address_manager, peers_file, shutdown
                ).await;
            }
        });

        // (d) Cleanup task: stale connection removal, expired ban cleanup (CNC-006)
        let cleanup_handle = tokio::spawn({
            let shutdown = shutdown_rx.clone();
            async move {
                cleanup::run_cleanup_loop(shutdown).await;
            }
        });

        // (e) Relay task: if configured, auto-reconnect (RLY-001, RLY-004)
        let relay_handle = if self.config.relay.is_some() {
            Some(tokio::spawn({
                let shutdown = shutdown_rx.clone();
                let relay_config = self.config.relay.clone().unwrap();
                async move {
                    relay::run_relay_service(relay_config, shutdown).await;
                }
            }))
        } else {
            None
        };

        // (f) Per-connection tasks are spawned dynamically by the listener
        //     and discovery tasks as connections are established.

        Ok(GossipHandle::new(
            shutdown_tx,
            inbound_rx,
            /* task handles for join on stop */
        ))
    }
}
```

### Per-Connection Task Spawning

```rust
/// Spawn a per-connection message handling task for a newly connected peer.
/// Called by both the listener (inbound) and discovery (outbound) tasks.
fn spawn_connection_task(
    peer_id: PeerId,
    peer: Peer,
    peer_rx: mpsc::Receiver<Message>,
    inbound_tx: mpsc::Sender<(PeerId, Message)>,
    outbound_rx: mpsc::Receiver<Message>,
    shutdown: watch::Receiver<bool>,
) -> JoinHandle<()> {
    tokio::spawn(async move {
        connection::run_peer_loop(
            peer_id, peer, peer_rx, inbound_tx, outbound_rx, shutdown
        ).await;
    })
}
```

### Shutdown Signal

All tasks share a `watch::Receiver<bool>` for cooperative shutdown. When `stop()` is called, the sender broadcasts `true`, and each task checks for this signal in its main loop via `tokio::select!`.

## Acceptance Criteria

- [ ] `GossipService::start()` spawns a Listener task
- [ ] `GossipService::start()` spawns a Discovery task
- [ ] `GossipService::start()` spawns a Serialization task
- [ ] `GossipService::start()` spawns a Cleanup task
- [ ] `GossipService::start()` spawns a Relay task when `config.relay` is `Some`
- [ ] `GossipService::start()` does NOT spawn a Relay task when `config.relay` is `None`
- [ ] Per-connection tasks are spawned dynamically when peers connect
- [ ] All tasks receive a shared `watch::Receiver<bool>` shutdown signal
- [ ] All tasks exit cleanly when the shutdown signal is received
- [ ] `start()` returns a `GossipHandle` for the caller

## Implementation Notes

- Primary files: `src/service/gossip_service.rs`, `src/connection/listener.rs`, `src/discovery/node_discovery.rs`
- Dependencies: `tokio::spawn`, `tokio::sync::watch`, `tokio::task::JoinHandle`
- Edge cases:
  - If the listener bind fails, `start()` should return an error before spawning other tasks
  - If the relay connection fails on initial connect, the relay task should enter its auto-reconnect loop (RLY-004) rather than failing `start()`
  - Per-connection tasks must be tracked (JoinHandle or AbortHandle) for cleanup on shutdown
  - A task panic should not crash the entire service; use `JoinHandle` error propagation or `catch_unwind`
