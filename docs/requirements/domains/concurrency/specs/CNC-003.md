# CNC-003: Shared State Synchronization Primitives

> **Authoritative requirement:** [NORMATIVE.md](../NORMATIVE.md)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **Spec reference:** [SPEC.md - Section 9.1](../../../resources/SPEC.md)

## Summary

Connection map and sender map MUST be protected by RwLock (read-heavy, low contention). Per-connection outbound channels MUST be lock-free mpsc. Stats counters MUST be AtomicU64. These choices optimize for the gossip workload where reads (routing lookups, stats queries) vastly outnumber writes (new connections, disconnects).

## Specification

### RwLock for Connection and Sender Maps

```rust
use std::collections::HashMap;
use std::sync::Arc;
use tokio::sync::RwLock;

/// Shared connection state, protected by RwLock.
/// Read-heavy: every message route checks the connection map.
/// Write-rare: only on connect/disconnect events.
pub struct SharedState {
    /// Map of peer_id -> PeerConnection metadata.
    /// Protected by RwLock for concurrent read access.
    pub connections: Arc<RwLock<HashMap<PeerId, PeerConnection>>>,

    /// Map of peer_id -> outbound message sender.
    /// Protected by RwLock. Readers: broadcast/routing.
    /// Writers: connect/disconnect.
    pub senders: Arc<RwLock<HashMap<PeerId, mpsc::Sender<Message>>>>,
}

impl SharedState {
    /// Broadcast to all connected peers (read lock only).
    pub async fn broadcast(
        &self,
        message: &Message,
        exclude: Option<PeerId>,
    ) -> usize {
        let senders = self.senders.read().await;
        let mut count = 0;
        for (peer_id, sender) in senders.iter() {
            if exclude.map_or(true, |ex| ex != *peer_id) {
                if sender.try_send(message.clone()).is_ok() {
                    count += 1;
                }
            }
        }
        count
    }

    /// Add a new connection (write lock).
    pub async fn add_connection(
        &self,
        peer_id: PeerId,
        connection: PeerConnection,
        sender: mpsc::Sender<Message>,
    ) {
        self.connections.write().await.insert(peer_id, connection);
        self.senders.write().await.insert(peer_id, sender);
    }

    /// Remove a connection (write lock).
    pub async fn remove_connection(&self, peer_id: &PeerId) {
        self.connections.write().await.remove(peer_id);
        self.senders.write().await.remove(peer_id);
    }
}
```

### Lock-Free mpsc for Per-Connection Outbound

```rust
use tokio::sync::mpsc;

/// Per-connection outbound message channel.
/// Lock-free: the sender is stored in the shared senders map,
/// the receiver is owned by the per-connection task.
///
/// Buffer size is bounded to provide backpressure.
const PER_CONNECTION_CHANNEL_SIZE: usize = 256;

fn create_outbound_channel() -> (mpsc::Sender<Message>, mpsc::Receiver<Message>) {
    mpsc::channel(PER_CONNECTION_CHANNEL_SIZE)
}
```

### AtomicU64 for Stats Counters

```rust
use std::sync::atomic::{AtomicU64, Ordering};

/// Atomic stats counters for lock-free increment/read.
/// These are hot-path counters updated on every message.
pub struct GossipStatsAtomic {
    pub messages_sent: AtomicU64,
    pub messages_received: AtomicU64,
    pub bytes_sent: AtomicU64,
    pub bytes_received: AtomicU64,
    pub connections_total: AtomicU64,
    pub disconnections_total: AtomicU64,
}

impl GossipStatsAtomic {
    pub fn new() -> Self {
        Self {
            messages_sent: AtomicU64::new(0),
            messages_received: AtomicU64::new(0),
            bytes_sent: AtomicU64::new(0),
            bytes_received: AtomicU64::new(0),
            connections_total: AtomicU64::new(0),
            disconnections_total: AtomicU64::new(0),
        }
    }

    pub fn inc_messages_sent(&self) {
        self.messages_sent.fetch_add(1, Ordering::Relaxed);
    }

    pub fn inc_messages_received(&self) {
        self.messages_received.fetch_add(1, Ordering::Relaxed);
    }

    /// Snapshot the current stats into a GossipStats struct.
    pub fn snapshot(&self) -> GossipStats {
        GossipStats {
            messages_sent: self.messages_sent.load(Ordering::Relaxed),
            messages_received: self.messages_received.load(Ordering::Relaxed),
            bytes_sent: self.bytes_sent.load(Ordering::Relaxed),
            bytes_received: self.bytes_received.load(Ordering::Relaxed),
            connections_total: self.connections_total.load(Ordering::Relaxed),
            disconnections_total: self.disconnections_total.load(Ordering::Relaxed),
        }
    }
}
```

## Acceptance Criteria

- [ ] Connection map (`HashMap<PeerId, PeerConnection>`) is behind `Arc<RwLock<>>`
- [ ] Sender map (`HashMap<PeerId, mpsc::Sender<Message>>`) is behind `Arc<RwLock<>>`
- [ ] Broadcast operations acquire only a read lock on the sender map
- [ ] Connect/disconnect operations acquire a write lock on both maps
- [ ] Per-connection outbound channels use `tokio::sync::mpsc` (lock-free)
- [ ] Per-connection channel is bounded (backpressure support)
- [ ] Stats counters (`messages_sent`, `messages_received`, `bytes_sent`, `bytes_received`) are `AtomicU64`
- [ ] Stats counters use `Ordering::Relaxed` (no cross-counter ordering needed)
- [ ] `GossipStatsAtomic::snapshot()` returns a non-atomic `GossipStats` copy

## Implementation Notes

- Primary files: `src/service/gossip_handle.rs`, `src/types/stats.rs`
- Dependencies: `tokio::sync::RwLock`, `tokio::sync::mpsc`, `std::sync::atomic::AtomicU64`
- Edge cases:
  - Write lock starvation: tokio's RwLock is write-preferring, so frequent writes could block reads; given connect/disconnect rarity, this is acceptable
  - Channel full: `try_send` returns an error when the per-connection buffer is full; the caller should handle this via backpressure (PRI domain)
  - Ordering::Relaxed is sufficient for independent counters; if cross-counter consistency is needed (e.g., "sent at time of read"), use `Ordering::SeqCst` or snapshot under a lock
  - `Arc<RwLock<HashMap>>` vs `DashMap`: RwLock is chosen for simplicity and fewer dependencies; DashMap could be considered if contention is measured
