# PRI-008: Normal Delay Under Backpressure

> **Authoritative requirement:** [NORMATIVE.md](../NORMATIVE.md)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **Spec reference:** [SPEC.md - Section 8.5](../../../resources/SPEC.md)

## Summary

When the outbound queue depth reaches or exceeds the `normal_delay_threshold` (default 100), Normal-priority messages are delayed rather than immediately enqueued. Delayed messages are batched and flushed every 500ms. Critical messages are never affected by any backpressure level, ensuring consensus-critical traffic always flows without delay.

## Specification

### Normal Delay Behavior

When `BackpressureLevel >= NormalDelay`, Normal-priority messages MUST be held in a delay buffer rather than enqueued directly:

```rust
use std::collections::VecDeque;
use std::time::{Duration, Instant};
use chia_protocol::Message;

/// Delay buffer for Normal messages under high backpressure.
pub struct NormalDelayBuffer {
    /// Buffered Normal messages waiting to be flushed.
    buffer: VecDeque<Message>,
    /// Interval between batch flushes.
    flush_interval: Duration,
    /// Last time the buffer was flushed.
    last_flush: Instant,
}

/// Default flush interval for delayed Normal messages.
pub const NORMAL_DELAY_FLUSH_INTERVAL_MS: u64 = 500;

impl NormalDelayBuffer {
    pub fn new() -> Self {
        Self {
            buffer: VecDeque::new(),
            flush_interval: Duration::from_millis(NORMAL_DELAY_FLUSH_INTERVAL_MS),
            last_flush: Instant::now(),
        }
    }

    /// Add a Normal message to the delay buffer.
    pub fn delay(&mut self, msg: Message) {
        self.buffer.push_back(msg);
    }

    /// Check if the buffer should be flushed.
    pub fn should_flush(&self) -> bool {
        !self.buffer.is_empty() && self.last_flush.elapsed() >= self.flush_interval
    }

    /// Flush all buffered messages into the outbound queue.
    /// Returns the messages to be enqueued.
    pub fn flush(&mut self) -> Vec<Message> {
        self.last_flush = Instant::now();
        self.buffer.drain(..).collect()
    }

    /// Number of messages currently buffered.
    pub fn len(&self) -> usize {
        self.buffer.len()
    }

    /// Returns true if no messages are buffered.
    pub fn is_empty(&self) -> bool {
        self.buffer.is_empty()
    }
}
```

### Enqueue Decision Flow (Complete)

The complete `enqueue_with_backpressure` method with all backpressure levels:

```rust
impl PriorityOutbound {
    pub fn enqueue_with_backpressure(
        &mut self,
        msg: Message,
        priority: MessagePriority,
        bp_config: &BackpressureConfig,
        tx_dedup: &mut TxDedupFilter,
        delay_buffer: &mut NormalDelayBuffer,
    ) -> EnqueueResult {
        let level = bp_config.level(self.total_len());

        // PRI-006: Transaction dedup at >= TxDedup
        if level >= BackpressureLevel::TxDedup {
            if let Some(tx_id) = extract_new_transaction_id(&msg) {
                if tx_dedup.should_suppress(&tx_id) {
                    return EnqueueResult::Suppressed;
                }
            }
        }

        // PRI-007: Bulk drop at >= BulkDrop
        if level >= BackpressureLevel::BulkDrop && priority == MessagePriority::Bulk {
            return EnqueueResult::Dropped;
        }

        // PRI-008: Normal delay at >= NormalDelay
        if level >= BackpressureLevel::NormalDelay && priority == MessagePriority::Normal {
            delay_buffer.delay(msg);
            return EnqueueResult::Delayed;
        }

        // Critical messages always enqueued immediately.
        // Normal messages enqueued immediately below NormalDelay.
        // Bulk messages enqueued normally below BulkDrop.
        self.enqueue(msg, priority);
        EnqueueResult::Enqueued
    }
}
```

### Flush Timer Integration

A periodic timer (tokio interval) in the connection's send task MUST flush the delay buffer every 500ms:

```rust
// In the connection send task:
let mut flush_interval = tokio::time::interval(
    Duration::from_millis(NORMAL_DELAY_FLUSH_INTERVAL_MS)
);

loop {
    tokio::select! {
        // ... other branches (new messages, drain_next, etc.) ...

        _ = flush_interval.tick() => {
            if delay_buffer.should_flush() {
                for msg in delay_buffer.flush() {
                    outbound.enqueue(msg, MessagePriority::Normal);
                }
            }
        }
    }
}
```

### Critical Always Unaffected

Critical messages MUST be enqueued immediately at all backpressure levels:

| Backpressure Level | Critical | Normal | Bulk |
|---------------------|----------|--------|------|
| None | Enqueued | Enqueued | Enqueued |
| TxDedup | Enqueued | Enqueued (dedup on NewTx) | Enqueued |
| BulkDrop | Enqueued | Enqueued (dedup on NewTx) | Dropped |
| NormalDelay | Enqueued | Delayed (500ms batch) | Dropped |

### Recovery Behavior

When queue depth drops below `normal_delay_threshold`:
1. New Normal messages are enqueued immediately (no longer delayed).
2. Any messages remaining in the `NormalDelayBuffer` MUST be flushed promptly (on the next flush tick or immediately upon detecting recovery).

## Acceptance Criteria

- [ ] At queue depth >= 100 (default), Normal messages return `EnqueueResult::Delayed`
- [ ] At queue depth < 100, Normal messages are enqueued immediately
- [ ] Critical messages are enqueued immediately at queue depth >= 100
- [ ] Critical messages are enqueued immediately at all backpressure levels
- [ ] Delayed Normal messages are batched in `NormalDelayBuffer`
- [ ] `NormalDelayBuffer` flushes every 500ms
- [ ] `NORMAL_DELAY_FLUSH_INTERVAL_MS` constant is 500
- [ ] Flushed messages are enqueued into the Normal lane of PriorityOutbound
- [ ] Flushed messages maintain FIFO order (delay order preserved)
- [ ] Bulk messages are dropped at NormalDelay level (cumulative with PRI-007)
- [ ] Tx dedup is active at NormalDelay level (cumulative with PRI-006)
- [ ] On recovery below threshold, buffered messages are flushed promptly
- [ ] On recovery, new Normal messages are enqueued immediately

## Implementation Notes

- Primary files: `src/gossip/backpressure.rs`, `src/gossip/priority.rs`
- Dependencies: `BackpressureConfig` (PRI-005), `PriorityOutbound` (PRI-002), `MessagePriority` (PRI-001), `TxDedupFilter` (PRI-006)
- Edge cases:
  - If the flush timer fires but backpressure is still at NormalDelay, the flushed messages are enqueued into the Normal lane. The drain algorithm (PRI-003) will process them in priority order. This may cause queue depth to remain above threshold, causing the next batch of Normal messages to also be delayed. This is by design: it creates a natural rate-limiting effect.
  - If the delay buffer grows very large, it acts as a secondary queue. In practice, 500ms batching at typical transaction rates (~100 tx/s) means ~50 messages per flush, which is manageable.
  - The 500ms interval is a balance between latency (lower is better) and batching efficiency (higher reduces per-message overhead). It should be configurable in future versions but 500ms is the hardcoded default per SPEC.
  - `Instant::now()` is monotonic, so clock adjustments do not affect flush timing.
  - The delay buffer is per-connection, so one congested connection does not affect others.
