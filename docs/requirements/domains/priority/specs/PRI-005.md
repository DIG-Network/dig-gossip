# PRI-005: BackpressureConfig

> **Authoritative requirement:** [NORMATIVE.md](../NORMATIVE.md)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **Spec reference:** [SPEC.md - Section 8.5](../../../resources/SPEC.md)

## Summary

`BackpressureConfig` defines the three queue-depth thresholds that control adaptive backpressure behavior. When outbound queue depth crosses these thresholds, progressively more aggressive measures are taken to reduce non-critical traffic: transaction deduplication, bulk dropping, and normal message delay. The thresholds have sensible defaults but are configurable per deployment.

## Specification

### BackpressureConfig Struct

```rust
/// Configuration for adaptive backpressure thresholds.
/// Queue depth is measured as `PriorityOutbound::total_len()`.
#[derive(Debug, Clone)]
pub struct BackpressureConfig {
    /// Queue depth at which Normal messages start being delayed
    /// (batched, sent every 500ms).
    /// Default: 100 messages.
    pub normal_delay_threshold: usize,

    /// Queue depth at which Bulk messages are dropped silently
    /// and ERLAY reconciliation is paused.
    /// Default: 50 messages.
    pub bulk_drop_threshold: usize,

    /// Queue depth at which duplicate NewTransaction announcements
    /// are suppressed (only first per tx_id passes).
    /// Default: 25 messages.
    pub tx_dedup_threshold: usize,
}

impl Default for BackpressureConfig {
    fn default() -> Self {
        Self {
            normal_delay_threshold: 100,
            bulk_drop_threshold: 50,
            tx_dedup_threshold: 25,
        }
    }
}
```

### Threshold Ordering Invariant

The thresholds MUST satisfy: `tx_dedup_threshold <= bulk_drop_threshold <= normal_delay_threshold`. This ensures backpressure escalates progressively. A constructor or validation method SHOULD enforce this:

```rust
impl BackpressureConfig {
    /// Create a custom BackpressureConfig.
    /// Panics if thresholds are not in ascending order.
    pub fn new(
        tx_dedup_threshold: usize,
        bulk_drop_threshold: usize,
        normal_delay_threshold: usize,
    ) -> Self {
        assert!(
            tx_dedup_threshold <= bulk_drop_threshold
                && bulk_drop_threshold <= normal_delay_threshold,
            "Backpressure thresholds must be in ascending order: \
             tx_dedup({}) <= bulk_drop({}) <= normal_delay({})",
            tx_dedup_threshold, bulk_drop_threshold, normal_delay_threshold,
        );
        Self {
            normal_delay_threshold,
            bulk_drop_threshold,
            tx_dedup_threshold,
        }
    }

    /// Returns the current backpressure level for a given queue depth.
    pub fn level(&self, queue_depth: usize) -> BackpressureLevel {
        if queue_depth >= self.normal_delay_threshold {
            BackpressureLevel::NormalDelay
        } else if queue_depth >= self.bulk_drop_threshold {
            BackpressureLevel::BulkDrop
        } else if queue_depth >= self.tx_dedup_threshold {
            BackpressureLevel::TxDedup
        } else {
            BackpressureLevel::None
        }
    }
}

/// Current backpressure severity level.
#[derive(Debug, Clone, Copy, PartialEq, Eq, PartialOrd, Ord)]
pub enum BackpressureLevel {
    /// No backpressure. All messages sent normally.
    None = 0,
    /// Duplicate transaction announcements suppressed.
    TxDedup = 1,
    /// Bulk messages dropped. ERLAY paused.
    BulkDrop = 2,
    /// Normal messages delayed (batched every 500ms).
    NormalDelay = 3,
}
```

### Backpressure Level Ranges

| Queue Depth | BackpressureLevel | Actions |
|-------------|-------------------|---------|
| 0 - 24 | None | Normal operation. All messages sent. |
| 25 - 49 | TxDedup | Duplicate NewTransaction suppressed (PRI-006). |
| 50 - 99 | BulkDrop | Bulk dropped, ERLAY paused (PRI-007). TxDedup also active. |
| 100+ | NormalDelay | Normal delayed (PRI-008). BulkDrop and TxDedup also active. |

### Cumulative Application

Backpressure levels are cumulative: at `NormalDelay`, all lower-level actions (TxDedup and BulkDrop) are also active. The `level()` method returns the highest active level; callers check `level >= threshold` for each action.

## Acceptance Criteria

- [ ] `BackpressureConfig` struct exists with fields `normal_delay_threshold`, `bulk_drop_threshold`, `tx_dedup_threshold`
- [ ] `Default::default()` returns thresholds 100, 50, 25 respectively
- [ ] `BackpressureConfig::new()` validates ascending order of thresholds
- [ ] `BackpressureConfig::new()` panics if thresholds are out of order
- [ ] `level(0)` returns `BackpressureLevel::None`
- [ ] `level(24)` returns `BackpressureLevel::None`
- [ ] `level(25)` returns `BackpressureLevel::TxDedup`
- [ ] `level(49)` returns `BackpressureLevel::TxDedup`
- [ ] `level(50)` returns `BackpressureLevel::BulkDrop`
- [ ] `level(99)` returns `BackpressureLevel::BulkDrop`
- [ ] `level(100)` returns `BackpressureLevel::NormalDelay`
- [ ] `level(500)` returns `BackpressureLevel::NormalDelay`
- [ ] `BackpressureLevel` derives `PartialOrd` with `None < TxDedup < BulkDrop < NormalDelay`
- [ ] Backpressure levels are cumulative (higher levels imply all lower-level actions)
- [ ] Custom thresholds are respected (e.g., `new(10, 20, 30)` adjusts all levels)

## Implementation Notes

- Primary file: `src/gossip/backpressure.rs`
- Dependencies: None (this is a configuration struct)
- Edge cases:
  - Zero thresholds are valid: `new(0, 0, 0)` means all backpressure is always active
  - Very large thresholds effectively disable backpressure (e.g., `new(usize::MAX, usize::MAX, usize::MAX)`)
  - The `level()` method is called on every enqueue decision, so it must be O(1) -- the simple threshold comparison satisfies this
  - `BackpressureConfig` should be stored in `GossipConfig` and shared (read-only) across all connections
