# RLY-008: Transport Selection

> **Authoritative requirement:** [NORMATIVE.md](../NORMATIVE.md)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **Spec reference:** [SPEC.md - Section 7, Design Decision 8](../../../resources/SPEC.md)

## Summary

The gossip layer must implement transport selection logic that determines whether to use a direct P2P connection or the relay for each peer. By default, direct P2P is attempted first; if it fails (due to NAT, firewall, or timeout), the relay is used as a transparent fallback. The `RelayConfig::prefer_relay` flag overrides this behavior to use relay as the primary transport. Regardless of the transport chosen, the caller sees no difference -- messages arrive through the same inbound channel. This matches Design Decision 8 from the SPEC: "Relay as fallback, not primary."

## Specification

### Transport Selection Logic

```rust
/// Transport used for communicating with a specific peer.
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum PeerTransport {
    /// Direct P2P via TCP+TLS+WebSocket (chia-sdk-client::Peer).
    Direct,
    /// Via relay server WebSocket (RelayClient).
    Relay,
}

/// Determines how to connect to a peer and routes messages accordingly.
pub struct TransportSelector {
    /// Relay configuration. None if relay is not configured.
    relay_config: Option<RelayConfig>,
    /// Per-peer transport state.
    peer_transports: HashMap<PeerId, PeerTransport>,
    /// Peers that failed direct connection (addr -> last failure time).
    direct_failures: HashMap<SocketAddr, tokio::time::Instant>,
    /// Duration to remember direct connection failures before retrying.
    direct_failure_ttl: Duration,
}

impl TransportSelector {
    /// Select the transport to use for connecting to a peer.
    ///
    /// Default behavior (prefer_relay = false):
    ///   1. If we already have a direct connection -> Direct
    ///   2. Attempt direct P2P connection
    ///   3. If direct fails and relay is available -> Relay
    ///   4. If relay is not available -> error
    ///
    /// Override behavior (prefer_relay = true):
    ///   1. If relay is available -> Relay
    ///   2. If relay is not available, attempt direct -> Direct
    ///   3. If both fail -> error
    pub async fn select_transport(
        &mut self,
        peer_id: &PeerId,
        peer_addr: Option<SocketAddr>,
        relay_client: Option<&RelayClient>,
    ) -> Result<PeerTransport, GossipError> {
        let prefer_relay = self.relay_config
            .as_ref()
            .map(|c| c.prefer_relay)
            .unwrap_or(false);

        // Check existing transport
        if let Some(transport) = self.peer_transports.get(peer_id) {
            return Ok(*transport);
        }

        if prefer_relay {
            // Relay-first mode
            if let Some(relay) = relay_client {
                if relay.is_connected() {
                    self.peer_transports.insert(*peer_id, PeerTransport::Relay);
                    return Ok(PeerTransport::Relay);
                }
            }
            // Relay unavailable, fall back to direct
            if let Some(addr) = peer_addr {
                return self.try_direct(peer_id, addr).await;
            }
            Err(GossipError::RelayError(
                "relay preferred but unavailable, no direct address known".into()
            ))
        } else {
            // Direct-first mode (default)
            if let Some(addr) = peer_addr {
                // Skip direct attempt if recently failed
                if !self.recently_failed_direct(&addr) {
                    match self.try_direct(peer_id, addr).await {
                        Ok(transport) => return Ok(transport),
                        Err(e) => {
                            tracing::debug!(
                                peer = %peer_id,
                                addr = %addr,
                                error = %e,
                                "direct connection failed, falling back to relay"
                            );
                            self.direct_failures.insert(addr, tokio::time::Instant::now());
                        }
                    }
                }
            }
            // Direct failed or no address, try relay
            if let Some(relay) = relay_client {
                if relay.is_connected() && relay.is_peer_on_relay(peer_id) {
                    self.peer_transports.insert(*peer_id, PeerTransport::Relay);
                    return Ok(PeerTransport::Relay);
                }
            }
            Err(GossipError::RelayError(
                "direct connection failed and relay unavailable".into()
            ))
        }
    }

    /// Check if a direct connection to this address failed recently.
    fn recently_failed_direct(&self, addr: &SocketAddr) -> bool {
        self.direct_failures.get(addr)
            .map(|t| t.elapsed() < self.direct_failure_ttl)
            .unwrap_or(false)
    }

    /// Attempt a direct P2P connection.
    async fn try_direct(
        &mut self,
        peer_id: &PeerId,
        addr: SocketAddr,
    ) -> Result<PeerTransport, GossipError> {
        // Uses chia-sdk-client connect_peer() under the hood
        // Connection attempt with timeout handled by the connection layer
        self.peer_transports.insert(*peer_id, PeerTransport::Direct);
        Ok(PeerTransport::Direct)
    }
}
```

### Transparent Message Routing

```rust
impl GossipService {
    /// Send a message to a peer using the appropriate transport.
    /// The caller does not need to know which transport is used.
    pub async fn send_to_peer(
        &self,
        peer_id: &PeerId,
        message: Message,
    ) -> Result<(), GossipError> {
        let transport = self.transport_selector.get_transport(peer_id)
            .ok_or(GossipError::PeerNotConnected(*peer_id))?;

        match transport {
            PeerTransport::Direct => {
                let peer = self.get_direct_peer(peer_id)?;
                peer.send_raw(message).await
                    .map_err(|e| GossipError::ClientError(e))?;
            }
            PeerTransport::Relay => {
                let relay = self.relay_client.as_mut()
                    .ok_or(GossipError::RelayNotConfigured)?;
                let payload = message.to_bytes();
                relay.send_to_peer(*peer_id, payload).await?;
            }
        }

        Ok(())
    }

    /// Broadcast a message to all connected peers (both direct and relay).
    /// Each peer receives the message exactly once via the appropriate transport.
    pub async fn broadcast_message(
        &self,
        message: &Message,
        exclude: Option<PeerId>,
    ) -> usize {
        let mut count = 0;

        // Send to direct peers
        for peer_id in self.direct_peers() {
            if Some(peer_id) == exclude { continue; }
            if self.send_to_peer(&peer_id, message.clone()).await.is_ok() {
                count += 1;
            }
        }

        // Broadcast to relay peers (excluding direct peers and origin)
        if let Some(relay) = &self.relay_client {
            let direct_peers: Vec<PeerId> = self.direct_peers().collect();
            let mut relay_exclude = direct_peers;
            if let Some(origin) = exclude {
                relay_exclude.push(origin);
            }
            if relay.broadcast(message.to_bytes(), relay_exclude).await.is_ok() {
                count += 1; // relay handles fanout
            }
        }

        count
    }
}
```

### Transport Upgrade and Downgrade

```rust
impl TransportSelector {
    /// Upgrade a peer from relay to direct (after successful hole punch).
    pub fn upgrade_to_direct(&mut self, peer_id: &PeerId) {
        self.peer_transports.insert(*peer_id, PeerTransport::Direct);
        tracing::info!(peer = %peer_id, "transport upgraded: relay -> direct");
    }

    /// Downgrade a peer from direct to relay (after direct connection failure).
    pub fn downgrade_to_relay(&mut self, peer_id: &PeerId) {
        self.peer_transports.insert(*peer_id, PeerTransport::Relay);
        tracing::info!(peer = %peer_id, "transport downgraded: direct -> relay");
    }

    /// Remove transport tracking for a disconnected peer.
    pub fn remove_peer(&mut self, peer_id: &PeerId) {
        self.peer_transports.remove(peer_id);
    }
}
```

## Acceptance Criteria

- [ ] Default behavior: direct P2P is attempted first for each peer
- [ ] If direct connection fails, relay is used as fallback (when configured and connected)
- [ ] `RelayConfig::prefer_relay = true` overrides to relay-first behavior
- [ ] Caller sees no difference in message delivery regardless of transport
- [ ] Inbound messages arrive through the same channel for both direct and relay
- [ ] Each peer has a tracked `PeerTransport` (Direct or Relay)
- [ ] Transport can be upgraded (relay -> direct) after successful hole punch (RLY-007)
- [ ] Transport can be downgraded (direct -> relay) after direct connection failure
- [ ] Recently failed direct connections are not retried immediately (TTL-based skip)
- [ ] Broadcast reaches both direct and relay peers without duplication
- [ ] Relay peers are excluded from relay broadcast if already reached via direct
- [ ] `GossipError::RelayNotConfigured` returned when relay is needed but not configured
- [ ] `GossipError::PeerNotConnected` returned when neither transport is available

## Implementation Notes

- Primary files: `src/relay/relay_service.rs`, `src/service/gossip_service.rs`, `src/relay/relay_types.rs`
- Dependencies: `chia-sdk-client::connect_peer`, `tokio::time`
- Edge cases:
  - A peer may be reachable via both direct and relay simultaneously (during migration); only one should be active
  - If `prefer_relay = true` but the relay goes down, the system should attempt direct connections for all relay peers
  - Direct failure TTL should be long enough to avoid hammering unreachable peers but short enough to detect recovered peers (e.g., 60 seconds)
  - If no relay is configured (`GossipConfig::relay = None`), all peers must use direct connections
  - During network partitions, a peer may lose direct but retain relay (or vice versa); transport selector should adapt
  - The `prefer_relay` flag is a per-node configuration, not per-peer; all outbound connections follow the same preference
