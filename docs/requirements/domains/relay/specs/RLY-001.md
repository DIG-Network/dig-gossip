# RLY-001: Relay Client Connect and Register

> **Authoritative requirement:** [NORMATIVE.md](../NORMATIVE.md)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **Spec reference:** [SPEC.md - Section 7, Section 5.1](../../../resources/SPEC.md)

## Summary

The relay client must establish a WebSocket connection to the configured relay server endpoint and perform a registration handshake. The client sends a `Register` message containing its peer identity, network affiliation, and protocol version. The server responds with `RegisterAck` indicating success or failure, along with the current count of connected peers on the relay. This is the entry point for all relay-based communication and must handle connection failures, timeouts, and rejected registrations gracefully.

## Specification

### Complete RelayMessage Enum

The full `RelayMessage` enum defines all relay protocol messages. Individual variants are specified in detail by RLY-001 through RLY-007. The complete enum is defined here as the canonical reference.

```rust
use serde::{Deserialize, Serialize};
use std::net::SocketAddr;

/// Relay protocol messages, serialized as JSON over WebSocket.
/// All relay communication uses this enum.
#[derive(Debug, Clone, Serialize, Deserialize)]
#[serde(tag = "type")]
pub enum RelayMessage {
    /// Sent by client immediately after WebSocket connection is established.
    /// (RLY-001)
    Register {
        /// The connecting peer's unique identifier (SHA256 of TLS public key).
        peer_id: PeerId,
        /// Network identifier (e.g., SHA256("dig_mainnet")).
        network_id: Bytes32,
        /// Protocol version for compatibility checking.
        protocol_version: u32,
    },
    /// Sent by server in response to Register.
    /// (RLY-001)
    RegisterAck {
        /// Whether registration was accepted.
        success: bool,
        /// Human-readable status or error message.
        message: String,
        /// Number of peers currently connected to this relay on the same network.
        connected_peers: usize,
    },
    /// Sent by client to disconnect gracefully from the relay.
    Unregister {
        /// The disconnecting peer's ID.
        peer_id: PeerId,
    },
    /// Forward a gossip message to a specific peer via the relay.
    /// (RLY-002)
    RelayGossipMessage {
        /// Sender's PeerId.
        from: PeerId,
        /// Target peer's PeerId.
        to: PeerId,
        /// Serialized message payload (chia-protocol::Message bytes).
        payload: Vec<u8>,
        /// Monotonically increasing sequence number per sender-receiver pair.
        seq: u64,
    },
    /// Broadcast a message to all relay-connected peers on the same network.
    /// (RLY-003)
    Broadcast {
        /// Sender's PeerId.
        from: PeerId,
        /// Serialized message payload.
        payload: Vec<u8>,
        /// PeerIds to exclude from receiving this broadcast.
        exclude: Vec<PeerId>,
    },
    /// Notification from relay that a new peer has connected.
    PeerConnected {
        /// The newly connected peer's ID.
        peer_id: PeerId,
        /// The peer's network ID.
        network_id: Bytes32,
    },
    /// Notification from relay that a peer has disconnected.
    PeerDisconnected {
        /// The disconnected peer's ID.
        peer_id: PeerId,
    },
    /// Request the list of peers connected to the relay on a given network.
    /// (RLY-005)
    GetPeers {
        /// Network ID to filter peers by.
        network_id: Bytes32,
    },
    /// Response containing the list of connected relay peers.
    /// (RLY-005)
    Peers {
        /// Peers currently connected to the relay on the requested network.
        peers: Vec<RelayPeerInfo>,
    },
    /// Keepalive ping sent by the client.
    /// (RLY-006)
    Ping {
        /// Timestamp (milliseconds since epoch) for RTT calculation.
        timestamp: u64,
    },
    /// Keepalive pong sent by the server in response to Ping.
    /// (RLY-006)
    Pong {
        /// Echoed timestamp from the Ping message.
        timestamp: u64,
    },
    /// Error message from the relay server.
    Error {
        /// Error code for programmatic handling.
        code: u32,
        /// Human-readable error message.
        message: String,
    },
    /// Request NAT traversal assistance.
    /// (RLY-007)
    HolePunchRequest {
        /// The requesting peer's ID.
        peer_id: PeerId,
        /// The target peer to attempt direct connection with.
        target_peer_id: PeerId,
        /// The requesting peer's observed external IP:port.
        external_addr: SocketAddr,
    },
    /// NAT traversal coordination from relay.
    /// (RLY-007)
    HolePunchCoordinate {
        /// The peer ID of the other side of the hole punch.
        peer_id: PeerId,
        /// The other peer's observed external IP:port to connect to.
        external_addr: SocketAddr,
    },
    /// NAT traversal result.
    /// (RLY-007)
    HolePunchResult {
        /// The peer reporting the result.
        peer_id: PeerId,
        /// Whether the direct connection was successfully established.
        success: bool,
    },
}
```

### Connection and Registration Flow

```rust
use tokio_tungstenite::connect_async;
use tokio::time::timeout;

impl RelayClient {
    /// Connect to the relay server and register this peer.
    ///
    /// # Errors
    /// Returns `GossipError::RelayError` if:
    /// - WebSocket connection fails or times out
    /// - Server responds with `RegisterAck { success: false, .. }`
    /// - No `RegisterAck` received within `connection_timeout_secs`
    pub async fn connect_and_register(
        config: &RelayConfig,
        peer_id: PeerId,
        network_id: Bytes32,
        protocol_version: u32,
    ) -> Result<Self, GossipError> {
        // 1. Establish WebSocket connection with timeout
        let (ws_stream, _) = timeout(
            Duration::from_secs(config.connection_timeout_secs),
            connect_async(&config.endpoint),
        )
        .await
        .map_err(|_| GossipError::RelayError("connection timeout".into()))?
        .map_err(|e| GossipError::RelayError(format!("websocket error: {e}")))?;

        let (mut write, mut read) = ws_stream.split();

        // 2. Send Register message
        let register = RelayMessage::Register {
            peer_id,
            network_id,
            protocol_version,
        };
        let json = serde_json::to_string(&register)
            .map_err(|e| GossipError::RelayError(format!("serialize error: {e}")))?;
        write.send(TungsteniteMessage::Text(json)).await
            .map_err(|e| GossipError::RelayError(format!("send error: {e}")))?;

        // 3. Await RegisterAck with timeout
        let ack_msg = timeout(
            Duration::from_secs(config.connection_timeout_secs),
            read.next(),
        )
        .await
        .map_err(|_| GossipError::RelayError("register ack timeout".into()))?
        .ok_or_else(|| GossipError::RelayError("connection closed before ack".into()))?
        .map_err(|e| GossipError::RelayError(format!("read error: {e}")))?;

        let ack: RelayMessage = serde_json::from_str(
            ack_msg.to_text()
                .map_err(|e| GossipError::RelayError(format!("non-text message: {e}")))?
        ).map_err(|e| GossipError::RelayError(format!("deserialize error: {e}")))?;

        // 4. Validate RegisterAck
        match ack {
            RelayMessage::RegisterAck { success: true, connected_peers, .. } => {
                tracing::info!(
                    relay = %config.endpoint,
                    connected_peers,
                    "registered with relay server"
                );
                Ok(Self { write, read, config: config.clone(), peer_id, network_id })
            }
            RelayMessage::RegisterAck { success: false, message, .. } => {
                Err(GossipError::RelayError(format!("registration rejected: {message}")))
            }
            other => {
                Err(GossipError::RelayError(
                    format!("unexpected message during registration: {other:?}")
                ))
            }
        }
    }
}
```

## Acceptance Criteria

- [ ] Relay client connects to the configured WebSocket endpoint
- [ ] `Register { peer_id, network_id, protocol_version }` is sent immediately after connection
- [ ] Client waits for `RegisterAck` within `connection_timeout_secs`
- [ ] On `RegisterAck { success: true }`, the client is considered connected and the `connected_peers` count is logged
- [ ] On `RegisterAck { success: false }`, `GossipError::RelayError` is returned with the server's message
- [ ] WebSocket connection timeout returns `GossipError::RelayError`
- [ ] Missing or malformed `RegisterAck` returns `GossipError::RelayError`
- [ ] All relay messages are serialized as JSON (not Chia binary protocol)
- [ ] `RelayConfig::enabled == false` prevents connection attempt

## Implementation Notes

- Primary files: `src/relay/relay_client.rs`, `src/relay/relay_types.rs`
- Dependencies: `tokio-tungstenite`, `serde_json`, `tokio::time::timeout`
- Edge cases:
  - Relay server may close the WebSocket before sending `RegisterAck` (EOF during read)
  - Server may send a non-JSON binary frame; the client must handle this gracefully
  - `protocol_version` mismatch may cause the server to reject registration
  - DNS resolution failure for the relay endpoint should produce a clear error
