# STR-004: Feature Flags

> **Authoritative requirement:** [NORMATIVE.md](../NORMATIVE.md)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **Spec reference:** [SPEC.md - Section 10.3](../../../resources/SPEC.md)

## Summary

Feature flags must match the SPEC Section 10.3 definitions exactly. Each feature gate controls conditional compilation of specific crate functionality: TLS backend selection (`native-tls`, `rustls`), relay support (`relay`), ERLAY transaction relay (`erlay`), and compact block relay (`compact-blocks`). The default feature set must enable the standard configuration for production nodes.

## Specification

### Feature Definitions

```toml
[features]
default = ["native-tls", "relay", "erlay", "compact-blocks", "dandelion"]
native-tls = ["chia-sdk-client/native-tls"]
rustls = ["chia-sdk-client/rustls"]
relay = []
erlay = ["minisketch-rs"]
compact-blocks = ["siphasher"]
dandelion = []
tor = ["arti-client", "tokio-socks"]
```

### Feature Behavior

| Feature | Dependency | Conditional Compilation Effect |
|---------|-----------|-------------------------------|
| `native-tls` | Forwards to `chia-sdk-client/native-tls` | Enables native TLS backend via `create_native_tls_connector()` |
| `rustls` | Forwards to `chia-sdk-client/rustls` | Enables rustls TLS backend via `create_rustls_connector()` |
| `relay` | None | Enables `relay/` module compilation and relay-related code paths |
| `erlay` | `minisketch-rs` | Enables `gossip/erlay.rs` and ERLAY reconciliation code paths |
| `compact-blocks` | `siphasher` | Enables `gossip/compact_block.rs` and compact block code paths |
| `dandelion` | None | Enables `privacy/dandelion.rs` and Dandelion++ stem/fluff code paths |
| `tor` | `arti-client`, `tokio-socks` | Enables `privacy/tor.rs` and Tor/SOCKS5 proxy transport (opt-in) |

### Conditional Compilation

Modules gated by feature flags MUST use `#[cfg(feature = "...")]`:

```rust
// In src/relay/mod.rs or src/lib.rs
#[cfg(feature = "relay")]
pub mod relay;

// In src/gossip/mod.rs
#[cfg(feature = "erlay")]
pub mod erlay;

#[cfg(feature = "compact-blocks")]
pub mod compact_block;

// In src/privacy/mod.rs or src/lib.rs
#[cfg(feature = "dandelion")]
pub mod dandelion;

#[cfg(feature = "tor")]
pub mod tor;
```

### TLS Backend Selection

`native-tls` and `rustls` are mutually exclusive. The crate MUST compile with either one but SHOULD NOT enable both simultaneously:

```rust
#[cfg(feature = "native-tls")]
fn create_tls_connector(cert: &[u8], key: &[u8]) -> Result<...> {
    chia_sdk_client::create_native_tls_connector(cert, key)
}

#[cfg(feature = "rustls")]
fn create_tls_connector(cert: &[u8], key: &[u8]) -> Result<...> {
    chia_sdk_client::create_rustls_connector(cert, key)
}
```

## Acceptance Criteria

- [ ] Feature `native-tls` is defined and forwards to `chia-sdk-client/native-tls`
- [ ] Feature `rustls` is defined and forwards to `chia-sdk-client/rustls`
- [ ] Feature `relay` is defined with no additional dependencies
- [ ] Feature `erlay` is defined and depends on `minisketch-rs`
- [ ] Feature `compact-blocks` is defined and depends on `siphasher`
- [ ] Feature `dandelion` is defined with no additional dependencies
- [ ] Feature `tor` is defined and depends on `arti-client` and `tokio-socks`
- [ ] Default features are `["native-tls", "relay", "erlay", "compact-blocks", "dandelion"]`
- [ ] `cargo check --no-default-features --features native-tls` compiles without relay, erlay, compact-blocks, dandelion, or tor code
- [ ] `cargo check --no-default-features --features rustls` compiles with rustls TLS backend
- [ ] `cargo check --features relay` compiles with relay module
- [ ] `cargo check --features erlay` compiles with erlay module
- [ ] `cargo check --features compact-blocks` compiles with compact block module
- [ ] Relay module code is not compiled when `relay` feature is disabled
- [ ] ERLAY code is not compiled when `erlay` feature is disabled
- [ ] Compact block code is not compiled when `compact-blocks` feature is disabled

## Implementation Notes

- Primary file: `Cargo.toml` (feature definitions), `src/lib.rs` and various `mod.rs` files (conditional compilation)
- Dependencies: `siphasher` and `minisketch-rs` are optional dependencies
- Edge cases:
  - Enabling both `native-tls` and `rustls` simultaneously may cause compile errors due to duplicate TLS connector definitions
  - Code that references relay types outside the relay module must also be feature-gated
  - Test modules that test feature-gated code must use appropriate `#[cfg(feature = "...")]` attributes
