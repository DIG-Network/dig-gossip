## Summary

The crate's module hierarchy (directory and file layout under `src/`) must match the structure defined in SPEC Section 10.1. This ensures consistent code organization, correct module visibility, and predictable file locations for all components of the gossip layer.

Each module directory MUST contain a `mod.rs` that declares and re-exports its submodules.

### Module Responsibilities

| Module | Responsibility |
|--------|---------------|
| `types/` | Data types shared across the crate (peer, config, stats, reputation, DIG messages) |
| `constants.rs` | DIG-specific constants and ported Chia Python constants |
| `error.rs` | Unified error type wrapping ClientError with gossip-specific variants |
| `service/` | GossipService construction/lifecycle and GossipHandle for runtime interaction |
| `connection/` | Inbound TCP/TLS listener (outbound uses chia-sdk-client directly) |
| `discovery/` | Address manager, node discovery loop, introducer client, vetted peers |
| `relay/` | Relay WebSocket client, relay service lifecycle, relay-specific types |
| `gossip/` | Plumtree, compact blocks, ERLAY, priority lanes, backpressure, dedup |
| `util/` | IP address helpers, AS lookup, latency tracking |

## Implementation Notes

- Primary file: All files under `src/`
- Dependencies: None (this is structural)
- Edge cases:
  - `relay/` module content should be conditionally compiled with `#[cfg(feature = "relay")]`
  - `gossip/erlay.rs` should be conditionally compiled with `#[cfg(feature = "erlay")]`
  - `gossip/compact_block.rs` should be conditionally compiled with `#[cfg(feature = "compact-blocks")]`
  - Each `mod.rs` must use `pub mod` for submodules that need to be accessible from other modules
