# STR-003: Public Re-exports in lib.rs

> **Authoritative requirement:** [NORMATIVE.md](../NORMATIVE.md)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **Spec reference:** [SPEC.md - Section 10.2](../../../resources/SPEC.md)

## Summary

The crate root (`lib.rs`) must re-export all Chia crate types used by the gossip layer and all DIG-specific types, so that consumers can access the full public API from the crate root without reaching into internal modules. This follows the SPEC Section 10.2 re-export list exactly.

## Specification

### Chia Crate Re-exports

`lib.rs` MUST contain the following re-exports from Chia crates:

```rust
// =========================================================================
// Re-exports from Chia crates (NOT reimplemented)
// =========================================================================
pub use chia_protocol::{
    Bytes32, Handshake, Message, NodeType, ProtocolMessageTypes,
    NewPeak, NewTransaction, RequestTransaction, RespondTransaction,
    RequestBlock, RespondBlock, RejectBlock,
    RequestBlocks, RespondBlocks, RejectBlocks,
    NewUnfinishedBlock, RequestUnfinishedBlock, RespondUnfinishedBlock,
    RequestMempoolTransactions,
    RequestPeers, RespondPeers,
    SpendBundle, FullBlock, TimestampedPeerInfo,
    ChiaProtocolMessage,
};
// Introducer opcodes 63/64 — wire structs live in-tree (`chia-protocol` 0.26 only exposes enum variants).
pub use discovery::introducer_wire::{RequestPeersIntroducer, RespondPeersIntroducer};
pub use chia_sdk_client::{
    Peer, PeerOptions, Client, ClientState, Network,
    RateLimiter, RateLimits, RateLimit, V2_RATE_LIMITS,
    ClientError, load_ssl_cert,
};
pub use chia_ssl::ChiaCertificate;
pub use chia_traits::Streamable;
```

### DIG-specific Type Re-exports

`lib.rs` MUST also re-export the following DIG-specific types:

```rust
// =========================================================================
// DIG-specific types (implemented in this crate)
// =========================================================================
pub use types::peer::{PeerId, PeerInfo, PeerConnection};
pub use types::config::{GossipConfig, IntroducerConfig, RelayConfig};
pub use types::stats::{GossipStats, RelayStats};
pub use types::reputation::{PeerReputation, PenaltyReason};
pub use types::dig_messages::DigMessageType;

pub use service::gossip_service::GossipService;
pub use service::gossip_handle::GossipHandle;

pub use discovery::address_manager::AddressManager;
pub use discovery::introducer_client::IntroducerClient;
pub use discovery::introducer_peers::{IntroducerPeers, VettedPeer};

pub use relay::relay_types::{RelayPeerInfo, RelayMessage};

#[cfg(feature = "compact-blocks")]
pub use gossip::compact_block::{CompactBlock, ShortTxId, PrefilledTransaction, RequestBlockTransactions, RespondBlockTransactions};

#[cfg(feature = "erlay")]
pub use gossip::erlay::{ErlayState, ReconciliationSketch};

#[cfg(feature = "dandelion")]
pub use privacy::dandelion::StemTransaction;

pub use error::GossipError;
pub use constants::*;
```

### Consumer Usage

A downstream crate MUST be able to use all types from the crate root:

```rust
use dig_gossip::{
    // Chia types
    Bytes32, Handshake, Message, NodeType, ProtocolMessageTypes,
    Peer, RateLimiter, V2_RATE_LIMITS, ChiaCertificate, Streamable,
    // DIG types
    GossipService, GossipHandle, GossipConfig, GossipError,
    PeerId, PeerInfo, PeerConnection,
    PeerReputation, PenaltyReason,
    DigMessageType, AddressManager, VettedPeer,
};
```

## Acceptance Criteria

- [ ] `lib.rs` re-exports `Bytes32`, `Handshake`, `Message`, `NodeType`, `ProtocolMessageTypes` from `chia-protocol`
- [ ] `lib.rs` re-exports all full node protocol messages (`NewPeak`, `NewTransaction`, `RequestTransaction`, `RespondTransaction`, `RequestBlock`, `RespondBlock`, `RejectBlock`, `RequestBlocks`, `RespondBlocks`, `RejectBlocks`, `NewUnfinishedBlock`, `RequestUnfinishedBlock`, `RespondUnfinishedBlock`, `RequestMempoolTransactions`)
- [ ] `lib.rs` re-exports `RequestPeers`, `RespondPeers` from `chia-protocol`
- [ ] `lib.rs` re-exports `RequestPeersIntroducer`, `RespondPeersIntroducer` from `discovery::introducer_wire` (DSC-004 wire bodies; enum variants remain on `ProtocolMessageTypes`)
- [ ] `lib.rs` re-exports `SpendBundle`, `FullBlock`, `TimestampedPeerInfo`, `ChiaProtocolMessage`
- [ ] `lib.rs` re-exports `Peer`, `PeerOptions`, `Client`, `ClientState`, `Network` from `chia-sdk-client`
- [ ] `lib.rs` re-exports `RateLimiter`, `RateLimits`, `RateLimit`, `V2_RATE_LIMITS` from `chia-sdk-client`
- [ ] `lib.rs` re-exports `ClientError`, `load_ssl_cert` from `chia-sdk-client`
- [ ] `lib.rs` re-exports `ChiaCertificate` from `chia-ssl`
- [ ] `lib.rs` re-exports `Streamable` from `chia-traits`
- [ ] `lib.rs` re-exports `PeerId`, `PeerInfo`, `PeerConnection` from `types::peer`
- [ ] `lib.rs` re-exports `GossipConfig`, `IntroducerConfig`, `RelayConfig` from `types::config`
- [ ] `lib.rs` re-exports `GossipStats`, `RelayStats` from `types::stats`
- [ ] `lib.rs` re-exports `PeerReputation`, `PenaltyReason` from `types::reputation`
- [ ] `lib.rs` re-exports `DigMessageType` from `types::dig_messages`
- [ ] `lib.rs` re-exports `GossipService`, `GossipHandle` from `service`
- [ ] `lib.rs` re-exports `AddressManager`, `IntroducerClient`, `IntroducerPeers`, `VettedPeer` from `discovery`
- [ ] `lib.rs` re-exports `RelayPeerInfo`, `RelayMessage` from `relay`
- [ ] `lib.rs` re-exports `GossipError` from `error`
- [ ] `lib.rs` glob re-exports all constants via `pub use constants::*`
- [ ] A downstream crate can `use dig_gossip::Bytes32` without reaching into submodules

## Implementation Notes

- Primary file: `src/lib.rs`
- Dependencies: All internal modules must be declared before re-exports
- Edge cases:
  - Re-exports from `relay` module should be gated with `#[cfg(feature = "relay")]`
  - The `constants::*` glob export must not conflict with re-exported names from Chia crates
  - If `chia-sdk-client` changes its public API between versions, re-exports must be updated accordingly
