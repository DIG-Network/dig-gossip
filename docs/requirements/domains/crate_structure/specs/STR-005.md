# STR-005: Test Infrastructure

> **Authoritative requirement:** [NORMATIVE.md](../NORMATIVE.md)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)

## Summary

The crate must include test infrastructure that provides helper utilities for creating mock peers, temporary directories, and test `GossipConfig` instances. This shared test infrastructure enables consistent and ergonomic integration and unit testing across all crate modules without duplicating setup code in each test file.

## Specification

### Test Helper Module

A test helper module MUST be provided (e.g., `tests/common/mod.rs` or `src/test_utils.rs` behind `#[cfg(test)]`) that provides:

#### 1. Mock Peer Creation

```rust

/// Generate a random PeerId (Bytes32).
pub fn random_peer_id() -> PeerId {
    let mut bytes = [0u8; 32];
    rand::thread_rng().fill(&mut bytes);
    Bytes32::from(bytes)
}
```

#### 4. TLS Certificate Generation for Tests

```rust
/// Generate test TLS certificates using chia-ssl.
/// Writes cert and key files to the given directory.
pub fn generate_test_certs(dir: &Path) -> (String, String) {
    let cert = ChiaCertificate::generate().expect("Failed to generate cert");
    let cert_path = dir.join("test.crt");
    let key_path = dir.join("test.key");
    std::fs::write(&cert_path, &cert.cert_pem).unwrap();
    std::fs::write(&key_path, &cert.key_pem).unwrap();
    (
        cert_path.to_string_lossy().to_string(),
        key_path.to_string_lossy().to_string(),
    )
}
```

## Acceptance Criteria

- [ ] Test helper module exists and is importable from integration tests
- [ ] `random_peer_id()` generates unique `PeerId` values on each call
- [ ] `mock_peer_connection()` creates a valid `PeerConnection` with configurable `is_outbound`
- [ ] `test_temp_dir()` creates a temporary directory that is cleaned up on drop
- [ ] `test_gossip_config()` creates a valid `GossipConfig` with localhost and OS-assigned port
- [ ] `generate_test_certs()` produces valid TLS certificate and key files via `chia-ssl`
- [ ] `connected_test_pair()` creates two connected gossip instances for integration testing
- [ ] Test helpers do not leak resources (temp dirs cleaned up, connections closed)
- [ ] Test helpers work on all supported platforms (Linux, macOS, Windows)

## Implementation Notes

- Primary file: `tests/common/mod.rs` or `src/test_utils.rs`
- Dependencies: `tempfile` (dev-dependency), `rand`, `chia-ssl`
- Edge cases:
  - Port 0 binding requires checking the actual bound address after listener creation
  - TLS certificate generation may be slow; consider caching per test session
  - Mock peers may need to simulate a minimal WebSocket endpoint for integration tests
  - Windows path handling for temp directories differs from Unix
