# INT-013: Clean Public API Surface

> **Authoritative requirement:** [NORMATIVE.md](../NORMATIVE.md)
> **Spec reference:** [SPEC.md](../../../../resources/SPEC.md) — Section 10.2

## Summary

The crate's public API (`pub use` in `lib.rs`) MUST expose only types that external callers need. Internal types (ServiceState, test helpers, wiring utilities) MUST be `pub(crate)` or `#[doc(hidden)]`. Test-only methods on GossipHandle MUST be `#[doc(hidden)]` and prefixed with `__` (already done) or gated behind `#[cfg(test)]`.

## Acceptance Criteria

- [ ] `lib.rs` re-exports organized into clear sections: Core, Config, Types, Protocol, Discovery, Relay, Gossip, Privacy
- [ ] `ServiceState` NOT in public API (move to `pub(crate)`)
- [ ] `peer_id_for_addr`, `apply_inbound_rate_limit_violation` NOT in public API
- [ ] Test helper functions (`__*_for_tests`) marked `#[doc(hidden)]`
- [ ] Internal module types (LiveSlot, StubPeer, PeerSlot) NOT in public API
- [ ] `cargo doc` produces clean documentation without internal noise
