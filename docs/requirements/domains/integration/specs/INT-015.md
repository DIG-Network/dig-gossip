# INT-015: End-to-End Lifecycle Integration Test

> **Authoritative requirement:** [NORMATIVE.md](../NORMATIVE.md)
> **Spec reference:** [SPEC.md](../../../../resources/SPEC.md) — Section 3

## Summary

A single integration test MUST prove the complete crate lifecycle works end-to-end:
1. Create GossipConfig with defaults
2. GossipService::new(config) succeeds
3. service.start() returns GossipHandle
4. handle.health_check() returns Ok
5. handle.stats() returns valid GossipStats
6. handle.peer_count() returns 0 (no peers connected)
7. service.stop() succeeds
8. handle.health_check() returns ServiceNotStarted error

This proves the public interface works as a complete unit.

## Acceptance Criteria

- [ ] Full lifecycle test passes
- [ ] All GossipHandle methods callable between start() and stop()
- [ ] Methods return ServiceNotStarted after stop()
- [ ] No panics during lifecycle

## Test Plan

| Test | Type | Description | Expected Result |
|------|------|-------------|-----------------|
| test_full_lifecycle | Integration | Create → start → use → stop | All steps succeed |
| test_methods_after_stop | Integration | Call methods after stop | ServiceNotStarted error |
