# API-006: PeerReputation and PenaltyReason

> **Authoritative requirement:** [NORMATIVE.md](../NORMATIVE.md)

## Summary

PeerReputation extends `chia-sdk-client::ClientState`'s binary ban/trust model with numeric penalty tracking, RTT-based latency scoring, and AS-level grouping. PenaltyReason enumerates the reasons a peer can be penalized. Together they provide latency-aware peer selection and graduated penalty enforcement for the gossip layer.

## Specification

### PenaltyReason Enum

```rust
/// Reasons a peer can be penalized.
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub enum PenaltyReason {
    InvalidBlock,
    InvalidAttestation,
    MalformedMessage,
    Spam,
    ConnectionIssue,
    ProtocolViolation,
    RateLimitExceeded,
    ConsensusError,
}
```

### PeerReputation Struct

```rust
/// Reputation tracking for a peer.
#[derive(Debug, Clone, Default)]
pub struct PeerReputation {
    /// Cumulative penalty points (higher = worse).
    pub penalty_points: u32,
    /// Whether this peer is temporarily banned.
    pub is_banned: bool,
    /// Ban expiry timestamp (Unix seconds).
    pub ban_until: Option<u64>,
    /// Last penalty reason.
    pub last_penalty_reason: Option<PenaltyReason>,
    /// Rolling average RTT in milliseconds (from Ping/Pong).
    /// Used for latency-aware peer selection and Plumtree tree optimization.
    pub avg_rtt_ms: Option<u64>,
    /// Recent RTT measurements (circular buffer, last RTT_WINDOW_SIZE pings).
    pub rtt_history: VecDeque<u64>,
    /// Composite peer score: trust_score * (1 / avg_rtt_ms).
    /// Higher = better. Used for outbound peer selection preference.
    pub score: f64,
    /// AS number for this peer's IP (cached from BGP lookup).
    pub as_number: Option<u32>,
}
```

The `Default` implementation MUST produce:

| Field | Default |
|-------|---------|
| `penalty_points` | `0` |
| `is_banned` | `false` |
| `ban_until` | `None` |
| `last_penalty_reason` | `None` |
| `avg_rtt_ms` | `None` |
| `rtt_history` | Empty `VecDeque` |
| `score` | `0.0` |
| `as_number` | `None` |

### Penalty Mechanics

- Each `PenaltyReason` has an associated penalty point value (implementation-defined, but typically: `InvalidBlock` = 100, `Spam` = 20, `ConnectionIssue` = 10, etc.)
- When `penalty_points >= PENALTY_BAN_THRESHOLD` (100), the peer is auto-banned
- `ban_until` is set to `current_time + BAN_DURATION_SECS` (3600)
- Bans are time-limited; after `ban_until`, the peer may reconnect

### RTT Tracking

- `rtt_history` is a circular buffer of the last `RTT_WINDOW_SIZE` (10) measurements
- `avg_rtt_ms` is recomputed as the mean of `rtt_history` after each new measurement
- `score` is computed as `trust_factor * (1.0 / avg_rtt_ms)` where `trust_factor` accounts for penalty history

### AS Number Caching

- `as_number` is populated via BGP prefix table lookup on first connection
- Used for AS-level diversity enforcement (one outbound per AS)

## Implementation Notes

- Primary file: `src/types/reputation.rs`
- Dependencies: `std::collections::VecDeque`, constants `PENALTY_BAN_THRESHOLD`, `BAN_DURATION_SECS`, `RTT_WINDOW_SIZE`
- Edge cases:
  - `score` computation must handle `avg_rtt_ms = 0` (avoid division by zero)
  - `rtt_history` must evict oldest entry when at capacity before pushing new measurement
  - `penalty_points` overflow: use saturating addition to avoid wrapping
  - When a ban expires (`current_time > ban_until`), `is_banned` should be set to `false` on next check
  - `PenaltyReason` does not need `Hash` since it is not used as a map key
