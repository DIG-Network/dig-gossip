## Summary

GossipError is the unified error type for the gossip crate. It wraps `chia-sdk-client::ClientError` via the `From` trait and adds gossip-specific error variants covering peer management, connection limits, discovery, relay, and service lifecycle errors. It derives `Debug`, `Clone`, and uses `thiserror::Error` for display formatting.

## Specification

### Enum Definition

```rust
#[derive(Debug, Clone, thiserror::Error)]
pub enum GossipError {
    /// Wraps chia-sdk-client::ClientError for connection-level errors.
    #[error("client error: {0}")]
    ClientError(#[from] ClientError),

    #[error("peer not connected: {0}")]
    PeerNotConnected(PeerId),

    #[error("peer banned: {0}")]
    PeerBanned(PeerId),

    #[error("max connections reached ({0})")]
    MaxConnectionsReached(usize),

    #[error("duplicate connection to peer {0}")]
    DuplicateConnection(PeerId),

    #[error("self connection detected")]
    SelfConnection,

    #[error("request timeout")]
    RequestTimeout,

    #[error("introducer not configured")]
    IntroducerNotConfigured,

    #[error("introducer error: {0}")]
    IntroducerError(String),

    #[error("relay not configured")]
    RelayNotConfigured,

    #[error("relay error: {0}")]
    RelayError(String),

    #[error("service not started")]
    ServiceNotStarted,

    /// Peers-file snapshot invalid or unreadable (DSC-002).
    #[error("address manager store: {0}")]
    AddressManagerStore(String),

    #[error("channel closed")]
    ChannelClosed,

    #[error("I/O error: {0}")]
    IoError(String),

    #[error("sketch error: {0}")]
    SketchError(String),

    #[error("sketch decode failed")]
    SketchDecodeFailed,
}
```

### Variant Semantics

| Variant | When Raised |
|---------|-------------|
| `ClientError` | Any error from `chia-sdk-client` operations (connect, send, TLS) |
| `PeerNotConnected` | `send_to`, `request`, `disconnect`, `ban_peer`, `penalize_peer` on unknown peer |
| `PeerBanned` | Attempting to connect to or send to a banned peer |
| `MaxConnectionsReached` | `connect_to` when `max_connections` limit hit |
| `DuplicateConnection` | `connect_to` for an already-connected peer |
| `SelfConnection` | `connect_to` resolving to own `peer_id` |
| `RequestTimeout` | `request` exceeding configured timeout |
| `IntroducerNotConfigured` | `discover_from_introducer` or `register_with_introducer` with no introducer config |
| `IntroducerError` | Introducer communication failure |
| `RelayNotConfigured` | Relay operations when relay not configured |
| `RelayError` | Relay communication failure |
| `AddressManagerStore` | Address manager peers-file load/save failed (corrupt snapshot, version mismatch, invariant violation) |
| `ServiceNotStarted` | Any handle method called after `stop()` or before `start()` |
| `ChannelClosed` | Internal channel (mpsc) closed unexpectedly |
| `IoError` | File I/O errors (TLS cert loading, address manager persistence) |
| `SketchError` | Minisketch encoding/decoding errors during ERLAY set reconciliation |
| `SketchDecodeFailed` | Sketch decoding produced no result (symmetric difference exceeds sketch capacity) |

### From Implementations

```rust
// Automatic via #[from] on ClientError variant
impl From<ClientError> for GossipError {
    fn from(err: ClientError) -> Self {
        GossipError::ClientError(err)
    }
}
```

## Implementation Notes

- Primary file: `src/error.rs`
- Dependencies: `chia-sdk-client::ClientError`, `PeerId` (`chia-protocol::Bytes32`), `thiserror`
- Edge cases:
  - `ClientError` must implement `Clone` for `GossipError` to derive `Clone`; if it does not, the `ClientError` variant may need to store a `String` representation instead
  - `PeerId` display in error messages uses `Bytes32`'s `Display` impl (hex string)
  - `IoError` wraps `String` rather than `std::io::Error` because `std::io::Error` does not implement `Clone`
