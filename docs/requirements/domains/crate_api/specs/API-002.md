## Summary

GossipHandle is the primary runtime interface to the gossip layer. It is returned by `GossipService::start()`, is cheaply cloneable via an inner `Arc`, and exposes all messaging, peer management, discovery, and statistics operations. All methods are async and route through internal channels to the service's background tasks.

### Message Sending Methods

```rust
impl GossipHandle {
    /// Broadcast a chia-protocol::Message to connected peers via gossip fanout.
    /// Returns the number of peers the message was sent to.
    /// Optionally excludes one peer (typically the sender of the original message).
    pub async fn broadcast(
        &self,
        message: Message,
        exclude: Option<PeerId>,
    ) -> Result<usize, GossipError>;

    /// Broadcast a typed Streamable + ChiaProtocolMessage.
    /// Serializes to Message internally using chia-traits::Streamable.
    pub async fn broadcast_typed<T: Streamable + ChiaProtocolMessage>(
        &self,
        body: T,
        exclude: Option<PeerId>,
    ) -> Result<usize, GossipError>;

    /// Send a message to a specific peer (via their chia-sdk-client::Peer).
    pub async fn send_to<T: Streamable + ChiaProtocolMessage>(
        &self,
        peer_id: PeerId,
        body: T,
    ) -> Result<(), GossipError>;

    /// Send a request and await a typed response (uses Peer::request_infallible).
    pub async fn request<T, B>(
        &self,
        peer_id: PeerId,
        body: B,
    ) -> Result<T, GossipError>
    where
        T: Streamable + ChiaProtocolMessage,
        B: Streamable + ChiaProtocolMessage;
}
```

### Peer Management Methods

```rust
impl GossipHandle {
    /// Get all connected peers with their extended state.
    pub async fn connected_peers(&self) -> Vec<PeerConnection>;

    /// Get number of connected peers.
    pub async fn peer_count(&self) -> usize;

    /// Get connections filtered by node type and direction.
    pub async fn get_connections(
        &self,
        node_type: Option<NodeType>,
        outbound_only: bool,
    ) -> Vec<PeerConnection>;

    /// Connect to a peer (uses chia-sdk-client::connect_peer internally).
    pub async fn connect_to(&self, addr: SocketAddr) -> Result<PeerId, GossipError>;

    /// Disconnect a peer.
    pub async fn disconnect(&self, peer_id: &PeerId) -> Result<(), GossipError>;

    /// Ban a peer (delegates to ClientState::ban + PeerReputation).
    pub async fn ban_peer(
        &self,
        peer_id: &PeerId,
        reason: PenaltyReason,
    ) -> Result<(), GossipError>;

    /// Apply a reputation penalty.
    pub async fn penalize_peer(
        &self,
        peer_id: &PeerId,
        reason: PenaltyReason,
    ) -> Result<(), GossipError>;
}
```

### Usage Example

```rust
use dig_gossip::{GossipHandle, Message, PeerId, NodeType};

async fn use_handle(handle: GossipHandle) -> Result<(), dig_gossip::GossipError> {
    // Broadcast a message
    let msg = Message { /* ... */ };
    let peers_sent = handle.broadcast(msg, None).await?;

    // Send to a specific peer
    handle.send_to(some_peer_id, my_request).await?;

    // Request/response
    let response: RespondBlock = handle.request(some_peer_id, block_request).await?;

    // Peer management
    let count = handle.peer_count().await;
    let outbound = handle.get_connections(Some(NodeType::FullNode), true).await;
    let new_peer = handle.connect_to("1.2.3.4:9444".parse().unwrap()).await?;
    handle.disconnect(&stale_peer_id).await?;

    // Discovery
    let peers = handle.discover_from_introducer().await?;
    let _reg_ack = handle.register_with_introducer().await?;

    // Stats
    let stats = handle.stats().await;
    println!("Connected: {}", stats.connected_peers);

    Ok(())
}
```

## Acceptance Criteria

- [ ] `GossipHandle` implements `Clone`
- [ ] `GossipHandle::broadcast` sends a `Message` to connected peers and returns the count
- [ ] `GossipHandle::broadcast` respects the `exclude` parameter
- [ ] `GossipHandle::broadcast_typed` serializes a `Streamable + ChiaProtocolMessage` to `Message`
- [ ] `GossipHandle::send_to` sends to a specific peer; returns `PeerNotConnected` if unknown
- [ ] `GossipHandle::request` sends a request and awaits a typed response
- [ ] `GossipHandle::request` returns `RequestTimeout` on timeout
- [ ] `GossipHandle::inbound_receiver` returns a receiver for `(PeerId, Message)` tuples
- [ ] `GossipHandle::connected_peers` returns all `PeerConnection`s
- [ ] `GossipHandle::peer_count` returns the current connection count
- [ ] `GossipHandle::get_connections` filters by `NodeType` and outbound direction
- [ ] `GossipHandle::connect_to` establishes a connection via `connect_peer()` and returns the `PeerId`
- [ ] `GossipHandle::connect_to` returns `MaxConnectionsReached` when at limit
- [ ] `GossipHandle::connect_to` returns `DuplicateConnection` for already-connected peers
- [ ] `GossipHandle::connect_to` returns `SelfConnection` when connecting to own address
- [ ] `GossipHandle::disconnect` closes the peer connection
- [ ] `GossipHandle::ban_peer` bans the peer via `ClientState::ban` and updates `PeerReputation`
- [ ] `GossipHandle::penalize_peer` increments penalty points; auto-bans at threshold
- [ ] `GossipHandle::discover_from_introducer` returns `IntroducerNotConfigured` when no introducer
- [ ] `GossipHandle::register_with_introducer` returns `IntroducerNotConfigured` when no introducer
- [ ] `GossipHandle::request_peers_from` sends `RequestPeers` and returns `RespondPeers`
- [ ] `GossipHandle::stats` returns current `GossipStats`
- [ ] `GossipHandle::relay_stats` returns `None` when relay is not configured

## Implementation Notes

- Primary file: `src/service/gossip_handle.rs`
- Dependencies: `chia-sdk-client::Peer`, `chia-sdk-client::connect_peer`, `chia-protocol::Message`, `chia-traits::Streamable`, `chia-protocol::ChiaProtocolMessage`, `tokio::sync::mpsc`, `PeerConnection`, `GossipError`, `GossipStats`, `RelayStats`
- Edge cases:
  - `broadcast` with zero connected peers should return `Ok(0)`, not an error
  - `send_to` and `request` must check peer is still connected before sending
  - `request` should use a configurable timeout (from `GossipConfig`) and return `RequestTimeout` on expiry
  - `ban_peer` should also disconnect the peer immediately
  - `penalize_peer` should check if accumulated points exceed `PENALTY_BAN_THRESHOLD` and auto-ban
  - All methods should return `GossipError::ServiceNotStarted` if the service has been stopped
