# API-007: PeerId Type Alias and PeerInfo

> **Authoritative requirement:** [NORMATIVE.md](../NORMATIVE.md)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **Spec reference:** [SPEC.md - Sections 2.2, 2.7](../../../resources/SPEC.md)

## Summary

PeerId is a type alias for `chia-protocol::Bytes32`, representing a unique peer identifier derived from SHA256 of the TLS public key. PeerInfo is a resolved peer address with `get_group()` and `get_key()` methods required by the address manager for bucket computation and network-group diversity enforcement. PeerInfo is a Rust port of Chia's `peer_info.py:20-57`.

## Specification

### PeerId Type Alias

```rust
/// A unique identifier for a peer, derived from SHA256(TLS public key).
/// Uses `Bytes32` from `chia-protocol`.
pub type PeerId = Bytes32;
```

### PeerInfo Struct

```rust
/// Resolved peer address with bucket computation methods.
/// Provides get_group() and get_key() for address manager bucketing.
/// Chia: peer_info.py:20-57.
#[derive(Debug, Clone, PartialEq, Eq, Hash)]
pub struct PeerInfo {
    pub host: String,
    pub port: u16,
}
```

### PeerInfo Methods

```rust
impl PeerInfo {
    /// Get the network group (/16 for IPv4, /32 for IPv6).
    /// Used for one-connection-per-group policy.
    /// Chia: peer_info.py:51-56.
    pub fn get_group(&self) -> Vec<u8>;

    /// Get a unique key for bucket computation.
    /// Chia: peer_info.py:43-49.
    pub fn get_key(&self) -> Vec<u8>;
}
```

### get_group() Behavior

For IPv4 addresses:
- Returns the first 2 octets (the /16 prefix) as bytes
- Example: `192.168.1.5` returns `[192, 168]`

For IPv6 addresses:
- Returns the first 4 bytes (the /32 prefix)
- Example: `2001:0db8:85a3::1` returns `[0x20, 0x01, 0x0d, 0xb8]`

This matches Chia's `peer_info.py:51-56` grouping logic used for eclipse attack resistance (one outbound per group).

### get_key() Behavior

Returns a unique byte representation suitable for hashing in bucket computation:
- For IPv4: the 4-byte address concatenated with the 2-byte port (big-endian)
- For IPv6: the 16-byte address concatenated with the 2-byte port (big-endian)

This matches Chia's `peer_info.py:43-49` key derivation.

## Acceptance Criteria

- [ ] `PeerId` is a type alias for `chia-protocol::Bytes32`
- [ ] `PeerId` can be used interchangeably with `Bytes32`
- [ ] `PeerInfo` derives `Debug, Clone, PartialEq, Eq, Hash`
- [ ] `PeerInfo` has field `host: String`
- [ ] `PeerInfo` has field `port: u16`
- [ ] `PeerInfo::get_group()` returns `/16` prefix bytes for IPv4
- [ ] `PeerInfo::get_group()` returns `/32` prefix bytes for IPv6
- [ ] `PeerInfo::get_key()` returns address bytes concatenated with port bytes
- [ ] `PeerInfo::get_key()` produces unique keys for different host:port combinations
- [ ] `PeerInfo` can be used as a `HashMap` key (via `Eq + Hash`)

## Implementation Notes

- Primary file: `src/types/peer.rs`
- Dependencies: `chia-protocol::Bytes32`, `std::net::IpAddr`
- Edge cases:
  - `host` may be a hostname rather than IP address; `get_group()` and `get_key()` require parsing to `IpAddr` first
  - If `host` cannot be parsed as an IP address, `get_group()` should return a hash of the hostname string
  - IPv4-mapped IPv6 addresses (e.g., `::ffff:192.168.1.1`) should be treated as IPv4 for grouping
  - Port `0` is valid but unusual; `get_key()` should still work
  - `get_group()` and `get_key()` must be deterministic (same input always produces same output)
