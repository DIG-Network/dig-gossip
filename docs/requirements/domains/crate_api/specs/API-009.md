# API-009: DigMessageType Enum

## Summary

DigMessageType defines DIG L2-specific protocol message types that extend Chia's `ProtocolMessageTypes`. These use message type IDs in the 200+ range to avoid collision with Chia's existing message type discriminants. The enum covers attestation gossip, checkpoint protocol messages, status exchange, and validator directory announcements.

## Specification

### Enum Definition

```rust
/// DIG-specific protocol message type extensions.
/// These use message type IDs in the 200+ range to avoid collision
/// with Chia's ProtocolMessageTypes.
#[derive(Debug, Clone, Copy, PartialEq, Eq, Hash, Serialize, Deserialize)]
#[repr(u8)]
pub enum DigMessageType {
    /// Attestation gossip (validator attestation for a block).
    NewAttestation = 200,
    /// Checkpoint proposal from epoch proposer.
    NewCheckpointProposal = 201,
    /// BLS signature for checkpoint aggregation.
    NewCheckpointSignature = 202,
    /// Checkpoint signature request.
    RequestCheckpointSignatures = 203,
    /// Checkpoint signature response.
    RespondCheckpointSignatures = 204,
    /// Status request.
    RequestStatus = 205,
    /// Status response.
    RespondStatus = 206,
    /// Checkpoint submission (after BLS aggregation).
    NewCheckpointSubmission = 207,
    /// Validator directory announcement.
    ValidatorAnnounce = 208,
    /// Request missing transactions for compact block reconstruction (SPEC Section 8.2).
    RequestBlockTransactions = 209,
    /// Response with missing transactions for compact block reconstruction (SPEC Section 8.2).
    RespondBlockTransactions = 210,
    /// ERLAY reconciliation sketch exchange (SPEC Section 8.3).
    ReconciliationSketch = 211,
    /// ERLAY reconciliation response with missing tx_ids (SPEC Section 8.3).
    ReconciliationResponse = 212,
    /// Dandelion++ stem-phase transaction (SPEC Section 1.9.1).
    StemTransaction = 213,
    /// Plumtree lazy hash-only announcement (SPEC Section 8.1).
    PlumtreeLazyAnnounce = 214,
    /// Plumtree prune message -- demote sender to lazy (SPEC Section 8.1).
    PlumtreePrune = 215,
    /// Plumtree graft message -- promote sender to eager (SPEC Section 8.1).
    PlumtreeGraft = 216,
    /// Plumtree request full message by hash (SPEC Section 8.1).
    PlumtreeRequestByHash = 217,
}
```

### Collision Avoidance

Chia's `ProtocolMessageTypes` uses values 0-99 for the full node protocol. The 200+ range is reserved for DIG extensions, ensuring no overlap with existing or future Chia message types.

### Derive Requirements

The enum MUST derive:
- `Debug` - for logging and error messages
- `Clone`, `Copy` - for value semantics
- `PartialEq`, `Eq` - for comparison
- `Hash` - for use as HashMap/HashSet key
- `Serialize`, `Deserialize` - for wire serialization via serde

### repr(u8)

The `#[repr(u8)]` attribute ensures the discriminant fits in a single byte and has a well-defined numeric representation for wire encoding.

## Implementation Notes

- Primary file: `src/types/dig_messages.rs`
- Dependencies: `serde::{Serialize, Deserialize}`
- Edge cases:
  - Casting to `u8` via `as u8` must produce the correct discriminant value
  - Converting from `u8` to `DigMessageType` should be handled by a `TryFrom<u8>` implementation to reject invalid values
  - When constructing a `chia-protocol::Message`, the `msg_type` field must use the `DigMessageType` discriminant as a `u8`, since Chia's `ProtocolMessageTypes` enum won't recognize it
  - Future DIG message types should continue from 209+ to maintain the collision-free range
