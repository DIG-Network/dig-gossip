# API-008: GossipStats and RelayStats

## Summary

GossipStats and RelayStats are data structs that provide runtime observability into the gossip layer. GossipStats covers overall connection and message metrics. RelayStats covers relay-specific metrics. Both derive `Debug`, `Clone`, and `Default`, and are returned by `GossipHandle::stats()` and `GossipHandle::relay_stats()` respectively.

## Specification

### GossipStats Struct

```rust
#[derive(Debug, Clone, Default)]
pub struct GossipStats {
    /// Total connections ever established (cumulative).
    pub total_connections: usize,
    /// Currently connected peers.
    pub connected_peers: usize,
    /// Current inbound connection count.
    pub inbound_connections: usize,
    /// Current outbound connection count.
    pub outbound_connections: usize,
    /// Total messages sent (cumulative).
    pub messages_sent: u64,
    /// Total messages received (cumulative).
    pub messages_received: u64,
    /// Total bytes sent (cumulative).
    pub bytes_sent: u64,
    /// Total bytes received (cumulative).
    pub bytes_received: u64,
    /// Number of known addresses in the address manager.
    pub known_addresses: usize,
    /// Number of message hashes in the seen set.
    pub seen_messages: usize,
    /// Whether the relay connection is active.
    pub relay_connected: bool,
    /// Number of peers known via relay.
    pub relay_peer_count: usize,
}
```

### RelayStats Struct

```rust
#[derive(Debug, Clone, Default)]
pub struct RelayStats {
    /// Whether currently connected to the relay.
    pub connected: bool,
    /// Total messages sent via relay (cumulative).
    pub messages_sent: u64,
    /// Total messages received via relay (cumulative).
    pub messages_received: u64,
    /// Total bytes sent via relay (cumulative).
    pub bytes_sent: u64,
    /// Total bytes received via relay (cumulative).
    pub bytes_received: u64,
    /// Number of reconnect attempts since last successful connection.
    pub reconnect_attempts: u32,
    /// Timestamp of last successful connection (Unix seconds).
    pub last_connected_at: Option<u64>,
    /// Number of peers known via relay.
    pub relay_peer_count: usize,
    /// Current latency to relay server in milliseconds.
    pub latency_ms: Option<u64>,
}
```

### Default Values

All numeric fields default to `0` / `false` / `None` via the `Default` derive.

### Relationship to GossipHandle

- `GossipHandle::stats()` returns a snapshot `GossipStats` reflecting current state
- `GossipHandle::relay_stats()` returns `Some(RelayStats)` when relay is configured, `None` otherwise
- Stats are computed on-demand from shared atomic counters and connection state

## Implementation Notes

- Primary file: `src/types/stats.rs`
- Dependencies: None (pure data structs)
- Edge cases:
  - `connected_peers` should equal `inbound_connections + outbound_connections`
  - `total_connections` is cumulative and never decreases (even after disconnects)
  - Stats should be cheap to compute; avoid holding locks while assembling the struct
  - `relay_connected` in GossipStats and `connected` in RelayStats should be consistent
  - Counter overflow: `u64` counters should not overflow in practice but implementations may use `AtomicU64` for thread-safe incrementing
