# ERL-003: Minisketch Encode/Decode for tx_id Sets

> **Authoritative requirement:** [NORMATIVE.md](../NORMATIVE.md)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **Spec reference:** [SPEC.md - Section 8.3](../../../resources/SPEC.md)

## Summary

Set reconciliation between peers uses minisketch, a library for efficient set reconciliation based on BCH error-correcting codes. The `minisketch-rs` Rust bindings are used to encode local tx_id sets into compact sketches, exchange them with peers, and decode the symmetric difference. The sketch capacity is `ERLAY_SKETCH_CAPACITY` (20), meaning each reconciliation round can decode up to 20 differing elements between the two peers' sets.

## Specification

### Minisketch Integration

```rust
use minisketch_rs::Minisketch;
use chia_protocol::Bytes32;
use crate::constants::ERLAY_SKETCH_CAPACITY;

/// Element bit width for minisketch.
/// tx_ids are 32 bytes (256 bits), but we use a 64-bit truncation
/// for sketch elements. Collision probability is negligible at 2^-64.
const SKETCH_ELEMENT_BITS: usize = 64;

/// Encode a set of tx_ids into a minisketch.
pub fn encode_sketch(tx_ids: &HashSet<Bytes32>) -> Result<Vec<u8>, GossipError> {
    let mut sketch = Minisketch::try_new(
        SKETCH_ELEMENT_BITS as u32,
        0,  // implementation index (0 = default)
        ERLAY_SKETCH_CAPACITY as u32,
    ).map_err(|e| GossipError::SketchError(e.to_string()))?;

    for tx_id in tx_ids {
        // Truncate 32-byte tx_id to 64-bit sketch element
        let element = u64::from_le_bytes(tx_id.as_ref()[..8].try_into().unwrap());
        sketch.add(element);
    }

    Ok(sketch.serialize())
}

/// Decode the symmetric difference between local and remote sketches.
/// Returns the set of tx_id prefixes that differ between the two sets.
pub fn decode_difference(
    local_sketch: &[u8],
    remote_sketch: &[u8],
) -> Result<Vec<u64>, GossipError> {
    let mut local = Minisketch::try_new(
        SKETCH_ELEMENT_BITS as u32,
        0,
        ERLAY_SKETCH_CAPACITY as u32,
    ).map_err(|e| GossipError::SketchError(e.to_string()))?;
    local.deserialize(local_sketch);

    let mut remote = Minisketch::try_new(
        SKETCH_ELEMENT_BITS as u32,
        0,
        ERLAY_SKETCH_CAPACITY as u32,
    ).map_err(|e| GossipError::SketchError(e.to_string()))?;
    remote.deserialize(remote_sketch);

    // Merge (XOR) sketches to get symmetric difference sketch
    local.merge(&remote)
        .map_err(|e| GossipError::SketchError(e.to_string()))?;

    // Decode the difference
    let mut differences = vec![0u64; ERLAY_SKETCH_CAPACITY];
    let num_differences = local.decode(&mut differences)
        .map_err(|_| GossipError::SketchDecodeFailed)?;

    differences.truncate(num_differences);
    Ok(differences)
}
```

### Constants

```rust
/// Minisketch capacity (max decodable symmetric difference).
/// Default: 20 (handles up to 20 tx difference per reconciliation).
pub const ERLAY_SKETCH_CAPACITY: usize = 20;
```

### Wire Format

The serialized sketch is exchanged between peers as part of the reconciliation protocol:

```
ReconciliationSketch {
    sketch: Vec<u8>,        // Serialized minisketch
    sketch_capacity: u32,   // ERLAY_SKETCH_CAPACITY, so receiver can verify
}
```

### Invariants

- Sketch element bit width MUST be 64 bits (truncated from 256-bit tx_ids).
- Sketch capacity MUST be `ERLAY_SKETCH_CAPACITY` (20).
- If the actual symmetric difference exceeds the sketch capacity, decoding fails and the reconciliation round MUST fall back to requesting the full tx_id list.
- The `minisketch-rs` crate MUST be gated behind the `erlay` feature flag.

## Acceptance Criteria

- [ ] `encode_sketch` creates a minisketch from a set of `Bytes32` tx_ids
- [ ] `decode_difference` computes the symmetric difference between two serialized sketches
- [ ] Sketch element bit width is 64
- [ ] Sketch capacity is `ERLAY_SKETCH_CAPACITY` (20)
- [ ] Round-trip encode/decode recovers the correct symmetric difference for sets differing by up to 20 elements
- [ ] Decoding fails gracefully when symmetric difference exceeds capacity
- [ ] Serialized sketch size is deterministic for a given capacity
- [ ] `ERLAY_SKETCH_CAPACITY` constant is defined as `20` in `constants.rs`
- [ ] `minisketch-rs` is gated behind the `erlay` feature flag

## Implementation Notes

- Primary file: `src/gossip/erlay.rs`
- Constants file: `src/constants.rs`
- Dependencies: `minisketch-rs` (feature-gated under `erlay`), `chia-protocol` for `Bytes32`
- Edge cases:
  - Empty set: sketch is valid but encodes zero elements; difference decode returns empty
  - Identical sets on both sides: symmetric difference is empty; decode returns zero elements
  - Difference exactly at capacity (20): should decode successfully
  - Difference at 21: decode MUST fail; caller falls back to full set exchange
  - Hash truncation collisions (64-bit): probability ~2^-64 per pair; negligible but should be handled gracefully if a requested tx_id is not found
