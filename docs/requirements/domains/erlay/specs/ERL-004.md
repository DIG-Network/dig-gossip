# ERL-004: Periodic Set Reconciliation

> **Authoritative requirement:** [NORMATIVE.md](../NORMATIVE.md)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **Spec reference:** [SPEC.md - Section 8.3](../../../resources/SPEC.md)

## Summary

Set reconciliation runs on a periodic timer for each non-flood peer. Every `ERLAY_RECONCILIATION_INTERVAL_MS` (2000) milliseconds, the local node encodes its pending tx_ids into a minisketch, sends it to the peer, receives the peer's sketch in return, computes the symmetric difference, and initiates resolution of missing transactions. This is the bandwidth-efficient complement to low-fanout flooding, ensuring all peers eventually learn about all transactions.

## Specification

### Reconciliation Loop

```rust
use crate::constants::ERLAY_RECONCILIATION_INTERVAL_MS;
use tokio::time::{interval, Duration};

impl ErlayState {
    /// Run the reconciliation loop for a single non-flood peer.
    /// Called as a spawned task per non-flood peer.
    pub async fn reconciliation_loop(
        &mut self,
        peer_id: PeerId,
        peer_conn: &PeerConnection,
    ) -> Result<(), GossipError> {
        let mut timer = interval(
            Duration::from_millis(ERLAY_RECONCILIATION_INTERVAL_MS)
        );

        loop {
            timer.tick().await;

            // Skip if peer is in flood set (they get direct flooding)
            if self.flood_set.contains(&peer_id) {
                continue;
            }

            // Step a: Compute minisketch of local tx_ids added since last reconciliation
            let pending_tx_ids = self.reconciliation_sets
                .get(&peer_id)
                .cloned()
                .unwrap_or_default();

            let local_sketch = encode_sketch(&pending_tx_ids)?;

            // Step b: Send ReconciliationSketch to peer
            peer_conn.peer.send(ReconciliationSketch {
                sketch: local_sketch.clone(),
                sketch_capacity: ERLAY_SKETCH_CAPACITY as u32,
            }).await?;

            // Step c: Receive peer's sketch
            let remote_sketch = peer_conn.receive_sketch().await?;

            // Step d-e: Compute symmetric difference
            let differences = decode_difference(
                &local_sketch,
                &remote_sketch.sketch,
            )?;

            // Step f-g: Resolve differences (see ERL-005)
            self.resolve_differences(
                &peer_id,
                &differences,
                &pending_tx_ids,
                peer_conn,
            ).await?;

            // Clear pending set for this peer after successful reconciliation
            self.reconciliation_sets
                .entry(peer_id.clone())
                .or_default()
                .clear();
        }
    }
}
```

### Constants

```rust
/// Set reconciliation interval per peer (ms).
/// Default: 2000.
pub const ERLAY_RECONCILIATION_INTERVAL_MS: u64 = 2000;
```

### Protocol Sequence

```
Every ERLAY_RECONCILIATION_INTERVAL_MS (2000ms) per non-flood peer:
  a. Compute minisketch of local tx_ids added since last reconciliation
  b. Send ReconciliationSketch { sketch, sketch_capacity }
  c. Receive peer's ReconciliationSketch
  d. Compute symmetric difference (XOR of sketches)
  e. Decode difference -> set of tx_id prefixes differing between peers
  f. Request missing tx_ids via RequestTransaction (see ERL-005)
  g. Send tx_ids the peer is missing via RespondTransaction (see ERL-005)
```

### Invariants

- Reconciliation MUST run independently per non-flood peer with its own timer.
- Reconciliation MUST NOT run for peers currently in the flood set.
- The reconciliation set MUST be cleared after a successful round to prevent re-reconciling already-handled transactions.
- If sketch decoding fails (difference exceeds capacity), the node MUST fall back to requesting a full tx_id list from the peer.
- The interval MUST be configurable via `ErlayConfig.reconciliation_interval_ms`.

## Acceptance Criteria

- [ ] Reconciliation triggers every `ERLAY_RECONCILIATION_INTERVAL_MS` (2000ms) per non-flood peer
- [ ] Reconciliation does not trigger for flood-set peers
- [ ] Local sketch is computed from pending tx_ids accumulated since last reconciliation
- [ ] `ReconciliationSketch` message is sent to the peer containing the serialized sketch
- [ ] Remote peer's sketch is received and decoded
- [ ] Symmetric difference is computed by merging (XOR) local and remote sketches
- [ ] Pending reconciliation set is cleared after a successful round
- [ ] On decode failure, fallback to full tx_id list exchange occurs
- [ ] `ERLAY_RECONCILIATION_INTERVAL_MS` constant is defined as `2000` in `constants.rs`
- [ ] Timer runs independently per peer (not a single global timer)

## Implementation Notes

- Primary file: `src/gossip/erlay.rs`
- Constants file: `src/constants.rs`
- Dependencies: `tokio` for async timers, `minisketch-rs` for sketch operations, `chia-protocol` for message types
- Edge cases:
  - Peer disconnects mid-reconciliation: the loop should terminate gracefully
  - Empty pending set: still send a sketch (it will have zero elements); this confirms "nothing new" to the peer
  - Peer moved into flood set between timer ticks: skip reconciliation for that tick
  - High transaction volume causing pending set > sketch capacity: decoding may fail; fall back to full exchange
  - Network latency causing overlapping reconciliation rounds: each round should be independent; use sequence numbers or timestamps if needed
