# ERL-008: ErlayConfig Struct

## Summary

ErlayConfig is the configuration struct for ERLAY-style transaction relay. It controls the number of peers receiving immediate `NewTransaction` flooding, the interval between periodic set reconciliation rounds with non-flood peers, and the minisketch sketch capacity (maximum decodable symmetric difference per reconciliation round). All fields have sensible defaults matching the ERLAY paper recommendations.

## Specification

### Struct Definition

```rust
/// Configuration for ERLAY-style transaction relay.
/// SPEC Section 8.3.
pub struct ErlayConfig {
    /// Number of peers to flood NewTransaction to immediately.
    /// Remaining peers use set reconciliation.
    /// Default: 8 (matching ERLAY paper recommendation).
    pub flood_peer_count: usize,
    /// Interval between reconciliation rounds per peer (ms).
    /// Default: 2000.
    pub reconciliation_interval_ms: u64,
    /// Minisketch capacity (max set difference decodable per round).
    /// Default: 20 (handles up to 20 tx difference per reconciliation).
    pub sketch_capacity: usize,
}
```

### Relationship to Constants

The default values of `ErlayConfig` fields MUST match the corresponding constants defined in `src/constants.rs`:
- `flood_peer_count` defaults to `ERLAY_FLOOD_PEER_COUNT` (8)
- `reconciliation_interval_ms` defaults to `ERLAY_RECONCILIATION_INTERVAL_MS` (2000)
- `sketch_capacity` defaults to `ERLAY_SKETCH_CAPACITY` (20)

### Usage

`ErlayConfig` is an optional sub-config of `GossipConfig`. When the `erlay` feature is enabled, the gossip service reads these values to configure the ERLAY state machine (ERL-001 through ERL-007).

## Implementation Notes

- Primary file: `src/types/config.rs` (or `src/gossip/erlay.rs`)
- Dependencies: None (pure data struct)
- Edge cases:
  - `flood_peer_count = 0` disables flooding entirely; all relay is via reconciliation
  - `sketch_capacity` must be > 0; a capacity of 0 means no differences can be decoded
  - Very short `reconciliation_interval_ms` (e.g., < 100ms) may create excessive network traffic
