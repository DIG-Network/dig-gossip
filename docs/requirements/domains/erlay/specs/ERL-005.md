# ERL-005: Symmetric Difference Resolution

> **Authoritative requirement:** [NORMATIVE.md](../NORMATIVE.md)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **Spec reference:** [SPEC.md - Section 8.3](../../../resources/SPEC.md)

## Summary

After decoding the symmetric difference from minisketch, both peers must resolve which transactions each side is missing. The local node requests tx_ids it does not have via `RequestTransaction` and sends tx_ids the remote peer is missing via `RespondTransaction`. After a successful reconciliation round, both peers MUST converge to the same transaction set (for the transactions covered by that round).

## Specification

### Difference Resolution Algorithm

```rust
use chia_protocol::{Bytes32, RequestTransaction, RespondTransaction};

impl ErlayState {
    /// Resolve the symmetric difference after sketch decoding.
    /// - Elements in the difference that we DON'T have locally: request from peer.
    /// - Elements in the difference that we DO have locally: send to peer.
    pub async fn resolve_differences(
        &self,
        peer_id: &PeerId,
        differences: &[u64],
        local_tx_ids: &HashSet<Bytes32>,
        peer_conn: &PeerConnection,
    ) -> Result<(), GossipError> {
        // Build a lookup of local tx_id truncations for matching
        let local_truncated: HashMap<u64, Bytes32> = local_tx_ids
            .iter()
            .map(|tx_id| {
                let truncated = u64::from_le_bytes(
                    tx_id.as_ref()[..8].try_into().unwrap()
                );
                (truncated, *tx_id)
            })
            .collect();

        let mut tx_ids_to_request: Vec<u64> = Vec::new();
        let mut tx_ids_to_send: Vec<Bytes32> = Vec::new();

        for &diff_element in differences {
            if let Some(full_tx_id) = local_truncated.get(&diff_element) {
                // We have it, peer doesn't -> send to peer
                tx_ids_to_send.push(*full_tx_id);
            } else {
                // Peer has it, we don't -> request from peer
                tx_ids_to_request.push(diff_element);
            }
        }

        // Request missing transactions from peer
        for truncated_id in &tx_ids_to_request {
            // The peer maps truncated IDs back to full tx_ids on their side
            peer_conn.peer.send(RequestTransaction {
                transaction_id: Bytes32::from(/* reconstruct or use truncated lookup */),
            }).await?;
        }

        // Send transactions the peer is missing
        for tx_id in &tx_ids_to_send {
            // Look up the full transaction from mempool and send
            if let Some(spend_bundle) = self.get_transaction(tx_id)? {
                peer_conn.peer.send(RespondTransaction {
                    transaction: spend_bundle,
                }).await?;
            }
        }

        Ok(())
    }
}
```

### Protocol Flow

```
After decode_difference() returns a set of differing tx_id prefixes:

  For each element in the symmetric difference:
    IF element matches a local tx_id:
      -> We have it, peer doesn't
      -> Send RespondTransaction { transaction } to peer
    ELSE:
      -> Peer has it, we don't
      -> Send RequestTransaction { transaction_id } to peer
      -> Receive RespondTransaction { transaction } from peer

  After all requests/responses complete:
    Both peers have the union of their transaction sets.
```

### Invariants

- Every element in the decoded symmetric difference MUST be resolved (either requested or sent).
- `RequestTransaction` MUST be sent for each tx_id the local node does not possess.
- `RespondTransaction` MUST be sent for each tx_id the local node possesses but the peer does not.
- After a successful reconciliation round, the symmetric difference between the two peers' tx sets (for the reconciled set) MUST be empty.
- If `RequestTransaction` receives no response (timeout), the tx_id is retried in the next reconciliation round.
- The resolution process uses `chia-protocol`'s existing `RequestTransaction` and `RespondTransaction` message types -- no custom messages are needed.

## Acceptance Criteria

- [ ] `resolve_differences` correctly classifies each difference element as "we have / peer missing" or "peer has / we missing"
- [ ] `RequestTransaction` is sent for each tx_id the local node does not have
- [ ] `RespondTransaction` is sent for each tx_id the peer does not have
- [ ] After resolution, both peers possess the union of their pre-reconciliation tx sets
- [ ] Missing transactions that time out on request are not lost (retried next round)
- [ ] The method uses `chia-protocol` `RequestTransaction` and `RespondTransaction` types
- [ ] Empty symmetric difference (identical sets) results in no requests or sends
- [ ] Large symmetric difference (all 20 elements) is handled correctly

## Implementation Notes

- Primary file: `src/gossip/erlay.rs`
- Dependencies: `chia-protocol` for `RequestTransaction`, `RespondTransaction`, `SpendBundle`; mempool access for looking up full transactions
- Edge cases:
  - Truncated tx_id collision: two different tx_ids map to the same 64-bit prefix. Extremely unlikely (2^-64) but should be handled by requesting the full tx_id and verifying
  - Transaction evicted from mempool between reconciliation and resolution: the `RespondTransaction` for that tx_id would fail; peer retries next round
  - Peer sends a `RespondTransaction` for a tx_id we already received (race with flooding): deduplicate on receipt
  - Both peers simultaneously request from each other: this is expected and correct; ERLAY symmetric difference is symmetric
