# PLT-005: Lazy Timeout and GRAFT

> **Authoritative requirement:** [NORMATIVE.md](../NORMATIVE.md)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **Spec reference:** [SPEC.md - Section 8.1](../../../resources/SPEC.md)

## Summary

When a `LazyAnnounce` is received for a hash not yet in the seen set, the hash and announcer are recorded in the `lazy_queue` with a timestamp. If the full message does not arrive via eager push within `lazy_timeout_ms`, the protocol sends a GRAFT message (promoting the lazy announcer to eager) along with a `RequestByHash` to pull the missing message. This mechanism repairs gaps in the spanning tree caused by failed eager links.

## Specification

### On Receiving LazyAnnounce

```rust
impl PlumtreeState {
    /// Handle receipt of a LazyAnnounce from a lazy peer.
    ///
    /// If the hash is already seen, the announcement is ignored.
    /// Otherwise, the announcer and timestamp are queued for timeout.
    pub fn on_lazy_announce(
        &mut self,
        from: PeerId,
        hash: Bytes32,
        seen_set: &SeenSet,
    ) -> bool {
        if seen_set.contains(&hash) {
            return false; // Already have this message
        }

        // Record the announcer and current timestamp
        self.lazy_queue
            .entry(hash)
            .or_insert_with(Vec::new)
            .push((from, now_ms()));

        true // Timer started
    }
}
```

### Lazy Timeout Processing

```rust
impl PlumtreeState {
    /// Check for lazy announcements that have timed out.
    ///
    /// Called periodically (e.g., every lazy_timeout_ms / 2).
    /// For each timed-out hash, sends GRAFT + RequestByHash to the
    /// first announcer and promotes them to eager.
    pub async fn process_lazy_timeouts(
        &mut self,
        now: u64,
        peers: &PeerMap,
    ) -> Vec<(PeerId, Bytes32)> {
        let mut grafted = Vec::new();
        let timeout = self.lazy_timeout_ms;

        let expired: Vec<Bytes32> = self.lazy_queue.iter()
            .filter(|(_, entries)| {
                entries.iter().any(|(_, ts)| now - ts >= timeout)
            })
            .map(|(hash, _)| *hash)
            .collect();

        for hash in expired {
            if let Some(entries) = self.lazy_queue.remove(&hash) {
                // Use the first announcer that has timed out
                if let Some((announcer, _)) = entries.iter()
                    .find(|(_, ts)| now - ts >= timeout)
                {
                    let announcer = *announcer;

                    // Promote announcer: lazy -> eager
                    self.lazy_peers.remove(&announcer);
                    self.eager_peers.insert(announcer);

                    // Send GRAFT + RequestByHash
                    if let Some(peer) = peers.get(&announcer) {
                        peer.send_raw(Graft { hash }.to_message()).await;
                        peer.send_raw(
                            RequestByHash { hash }.to_message()
                        ).await;
                    }

                    grafted.push((announcer, hash));
                }
            }
        }

        grafted
    }
}
```

### On Receiving GRAFT

```rust
impl PlumtreeState {
    /// Handle receipt of GRAFT from a peer.
    ///
    /// The peer wants to be promoted back to eager. If we have the
    /// message in cache, send it.
    pub async fn on_graft(
        &mut self,
        from: PeerId,
        hash: Bytes32,
        message_cache: &MessageCache,
        peers: &PeerMap,
    ) {
        // Promote requester: lazy -> eager
        self.lazy_peers.remove(&from);
        self.eager_peers.insert(from);

        // Serve the message from cache if available
        if let Some(message) = message_cache.get(&hash) {
            if let Some(peer) = peers.get(&from) {
                peer.send_raw(message.clone()).await;
            }
        }
    }
}
```

### State Transition Diagram

```
  LazyAnnounce(hash) from peer_b
         |
         v
  hash in seen_set? --yes--> ignore
         |no
         v
  lazy_queue.insert(hash, (peer_b, now))
         |
         |  ... lazy_timeout_ms elapses ...
         |  ... hash NOT received eagerly ...
         v
  timeout fires:
    1. lazy_peers.remove(peer_b)
    2. eager_peers.insert(peer_b)
    3. send GRAFT{hash} to peer_b
    4. send RequestByHash{hash} to peer_b
         |
         v
  peer_b receives GRAFT:
    1. lazy_peers.remove(us)
    2. eager_peers.insert(us)
    3. if cache has message: send_raw(message) to us
```

### Behavior Rules

- If the hash is already in the seen set when `LazyAnnounce` arrives, the announcement MUST be ignored.
- Multiple announcers for the same hash MUST be recorded (the first to time out wins).
- After timeout, the announcer MUST be promoted from `lazy_peers` to `eager_peers`.
- Both GRAFT and `RequestByHash` MUST be sent to the announcer.
- On receiving GRAFT, the requester MUST be promoted from `lazy_peers` to `eager_peers`.
- On receiving GRAFT, if the message is in cache, it MUST be sent to the requester.
- If the message arrives eagerly before timeout, the lazy_queue entry MUST be cancelled (handled in PLT-004).

## Acceptance Criteria

- [ ] `on_lazy_announce` ignores hashes already in the seen set
- [ ] `on_lazy_announce` records (announcer, timestamp) in `lazy_queue` for unseen hashes
- [ ] `on_lazy_announce` supports multiple announcers for the same hash
- [ ] `process_lazy_timeouts` fires after `lazy_timeout_ms` elapses without eager receipt
- [ ] On timeout, announcer is promoted from `lazy_peers` to `eager_peers`
- [ ] On timeout, GRAFT message is sent to the announcer
- [ ] On timeout, `RequestByHash{hash}` is sent to the announcer
- [ ] `on_graft` promotes the requester from `lazy_peers` to `eager_peers`
- [ ] `on_graft` sends the cached message if available
- [ ] `on_graft` does not error if message is not in cache
- [ ] Lazy queue entry is removed after timeout processing

## Implementation Notes

- Primary file: `src/gossip/plumtree.rs`
- Dependencies: `SeenSet` (PLT-008), `MessageCache` (PLT-007), GRAFT and RequestByHash DIG message types
- Edge cases:
  - Multiple lazy peers may announce the same hash; the first to time out triggers the GRAFT, but others should be cleaned up
  - If the announcer disconnects before the timeout fires, `process_lazy_timeouts` should skip it and try the next announcer (PLT-006 handles promotion of alternatives)
  - The timeout check should use `>=` to avoid off-by-one issues with timer precision
  - `process_lazy_timeouts` should be called periodically, not once per announcement; a tick interval of `lazy_timeout_ms / 2` or similar is recommended
  - GRAFT and RequestByHash can be sent as separate messages or combined into a single frame, depending on the transport layer
