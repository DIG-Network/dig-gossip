# PLT-008: Seen Set

> **Authoritative requirement:** [NORMATIVE.md](../NORMATIVE.md)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **Spec reference:** [SPEC.md - Section 8.1](../../../resources/SPEC.md)

## Summary

The seen set is an LRU-based deduplication filter that tracks message hashes to prevent processing the same message more than once. Every incoming message is checked against the seen set before processing; if the hash is already present, the message is dropped immediately. The hash is computed as SHA256(msg_type || data). The seen set has a capacity of 100,000 entries, large enough to cover the expected message volume while bounding memory usage.

## Specification

### SeenSet Structure

```rust
use lru::LruCache;
use std::num::NonZeroUsize;

/// LRU-based message deduplication set.
///
/// Tracks message hashes to detect and drop duplicates.
/// Capacity: 100,000 entries.
pub struct SeenSet {
    /// LRU set of message hashes. The value is () since we only
    /// need presence tracking, but LruCache requires a value type.
    set: LruCache<Bytes32, ()>,
}

impl SeenSet {
    /// Default capacity of the seen set.
    pub const DEFAULT_CAPACITY: usize = 100_000;

    pub fn new() -> Self {
        Self {
            set: LruCache::new(
                NonZeroUsize::new(Self::DEFAULT_CAPACITY).unwrap()
            ),
        }
    }

    /// Create a seen set with a custom capacity.
    pub fn with_capacity(capacity: usize) -> Self {
        Self {
            set: LruCache::new(
                NonZeroUsize::new(capacity).unwrap()
            ),
        }
    }

    /// Check if a hash has been seen.
    ///
    /// Returns true if the hash is in the set (message already processed).
    /// Also promotes the entry to most-recently-used.
    pub fn contains(&mut self, hash: &Bytes32) -> bool {
        self.set.get(hash).is_some()
    }

    /// Insert a hash into the seen set.
    ///
    /// If the set is at capacity, the least-recently-used entry is evicted.
    pub fn insert(&mut self, hash: Bytes32) {
        self.set.put(hash, ());
    }

    /// Current number of entries in the seen set.
    pub fn len(&self) -> usize {
        self.set.len()
    }

    /// Check if the seen set is empty.
    pub fn is_empty(&self) -> bool {
        self.set.is_empty()
    }
}
```

### Hash Computation

```rust
/// Compute the message hash for deduplication.
///
/// hash = SHA256(msg_type_u16_be || message_data)
///
/// This is the same hash used for lazy announcements (PLT-003)
/// and message cache keys (PLT-007).
pub fn compute_message_hash(msg_type: u16, data: &[u8]) -> Bytes32 {
    let mut hasher = Sha256::new();
    hasher.update(&msg_type.to_be_bytes());
    hasher.update(data);
    Bytes32::from(hasher.finalize())
}
```

### Usage in Broadcast Pipeline

```rust
// Entry point for all incoming and locally-originated messages:
fn broadcast(message: &Message, origin: Option<PeerId>) -> usize {
    let hash = compute_message_hash(message.msg_type, &message.data);

    // Step 2: Duplicate check
    if seen_set.contains(&hash) {
        return 0; // Already seen - drop immediately
    }

    // Step 3: Mark as seen
    seen_set.insert(hash);

    // Steps 4-8: Process, eager push, lazy push, relay, etc.
    // ...
}
```

### Usage in Eager Receive

```rust
// In on_eager_receive (PLT-004):
let hash = compute_message_hash(message.msg_type, &message.data);
if seen_set.contains(&hash) {
    // Duplicate from eager peer -> PRUNE
    // ...
}
```

### Usage in Lazy Announce Receive

```rust
// In on_lazy_announce (PLT-005):
if seen_set.contains(&hash) {
    return; // Already have this message, ignore announcement
}
```

### Behavior Rules

- Every incoming message MUST be checked against the seen set before processing.
- If the hash is in the seen set, the message MUST be dropped immediately (no further processing).
- After processing a new message, its hash MUST be inserted into the seen set.
- The seen set MUST use LRU eviction with a capacity of 100,000 entries.
- The hash MUST be computed as `SHA256(msg_type as u16 big-endian || data)`.
- The same hash function MUST be used across all Plumtree components (broadcast, eager receive, lazy announce, message cache).
- When capacity is reached, the least-recently-used entry MUST be evicted to make room.

## Acceptance Criteria

- [ ] `SeenSet::new()` creates a set with capacity 100,000
- [ ] `insert()` adds a hash to the set
- [ ] `contains()` returns `true` for a hash in the set
- [ ] `contains()` returns `false` for a hash not in the set
- [ ] Messages with hashes in the seen set are dropped immediately in broadcast
- [ ] Messages with hashes in the seen set trigger PRUNE in eager receive (PLT-004)
- [ ] Lazy announcements for hashes in the seen set are ignored (PLT-005)
- [ ] LRU eviction occurs when capacity exceeds 100,000
- [ ] Evicted hashes are no longer reported as seen (`contains` returns `false`)
- [ ] Hash is computed as `SHA256(msg_type_u16_be || data)`
- [ ] Same hash function is used consistently across all components

## Implementation Notes

- Primary file: `src/gossip/seen_set.rs`
- Dependencies: `lru` crate, `Bytes32` from chia-protocol, `sha2` crate for SHA256
- Edge cases:
  - After LRU eviction, an old message may be re-processed if received again; this is acceptable as the alternative (unbounded memory) is worse
  - The `contains()` method promotes the entry to most-recently-used via `LruCache::get`; this means frequently-seen hashes stay in the set longer
  - The hash function combines `msg_type` and `data` to ensure messages with the same payload but different types produce different hashes
  - `with_capacity()` allows test code to create smaller seen sets for easier testing of eviction behavior
  - Thread safety: if accessed from multiple tasks, wrap in `Mutex` or use the seen set from a single task with message-passing
  - The 100,000 capacity at 32 bytes per hash uses approximately 3.2 MB of memory for hashes alone, plus LRU overhead
