# PLT-007: Message Cache

> **Authoritative requirement:** [NORMATIVE.md](../NORMATIVE.md)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **Spec reference:** [SPEC.md - Section 8.1](../../../resources/SPEC.md)

## Summary

The message cache stores recently broadcast messages so they can be served in response to GRAFT requests. When a peer sends a GRAFT for a specific hash, the cache is consulted to retrieve the full message and send it back. The cache uses LRU eviction with a capacity of 1000 entries and a TTL of 60 seconds per entry, ensuring bounded memory usage while retaining messages long enough for the tree repair cycle to complete.

## Specification

### MessageCache Structure

```rust
use lru::LruCache;
use std::num::NonZeroUsize;
use std::time::Instant;

/// Cached message entry with insertion timestamp for TTL expiry.
struct CacheEntry {
    /// The full message payload.
    message: Message,
    /// When this entry was inserted (for TTL computation).
    inserted_at: Instant,
}

/// LRU message cache for serving GRAFT responses.
///
/// Capacity: 1000 entries.
/// TTL: 60 seconds per entry.
pub struct MessageCache {
    /// LRU cache mapping message hash -> cached entry.
    cache: LruCache<Bytes32, CacheEntry>,
    /// Time-to-live for each entry in milliseconds.
    ttl_ms: u64,
}

impl MessageCache {
    /// Capacity of the message cache.
    pub const DEFAULT_CAPACITY: usize = 1000;

    /// Default TTL in milliseconds.
    pub const DEFAULT_TTL_MS: u64 = 60_000;

    pub fn new() -> Self {
        Self {
            cache: LruCache::new(
                NonZeroUsize::new(Self::DEFAULT_CAPACITY).unwrap()
            ),
            ttl_ms: Self::DEFAULT_TTL_MS,
        }
    }

    /// Insert a message into the cache, keyed by its hash.
    pub fn insert(&mut self, hash: Bytes32, message: Message) {
        self.cache.put(hash, CacheEntry {
            message,
            inserted_at: Instant::now(),
        });
    }

    /// Retrieve a message by hash, returning None if not present
    /// or if the entry has expired (TTL exceeded).
    pub fn get(&mut self, hash: &Bytes32) -> Option<&Message> {
        if let Some(entry) = self.cache.get(hash) {
            if entry.inserted_at.elapsed().as_millis() as u64 <= self.ttl_ms {
                return Some(&entry.message);
            }
            // Entry expired - remove it
            self.cache.pop(hash);
        }
        None
    }

    /// Remove expired entries. Called periodically to free memory.
    pub fn evict_expired(&mut self) {
        let ttl = self.ttl_ms;
        // Collect expired keys
        let expired: Vec<Bytes32> = self.cache.iter()
            .filter(|(_, entry)| {
                entry.inserted_at.elapsed().as_millis() as u64 > ttl
            })
            .map(|(hash, _)| *hash)
            .collect();

        for hash in expired {
            self.cache.pop(&hash);
        }
    }

    /// Current number of entries in the cache.
    pub fn len(&self) -> usize {
        self.cache.len()
    }

    /// Check if the cache is empty.
    pub fn is_empty(&self) -> bool {
        self.cache.is_empty()
    }
}
```

### Usage in GRAFT Response

```rust
// In PlumtreeState::on_graft (see PLT-005):
if let Some(message) = message_cache.get(&hash) {
    peer.send_raw(message.clone()).await;
}
```

### Usage in Broadcast

```rust
// In broadcast pipeline, after seen_set.insert(hash):
message_cache.insert(hash, message.clone());
```

### Behavior Rules

- The cache MUST use LRU eviction with a capacity of 1000 entries.
- Entries MUST expire after 60 seconds (TTL), regardless of LRU position.
- `get()` MUST return `None` for expired entries, even if they are still in the LRU.
- Expired entries SHOULD be lazily removed on access or periodically via `evict_expired()`.
- Every newly broadcast message MUST be inserted into the cache.
- GRAFT responses MUST be served from the cache.
- If a GRAFT requests a hash not in the cache (evicted or expired), no message is sent.

## Acceptance Criteria

- [ ] `MessageCache::new()` creates a cache with capacity 1000
- [ ] `MessageCache::new()` creates a cache with TTL of 60 seconds
- [ ] `insert()` stores the message keyed by hash
- [ ] `get()` retrieves a message by hash when present and not expired
- [ ] `get()` returns `None` for a hash not in the cache
- [ ] `get()` returns `None` for an expired entry (older than 60 seconds)
- [ ] LRU eviction occurs when capacity exceeds 1000 (oldest-accessed entry evicted)
- [ ] `evict_expired()` removes all entries older than TTL
- [ ] Inserting the 1001st entry evicts the least-recently-used entry
- [ ] Cache is used to serve GRAFT responses
- [ ] Every broadcast message is inserted into the cache

## Implementation Notes

- Primary file: `src/gossip/message_cache.rs`
- Dependencies: `lru` crate, `Bytes32` from chia-protocol, `Message` from chia-protocol
- Edge cases:
  - Inserting the same hash again updates the entry and resets the TTL (LRU `put` replaces)
  - TTL expiry and LRU eviction are independent mechanisms; an entry can be evicted by LRU before TTL expires, or expire by TTL before LRU eviction
  - `evict_expired()` should be called periodically (e.g., every 10 seconds) to prevent memory accumulation from entries that are expired but not accessed
  - The `Message` type may be large (up to several MB for block messages); capacity of 1000 should be tuned based on expected message sizes in production
  - Thread safety: if accessed from multiple tasks, wrap in `Mutex` or use per-task cache instances
