# PLT-002: Eager Push

> **Authoritative requirement:** [NORMATIVE.md](../NORMATIVE.md)
> **Verification:** [VERIFICATION.md](../VERIFICATION.md)
> **Tracking:** [TRACKING.yaml](../TRACKING.yaml)
> **Spec reference:** [SPEC.md - Section 8.1](../../../resources/SPEC.md)

## Summary

Eager push is the primary message dissemination mechanism in Plumtree. When a new message is broadcast, the full message is sent immediately to all peers in the `eager_peers` set, excluding the origin peer (the peer that sent us the message). This forms the spanning tree edges that carry full message payloads with minimal latency.

## Specification

### Eager Push Algorithm

```rust
impl PlumtreeState {
    /// Send the full message to all eager peers, excluding the origin.
    ///
    /// Returns the number of peers the message was sent to.
    pub async fn eager_push(
        &self,
        message: &Message,
        origin: Option<&PeerId>,
        peers: &PeerMap,
    ) -> usize {
        let mut count = 0;
        for peer_id in &self.eager_peers {
            // Skip the origin peer to avoid sending back to sender
            if origin.map_or(false, |o| o == peer_id) {
                continue;
            }
            if let Some(peer) = peers.get(peer_id) {
                peer.send_raw(message.clone()).await;
                count += 1;
            }
        }
        count
    }
}
```

### Integration in Broadcast

```
broadcast(message: Message, origin: Option<PeerId>):
  1. hash = SHA256(message.msg_type || message.data)
  2. if seen_set.contains(hash) -> return 0
  3. seen_set.insert(hash)
  4. Deliver to local inbound channel
  5. eager_push(message, origin, peers)    // <-- THIS REQUIREMENT
  6. lazy_push(hash, msg_type, origin)
  7. If relay connected: relay.broadcast(message, exclude_list)
  8. Return count sent
```

### Behavior Rules

- The origin peer (sender of the message to us) MUST be excluded from eager push to prevent message reflection.
- If `origin` is `None` (locally originated message), all eager peers receive the message.
- `peer.send_raw(message)` is the transport-level send that writes the serialized message to the peer's outbound buffer.
- Eager push MUST NOT block on individual peer sends; failures on one peer MUST NOT prevent sending to other peers.
- The message MUST be sent as-is (no re-encoding or wrapping) via `send_raw`.

## Acceptance Criteria

- [ ] `eager_push` sends the full `Message` to each peer in `eager_peers`
- [ ] `eager_push` excludes the `origin` peer from the send list
- [ ] `eager_push` sends to all eager peers when `origin` is `None`
- [ ] `eager_push` uses `peer.send_raw(message)` for transport
- [ ] `eager_push` returns the count of peers the message was sent to
- [ ] A single peer send failure does not prevent sending to remaining peers
- [ ] Peers not in `eager_peers` (i.e., those in `lazy_peers`) do not receive the full message via eager push

## Implementation Notes

- Primary file: `src/gossip/plumtree.rs`
- Dependencies: `PeerMap` (peer connection registry), `Message` from chia-protocol, `PeerId`
- Edge cases:
  - If `eager_peers` is empty (all peers demoted to lazy), no messages are eagerly pushed; the tree is effectively broken and will self-heal via lazy timeout (PLT-005)
  - If the origin peer is not in `eager_peers` (e.g., it was already demoted), the exclusion check is harmless
  - Send failures should be logged via `tracing::warn!` but not propagated as errors
  - Concurrent peer disconnection during iteration: the `PeerMap` lookup may return `None` for recently disconnected peers; these are silently skipped
